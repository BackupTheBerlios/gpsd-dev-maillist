I already did and it passed just fine. I've been talking with Gary
Miller and Chris Kuethe off-list and have narrowed the problem down to
using or not using ARM GNU EABI. If I am not using it I get alignment
warnings when compiling garmin.c and get incorrect lat/long. If I use
EABI I don't get the alignment warnings and I get correct lat/long.

There's a quick FAQ on EABI here with some links:
http://www.openembedded.org/wiki/OeFaq
which are:
http://www.codesourcery.com/gnu_toolchains/arm/faq.html#q_gnu_linux_long_long
http://www.arm.com/products/DevTools/ABI.html
http://wiki.debian.org/ArmEabiPort

If you're interested in helping to debug this problem let me know and
I'll give you my logs.
