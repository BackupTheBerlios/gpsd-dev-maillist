Chris,
Technically, if you get any value for 0/0 you are doing good.  What you
are asking; is for a computer to determine what direction a zero length
vector is heading without knowledge of the limits on either side.  What
is this trying to prove?  

I tried Perl and it always returns zero but this is just as arbitrary.
perl -e '$a=$b=0.0;$a*=-1.0;
$c=atan2($a,$a);$d=atan2($b,$b);$e=atan2($a,$b);$f=atan2($b,$a);
print sprintf("%f\t%f\t%f\t%f", $c, $d, $e, $f), "\n";
print sprintf("%g\t%g\t%g\t%g", $c, $d, $e, $f), "\n";'

On 2.6.12-1.1381_FC3-i386 the C program returned 
$ gcc -lm zero.c -o zero; ./zero
0.000000 -3.141593 -0.000000 3.141593 0.000000 -0.000000
0 -3.14159 -0 3.14159 0 -0

One suggestion is to rewrite the atan2 function in C so that 0/0
returns zero.
