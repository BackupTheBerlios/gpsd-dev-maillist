I've noticed a minor, easy-to-fix bug in the file
/etc/udev/rules.d/50-gpsd.rules on Ubuntu 8.10

Namely, when the udev rules run SYMLINK="gps%n" it clears the list of
symlinks. So if I've got one of the usb-serial ports tested, and I've
got a rule set up in 10-local.rules adding a symlink for my usb-serial
port, when the gpsd rules are subsequently run, my symlink is removed.

Fix is to change from SYMLINK="gps%n" to SYMLINK+="gps%n" which adds
the symlink without clearing the list of symlinks.
