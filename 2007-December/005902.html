Exactly, but the same goes for subversion imho. Too many people are
using it, so there's rarely a chance to drop it.

What I know from several other projects is that the main tree is hosted
in svn - on sf.net or somewhere else. But at the same time one (or even
several) git tree exists, which is/are synced from and to svn and used
by some (or all) developers. Attaching git to svn is supposed to be
pretty easy and can be done by every developer. Probably an idea to use
both worlds.
