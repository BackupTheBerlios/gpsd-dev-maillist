Ujjwal Dasgupta <ujjwaldasgupta at gmail.com>:
> From what I have understood of gpsd till now, there are two approaches for
> building clients which can read GPS data. One is to communicate with the
> gpsd daemon using network protocols, and the other is to use libgps to
> directly access GPS data through programs. I had some problems with using
> libgps as I could not find documentation about the required functions, but I
> was able to communicate with the gpsd daemon using telnet.
> 
> I was confused about which approach I should go with, so any help with that
> would be much appreciated.

We recommend using libgps; it will insultate you from changes in the
reporting protocol.

I tried to document all the required functions in libgps(3).  You can find
usage examples in cgps.c and xgps.c.  If this material is inadequate to
teach people how to write client programs, that's a problem I want to fix.

If, after reading libgps(3), you are still confused, please ask me questions.
Perhaps that will help me improve the documentation.
