Rick Niles <niles at rickniles.com>:
> While not in the offical NEMA standard, there are receivers that deliver 
> low-level data important for various applications of GPS.  Things like 
> raw pseudo-range and carrier-phase.  Also, while all the current 
> supported recievers are using L1, that are L2 codeless and semi-codeless 
> recvrs out there and with any luck we'll have L2 and L5 satellites up in 
> a few years.  When dealing with augmentation systems like LAAS and WAAS, 
> it's important to be able to break out the particular data format from 
> those augmentation systems.   I would like to suggest a whole bunch of 
> additional important data that could be included in the data structure, 
> but I'm just wondering how receptive you are to adding this additional 
> information?   Perhaps, gpsd should be kept to just simple basics, but 
> I'd love to see a daemon that can handle the whole range of GPS data.

I'm not necessarily opposed to the idea, but -- how is this data reported
from the GPS if not through the NMEA data stream?

If there is some standard set of extension data items (like LAAS/WAAS data) 
that several different receivers report in compatible formats, then I'd look
seriously at adding support for it.

> BTW, gpsd seems to work perfect with NovaTel OEM 3 (1999-2000 vintage) 
> and OEM4 (2001-current) recvrs.

Here's what I just added to the documentation about how to report hardware:

<ol>
<li>The GPS model name</li>
<li>Whether or not it works</li>
<li>If possible, an URL to a technical reference on it</li>
<li>The GPS and (if applicable) USB-to-serial chip it uses</li>
<li>The interface type (RS232, USB, CF)</li>
<li>The vendor</li>
<li>The gpsd driver type (almost always 'n')</li>
<li>Version of gpsd you tested with</li>
<li>NMEA version it emits, or '?' if the documentation doesn't say</li>
<li>A list of NMEA sentences it emits</li>
<li>Notes on the device</li>
</ol>

<p>You should be able to determine many of these things by running
gpsprobe against the device and looking at the output.</p>
