I've got some recent code of mine that uses g_io_channels, which was fun
to do and can be used as a small example.  g_io_channels are a
reasonably well thought out event handling environment.  It's part of
glib.

http://quozl.linux.org.au/knoppix/compressloop/compressloop-distributed.c

When a slave connection is established, the socket file descriptor is
identified as a g_io_channel, and then a callback is requested for
readability and writeability.

g_main_run() is where all the work is done in this program, and glib
calls the callbacks as the socket becomes active.

The documentation was good, but I think at one point I ended up diving
under the hood to better understand it.
