> But...why a kernel-level driver?  I don't see any reason not to stay
> in userspace.  Makes debugging simpler.

  Why not?  The driver is far more capable of handling the device, given the 
CY4601s serial based platform.  It is a fully capable serial device, so 
providing this tty interface to it makes perfect sense.  I would love to 
access it in userspace, but have not found a clear way of doing so as 
described above.  In windows, the hid api provides read / write 
functionality, which is all one really needs with it.  It uses zero ioctls, 
and all line handling is done in the first byte of each transfer.

(From a host's perspective)
Outgoing:
[line control][mesg. length][ .....  data .....]
Incoming:
[uart status][mesg. length][ ...... data ......]

  It could not get much simpler than that.
