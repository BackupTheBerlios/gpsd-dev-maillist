described in the documentation I have now. For example, it is possible to
switch between NMEA and binary protol without reset; message 0x84:
10 02 06 84 00 00 00 84 10 03  # binary mode (84 00 00 00)
10 02 06 84 01 00 00 85 10 03  # NMEA mode (84 01 00 00)

It seems to me that even message 0x89 described in the documentation is in
real message 0x8f; 0x89 message is used to change baud rate without reset.

There is one more message, 0x85, it is some kind of initialisation, I don't
understand 0x85 message details.

With message 0x8e is possible to define how often each NMEA message is sent
(0-255 seconds); It is possible with message 0x8e to activate PEMT messages
those have information about time, position, velocity and HDOP.

$PEMT,101,1,03,21.0,230705190757,5004.6612,N,01425.1359,E,00269,045,0000*24
101 - message type
1 - (1 or 2) I guess that this is lag that position is valid
03 - number of used satelittes
21.0 - HDOP
230705190757 - date and time, UTC
5004.6612,N - Latitude (degree)
01425.1359,E - Longitude (degree)
00269 - Altitude (m)
045 - heading (degree)
0000 - ?? it could be speed

After reset, EverMore GPS sends this message, it has information about GPS
configuration:
$PEMT,100,05.42g,100303,180,05,1,20,15,08,0,0,2,1*5A
100 - message type
05.42g - firmware version
100303 - date of firmware release
180 -  datumn ID; 001 is WGT-84
05 - Elevation mask
1 - DOP mode (1 is auto DOP mask)
20 - GDOP mode
15 - PDOP
08 - HDOP
0 - Normal mode, without 1PPS
0 - ??
2 - ??
1 - ??

Configuration message ($PEMT,100,) could be requested with message 0x38:
0x38, 0x85 (10 02 04 38 85 bd 10 03)

I don't have documentation for those messages, I can only guess and after that
play "try and see" game. In the case that someone has this information, it
could save me some time.
