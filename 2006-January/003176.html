Hmm, now that I recall a bit more, it was not on this list, and the app
was a little different.  So ignore my siggestion of searching this list.

Bash allow you to redirect to/from the following psuedo device:
	/dev/tcp/[host]/[port]

If you have sendmail running on your local host try this:
	# cat < /dev/tcp/localhost/25

Or, if you have ftp running try this:
	# cat < /dev/tcp/localhost/21

But that only applies to redirects.  gpsd wants to open the GPS device
it self, so this would not work:
	gpsd /dev/tcp/[host]/[port]

You would need a way to open the /dev/tcp/[host]/[port] in a bash script
and figure out how to redirect it to gpsd.  Or we need to add a option to
gps for this.

What is at the remote end?  Why not just run gpsd on the remote and grab
the results remotely using port 2937?
