  As of Jan 1, I've been seeing a new problem with my Sirf receiver.
When I first fire up gpsd, everything is fine (running with -D2).
cgps shows happiness as the receiver starts finding sats.  As soon as
the fourth sat is found and the receiver shifts from 2D mode to 3D
mode, everything goes wonky.  gpsd starts sending updates only very
sporadically (every 15 to 30 seconds).  Restarting gpsd does not
help.  Power-cycling the gps receiver works, up until it goes into 3D
mode again.  Haven't had time to do any real debugging yet (planning
to, though), thought I'd check first and see if I was the only one
with this problem.  I switched the receiver to NMEA mode and rebuilt
gpsd with --disable-sirfII and at least I'm up and running again, but
I'd rather the sirf stuff worked.  For what it's worth, there were no
code changes on any of my machines (same behavior on 2.30 on OSX,
Fedora Core 4, and Kubuntu).  I did a 'svn update' and rebuilt with
the latest last night on OSX and Kubuntu, with no changes in
behavior.
