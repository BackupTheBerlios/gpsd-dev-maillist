> last I checked, a tcp port and a tcp socket were the same thing

A socket is an interface for interprocess or network communication. So a socket
have to be bound to a port to communicate over network. Sockets for local
interprocess communication is a POSIX standard called Unix Domain Sockets. You
can access it like a file

gpsd creates an interprocess socket in /var/run/gpsd.socket afaik.
