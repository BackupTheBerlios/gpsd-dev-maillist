> Bash allow you to redirect to/from the following psuedo device:
> 	/dev/tcp/[host]/[port]
>
> If you have sendmail running on your local host try this:
> 	# cat < /dev/tcp/localhost/25
>
> Or, if you have ftp running try this:
> 	# cat < /dev/tcp/localhost/21
>
> But that only applies to redirects.  gpsd wants to open the GPS device
> it self, so this would not work:
> 	gpsd /dev/tcp/[host]/[port]

Arghh!! To implement something like this in the shell instead of the
kernel seems more like the Windows approach (where shortcuts only work in
the explorer and not in applications).

Opening a tcp session as a special file would nicely fit into the
"everything is a file" model, but what use is it when this is restricted
to the shell :-(
