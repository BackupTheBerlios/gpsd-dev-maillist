> RINEX has been mentioned a lot, if we do that first can you help us

I have no experience on the inner workings of RINEX unfortunately. Is
it even suitable for streaming data?

> The GPS needs to be running at 38,400 bps to reliably send gpsd the
> 50bps data, so I really doubt we can compress that down to 1,000 bps.
> You might be able to write a server side program that only uploads the
> data you need but making it fit would be hard.

Hmm? The UBX RXM-SFRB message is 42 bytes. The commands

$ wget http://iki.fi/lindi/gps/experiment/13/rover.ubx

$ ubx/ubx-parse1.py < rover.ubx | head -n 2000 | cut --bytes=1-16 | uniq -c
