> Why did you do this rather than writing a tert for BSD minor device type?

1) because using "unknown" generates unwanted behavior (interferes
with writes to the device when I do want them)
2) it's a user-friendly assumption, given that most gps chips look a
lot like serial anyway. if we can hint that certain behaviors exist
(ie. high speed usb, or fragile bluetooth) then fine, but serial is a
reasonable default
3) because that's a huge maintenance nightmare. i can't guarantee that
all versions or architectures of openbsd will have the same device
major or minor, and don't want to go down that rat-hole for every
other posix-y platform we might run on.
