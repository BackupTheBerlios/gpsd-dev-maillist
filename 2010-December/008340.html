I'm looking at driver_ubx.c and there doesn't seem to be any code to 
verify the CRC on received messages. I see  ubx_write calculates the CRC 
trailer before sending messages, but I don't see any code to check the 
CRC on received messages.

For the GPS device I'm using, I see that the messages are sometimes 
garbled. Sometimes the device starts sending a UBX message and then 
(before finishing) starts sending another UBX message. This doesn't 
happen terribly often, but when it does happen the result is usually 
that the UBX header of the first message has an invalid length (often a 
huge number) and of course the CRC doesn't match.

It would seem important to check the CRC and validate the length. Are 
the checks there and I'm just not seeing them?
