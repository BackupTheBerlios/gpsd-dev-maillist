Yo Timo!

On Mon, 27 Dec 2010, Timo Juhani Lindfors wrote:

> 1) If I want to use this with gpsd do I need to teach rtklib to parse
> this JSON format or write some converter?

You are just the sort of person we are looking for.  We have wanted to
export ephemeris data from gpsd for a while, but are not quite sure what
the proper formats are.

The direction we are headed in is outputing the ephemeris data as JSON
only.  The data is sent only when the connnected client asks for it,
since as you say it is a lot of data.  I envision a number of clients
(or one all purpose client) that can read the JSON and then output
popular formats like RINEX.

RINEX has been mentioned a lot, if we do that first can you help us
validate our RINEX encoder?

> 2) Also, I'm using GPRS which seems to be able to upload about 1000
> bytes per second reliably. Is the JSON going to fit to that?

The GPS needs to be running at 38,400 bps to reliably send gpsd the
50bps data, so I really doubt we can compress that down to 1,000 bps.
You might be able to write a server side program that only uploads the
data you need but making it fit would be hard.
