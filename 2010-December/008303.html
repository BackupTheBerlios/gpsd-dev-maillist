libgen.h is in the Single Unix Specification and POSIX.

Under the new policy (since the big autoconf/header cleanup last week)
we will assume SuS/POSIX headers are available until we bump into a
platform that forces us to assume otherwise. That platform won't be
Linux, *BSD or Mac OS X, which are all pretty good about SuS
compliance; Windows might be an issue if and when the Windows port is
ever completed.

This assumption simplifies autoconfiguration a lot.  git head currently
has *zero* conditional includes of system files.  I'd prefer to keep it
that way as the resulting code is quite a bit simpler and lighter.
