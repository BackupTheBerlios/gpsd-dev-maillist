RXM-SFRB messages are 42 bytes. RXM-RAW messages are 8 + NSV*8 bytes
where NSV is the number of satellites.

>  It also looks JSON-y.  Got a link to some doc on it?

UBX is a binary protocol. My ubx-parse.py just prints it out in human
readable format. Feel free to read the ubx.py source code for a quick
list of messages.

The binary protocol is documented in

ANTARIS_Protocol_Specification(GPS.G3-X-03002).chm

but I don't have a license to distribute it. You might be able to find
it with google. I have composed a very incomplete listing at

http://lindi.iki.fi/lindi/gps/ubx.html

based on the ubx.h that was freely licensed (both RXM-RAW and RXM-SFRB
are wrong in that so don't trust it...).
