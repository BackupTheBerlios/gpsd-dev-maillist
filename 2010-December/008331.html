On 12/18/2010 10:09 AM, Greg Troxel wrote:
> libgpsd(3) does declare that callers must define gpsd_report

FWIW, this is an unconventional way to solve this kind of problem. The 
usual pattern is to have the library provide a default implementation of 
the function, and to allow the application to install an alternate 
implementation at runtime. For example, gpsd_report would be a global 
function pointer that initially points at gpsd_report_default_impl which 
is a function provided by the library, and the application would 
typically assign to gpsd_report in main before calling any libgpsd 
functions (or perhaps the libgpsd API would have a function 
gpsd_set_report_hook to select the report function).
