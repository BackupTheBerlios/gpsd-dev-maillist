Well you can commit locally without any problems at any time.
If you push - and the git was for some reason rolled back to an old state, your
push will fix that as you have all the information locally as soon as you've
cloned it. That's the nice thing on git, the more people clone your git, the
smaller is the chance that your commit history will be lost.

As my mirror http://git.debian.org/?p=users/bzed/mirror/gpsd.git still happily
mirrors, the commit history is not broken. So everything should be in a sane
state again. And thanks to the sha1 checksums we can be kinda sure the
repository still contains the right code.

The ssh key changes and the fact that the git repository seemed to be rolled
back to some older state (from a backup!?) definitely calls for a move away from
the sucking berlios.
