I just merged in a cleaned-up version of the changes for Linux BlueZ support that prahal has
been talling about on the #gpsd channel.  The effect is that if you hand gpsd a device
path that has a Bluetooth major number, the device fd will be a Bluetooth socket rather
than the bare serial file descriptor.

I don't know exactly what doing this is supposed to accomplish, as my knowledge of 
Bluetooth is near nonexistent and I have no Bluetooth hardware to test with. Bluetooth-
capable developers should try out thids code and tell me what's going on and how
it changes or simplifies operation from the ordinary user's point of view.

To build with the BlueZ support enabled, configure with --enable-bluetooth.  The code is
not configured in by default.
