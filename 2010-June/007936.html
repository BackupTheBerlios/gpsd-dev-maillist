  Hello,

  I have a few questions about gpxlogger. Does it have an active
maintainer/author now? I'd like to add some features to it, and
before starting to code, I'd like to know whether these features
are going to be accepted by the project.

I think the logger would benefit if it could:

1) daemonize
2) write to certain files, not to stdin
3) reopen files and/or start new tracks on SIGHUP
4) reopen files and/or start new tracks after the receiver didn't move for
   certain time for certain distance
5) suppress logging while receiver doesn't move
6) simplify logged track, e.g. if last three points fall into one line
   with given accuracy, then the middle point can be dropped.
