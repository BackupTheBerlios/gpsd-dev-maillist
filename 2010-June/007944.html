I pulled from berlios after Gary's push, and got a bit of a messy
situation.  The commits I had locally in master were also in
origin/master, but not identical in sha1.  I think that's because I
sometimes have local commits not yet pushed and do

  git pull
  git rebase origin/master

In the case of vanishing commits and forced-update of origin/master,
that would treat the public history (those 68 missing commits) as mine
and rebase it, removing the merge commits that happened because others
didn't rebase (hi esr).

So in the end I think things are sort of ok now.  And I agree with Jon
that this is a git win; with cvs or svn rolling back 3 weeks w/o backups
would have been a huge nightmare instead of a minor pain.
