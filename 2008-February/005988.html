>                 I'm trying to use DGPS with two devices connected, but I
> want to know how can I distinguish the data of each other? Could you

> give some clue?

Use the K command to list available devices. Use the F command to
set/query device selection for the current channel.

>                 I used gpspipe with the native data option (-w) and I
don´t
> know if the sentences that begin with "GPSD,O=" are from one of the GPS
and
> the sentences that begin with "GPSD,Y=" are from the other one, is it
true?

No. gpsd does not interleave device output and a single instance of
gpspipe can only read one device at a time. O messages are navigation
solutions, Y messages are constellation status. See the gpspipe
manpage for details on selecting source device at invokation time.
