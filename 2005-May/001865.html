Included are some changes in the base code that I made to support the TSIP
driver.
Please include those, as we are working in the same areas of the code and
it is difficult to remain in sync.

There is a possibility for buffer overflow in packet.c that worries me.
I have added a check and indeed trash input at the wrong baudrate
overflowed the output buffer after mis-recognizing a TSIP packet.
I'm not convinced it couldn't also happen for other packets.  The input
buffer is checked, but after validation of the packet it was blindly
copied into the output buffer that is smaller.
The Zodiac and Sirf handlers check the length, but what about NMEA?
