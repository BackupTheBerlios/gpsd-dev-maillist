>> same group set as the serial nodes in what I have seen so far.  Anyone?
>
> I'm open to a btter solution, whether that's through #ifdefs or looking
> at /dev/tty.  The main thing I want to avoid is wiring in a specific
> group name like uucp or dialout -- variant distro practices would break

I think the better solution is to use the vendor's tool, if one exists, 
but probably
a) none does
and
b) if it did, it happens at the wrong time.

SunOS 4 had /etc/fbtab, and Solaris has /etc/logindevperm. But that means 
the GPS tty must exist when the console user logs in. There are PAM 
modules which do this, also; I wrote one in like 1998.

Not every problem is ours to solve, though I will grant you probably none 
of the other solutions work either.
