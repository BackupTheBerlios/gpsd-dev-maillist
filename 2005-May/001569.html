I'll try to do that this weekend.  I just created a launchd .plist  
(XML) file to make gpsd start with OS X 10.4's new launchd.  I was  
using a System Startup script.

launchd looks to be pretty cool.  There's more info about it at:

http://developer.apple.com/documentation/Darwin/Reference/ManPages/ 
man8/launchd.8.html
http://developer.apple.com/documentation/Darwin/Reference/ManPages/ 
man5/launchd.plist.5.html

I'm looking for some suggestions/guidance as to how gpsd should be  
started with this.

This is the current file:

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple Computer//DTD PLIST 1.0//EN" "http:// 
www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
     <key>Label</key>
     <string>com.opensource.gpsd</string>
     <key>OnDemand</key>
     <false/>
     <key>ProgramArguments</key>
     <array>
         <string>/usr/sbin/gpsd</string>
         <string>-n</string>
         <string>-f /dev/cu.gps</string>
     </array>
     <key>ServiceDescription</key>
     <string>Provide shared GPS service</string>
     <key>Sockets</key>
     <dict>
         <key>Listeners</key>
         <dict>
             <key>Bonjour</key>
             <array>
                 <string>gpsd</string>
             </array>
             <key>SockServiceName</key>
             <string>gpsd</string>
         </dict>
     </dict>
     <key>StandardErrorPath</key>
     <string>/dev/null</string>
</dict>
</plist>

If anyone has time, please look over the two (2) man pages above, and  
see if I've missed anything.  It would be nice to have this install  
automagicly install via Mac, on OS X detected systems, with 'make  
install'.....
