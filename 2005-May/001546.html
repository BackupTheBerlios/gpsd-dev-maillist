Then it is probably best to forget about it.
It would be convenient to have a GPS access multiplexer on Windows, but
there are pitfalls.  Not only the development environment, but also the
existing apps in Windows which of course are not opensource so they should
be able to use it without modification.

And last but not least, it would probably only flourish as short while,
until Microsoft or some big GPS player comes up with a "standard GPS API"
and everybody switches to that.  Then we can just as well let them do all
the work :-)
