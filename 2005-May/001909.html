Rob Janssen <gpsd at xs4all.nl>:
> I see the R=2 processing is done in sirf.c
> I wonder if it would not be better to do it in packet.c so that it works
> for all protocols...  When I turn on R=2 for my Trimble receiver it does
> not output the raw packets.

I thought about this.  The problem is that we want to log the hex-dump in 
a driver-specific way (for example, the SIRF -D 5 message includes the hex
dump and breaks out the packet ID).  The choices are

(1) Compute the hexdump separately in each driver.
(2) Compute it once outside the driver and give up having driver-specific
    log messages,
(3) Comppute it once outside and once inside each driver
(4) Compute it once outside each driver and pass it in as an additional
    packet-handler argument.

I didn't like any of these choices, so I went with precedent.
