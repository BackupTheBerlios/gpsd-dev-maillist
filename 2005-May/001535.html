Rob Janssen <gpsd at xs4all.nl>:
> I think we should be careful with any command sent to the receiver that
> permanently alters its settings.
> Users that install gpsd just to give it a try may find that it works, but
> after that their original applications no longer work and/or their
> receiver behaves differently from before.

Alas, we're already there.  And I don't see any way to avoid being there.

The problem is that we need to do things like (for example):

1. Enabling GSA on some GPSes that don't send it by default. like the FV17.
2. Disabling GLL and VTG so that sending the PSF probe string to
   SiRF chips won't trigger a firmware buffer overflow that can 
   send the chip catatonic.

And neither stock nor vendor-extended NMEA gives us ways to capture 
the state of all the parameters we tweak so they can be restored when
gpsd exits.

So we have to do rude, irreversible things to GPSes.  What's making me
hesitate is the prospect of doing potentially rude things to the
control bits of tty devices.  That's an *avoidable* rudeness.
