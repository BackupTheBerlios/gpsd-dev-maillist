On Tue, 24 May 2005, Rob Janssen wrote:

> Yes, but this still allows for clever defines that transfom a fixed
> bitpattern (the 0x7fc00000 that is the value for NaN) into a float value,
> for example via a union.

Carefull here!  There is not one NaN, but a large number of them.
Really clever FPU programmers use different NaNs for different error
codes.  Sorta like throwing an exception, you insert a signalling NaN
into a failed FP computation.  The NaN will be propogated untouched to
the final value, Then you just need to test for the failure at the end,
instead of at each step.

Functions like isnan() have been around a long time and should be
portable and consistent with this usage.
