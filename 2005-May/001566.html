I think, it has not always been clear to everybody that there is an issue
of leap seconds and a 13 second difference with GPS time.
Times reported by the receiver were blindly assumed to be UTC.
Also, there were some quite serious issues w.r.t. timezones and daylight
saving time in earlier versions.

But those have been fixed, and attempts have been made to use time
information that is specified as being UTC, or to subtract the leap second
difference when that is not available (as in certain broken SiRF firmware
versions).  There is even code to derive the number of leap seconds from
the almanac data.

If it still does not work OK for you, we need to debug it to get it right.
