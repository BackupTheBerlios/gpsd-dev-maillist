This patch is a fix for the "PPS thread does not terminate" problem.
It is not a definitive fix but it should help.

What it does: PPS is only enabled (and the PPS monitor thread started)
when the -n option is passed to gpsd.

Rationale: anyone making serious use of PPS will always use the -n option.
 When -n is specified the device is kept open all the time, and the
reported problem is not occurring.

When -n is not specified, the device is opened and closed on client
demand.  In this case the PPS thread is not started and it will not cause
problems.
