On 06/01/2011 06:30 PM, Michael R. Davis wrote:
> GPSD Folks,
>  
>> I was just wondering why gpsd doesn't have a configuration file in /etc/gpsd.conf, like most other Unix/Linux software?
> 
> There is one capability that I (comming from the RPM packaging side) need that we currently don't support.
>  
> I need the capability to be able to deploy a binary that has every driver compliled into the executable but have a configuration file which can disable any particular protocols at run time.  I personnally do not want to package non-CRC protocols turned on by default.  I've had my issues with TSIP as many of you may remember.

Imho a mess of config files is not the way to go, instead - if having
such a feature is necessary - it should be implemented as command line
switches. Implementing some kind of config file reader would make a mess
for sure - instead it should be the admin's or packager's resposibility
to provide a proper way to configure such flags in the init skript or
whereever gpsd is being started.
