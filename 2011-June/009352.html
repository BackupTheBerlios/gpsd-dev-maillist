I don't know what sort of getting in the way Diego is referring to.  
Perhaps he'll clarify.

The package needs a scripting language (really, a language with
garbage collection and variable-extent types) because it contains a
number of programs for which C would be a poor choice, imposing 
a lot of low-level resource management complexity on code that
doesn't have performance requirements hard enough to justify it.

We use Python because I like Python. I'd like Lisp better, maybe, if
there were one with good enough cross-platform support.  I'm open to
rational argument about replacing Python, but the replacement would
have to have some pretty compelling advantage to pay for the 
conversion overhead.
