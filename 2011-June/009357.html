Right now, I don't use the standard method of launching gpsd. I run it from
a script when I need it. If I wanted to disable protocols, it would make the
script dependent on the gpsd version.

For this situation, either a gpsd.conf, or a Richard's ahrs system.conf
would be a better solution than making the gpsd launcher dependent on
particular gpsd versions( I guess you could imply that any protocol that
gpsd doesn't know is in 'disabled'). I would rather it be a gpsd.conf file
than a richards ahrs.conf, as it seems a setting that ahrs doesn't care
about and is entirely machine-specific.

That said gpsd currently provides all that I need for my project, so I'll
try not to argue for it. Basically if I needed to disable a protocol, I
think it might be cleaner in a conf file.
