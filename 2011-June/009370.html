> If you had never written any C code in your life I would understand

It doesn't, but that's a detail  You are right about me being very 
experienced - I've been writing C for 28 years - but that is precisely
*why* I kick and scream and resist the thought of writing more C when
a task could be done in a garbage-collected language. Little C programs
that are not maintainance nightmares have a way of accreting features 
until they become big ones that are.

Now, in this particular instance, I've already moved the Python code into
C.  But I want you to understand why I am generally quite obstreperous 
about this sort of move, and don't expect to be any less so in the future.
