Documentation page states:

gps_set_raw_hook() takes a function you specify and run it (synchronously) on the raw data pulled
by a gps_query() or gps_poll() call. The arguments passed to this hook will be a pointer to a
structure containing parsed data, and a buffer containining the raw gpsd response.

It should be stipulated that after parsing a K command the raw gpsd response is truncated after
the first device name, due to the use of the strtok_r() routine.

Apparently the right thing to do is use the parsed data from the structure
In case of the K command that is:

gpsdata->set | DEVICE_SET  is true
gpsdata->ndevices is set to the number of devices
gpsdata->devicelist is filled with ndevices strings
