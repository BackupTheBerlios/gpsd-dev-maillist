Somewhat fair call, although I would say that all the last patch did 
was allow existing side effects in the logic to show through.  I 
would certainly like folks to test it, which is why I'm suggesting it 
go in the SVN repository (not in a release, although I think what's 
currently in the release is even worse).  In my current patched 
version, CYCLE_START_SET still causes the line

	    session->gpsdata.set &=~ FIX_SET;

to execute, which should allow it to track what has been set in this 
cycle.  gpsdata.fix is initialised in the gpsd_init() function (line 
63 of libgpsd_core.c).

I'll certainly do some more testing here, and see if I observe any 
odd behaviour.
