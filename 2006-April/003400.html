Ville Nuorvala <vnuorval at tcs.hut.fi> writes:
> How is it with different 2.x versions then? I'd assume a RTCM 2.3
> listener is able to understand a RCTM 2.0 stream, but would a 2.0
> listener be able to get anything at all from a 2.3 stream?
> So which RTCM version(s) does gpsd currently support?

The decoder was written with a copy of v2.1 RTCM-SC104 in hand.  It
isn't a full decoder though, there are many more messages that are
possible but were never seen in a US Coast-Guard DGPS data stream.  I
only implemented the message types that I actually saw over the air.
(The missing messages mostly concerned information that only
survey-grade carrier-tracking gps's would care about anyway.)

Does it sound like the later RTCM standards might have changed the
format of the basic type 1 and 9 differential messages or simply added
other messages for newer types of corrections?
