Hi all,

While investigating Charles Curley's recent problem, I found a minor 
error/oversight in the command-line help text displayed with a 'xgps -h' 
command.

Apparently, the command-line switch to display the version was switched from 
'v' to 'V' (i.e. lowercase to uppercase), but the help text displayed with 
the '-h' option wasn't updated to reflect the change.

The attached patch should apply cleanly to 'gpsd' 2.32 source.

As before, can someone with write privileges please apply this?

Thanks!

Bill Marr

@@ -469,7 +469,7 @@
 	    smoothing = atoi(optarg);
 	    break;
 	case 'h': default:

+	    (void)fputs("usage:  xgps [-hV] [-speedunits {mph,kph,knots}] [-altunits {ft,meters}] [-l {d|m|s}] [-s smoothing] [server[:port:[device]]]\n", stderr);
 	    exit(1);
 	}
     }
