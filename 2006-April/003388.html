I think, gpsd shoud make life of other programers simpler and should hide
cycle changes. In the case that there is a reason and software wants to have
only valid data and do buffering itself, gpsd could work in two modes. One
mode that gpsd will report only valid data (we have it now) and other advanced
mode that gpsd will do buffering and report the best estimate data. New
command could be used to switch between these modes, in similar way that is
used to switch between binary and NMEA mode now. This compromise could make
everyone happy.
