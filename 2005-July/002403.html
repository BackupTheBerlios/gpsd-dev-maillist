Hello,

I have one more bad news; Evermore GPS working in binary protocol was
recognised as "Trimble TSIP".

From log files, it seems to me that GPSd doubles some input characters. I
attach two files, first is binary input from serial port. Second is log file
from GPSd. Not the same data but the same binary mode. But I cannot see some
sequences of characters in raw data, like 0x10 0x10 0x10 I see in gpsd log
file.

State machine is used to detect GPS now. It will be more complicated with
every GPS that will be added in the future. Maybe that other strategy could be
used. To collect several bytes of data to the buffer (like 500 bytes) and call
a scan procedure of every GPS driver. That scan procedure will try to find a
valid data in the buffer. It will report if valid packet was found. Some scale
of weigth could be used as a output (0 no, that is not my protocol, 5 that is
my protocol, 100%). This approach will be more difficult as just simple state
machine but more scalable and better accuracy results. Just an idea...
