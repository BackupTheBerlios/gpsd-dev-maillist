Eric S. Raymond <esr at thyrsus.com>:
> At the moment, the code assumes that if the satellite picture holds a
> PRN > 32, WAAS corrections are available.  
> 
> I'll go to a driver-specific check only if somebody tells me that
> non-WAAS-capable GPSes can see these IDs, or there are circumstances
> under which as WAAS-capable GPS can see these IDs but not be receiving
> corrections.

On further investigation, I was making this far too complicated.  An
NMEA receiver tells us whether it has DGPS via the NMEA status field
from RMC and GLL; equivalent bits exist, and are already checked, in
the SiRF and Garmin drivers.  So I'll just have the error-modeling
code look at that, presuming that satellite lock with PRN > 32 is
one of its tests.

The Zodiac and TSIP drivers never set the status field.  Should I 
assume they don't speak WAAS/Egnos?
