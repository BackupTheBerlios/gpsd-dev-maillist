On Mon, 25 Jul 2005, Chris Kuethe wrote:

> Attached is a fragment of some configure code I once wrote to go
> searching for db-4.x. This has the same sort of problem: I've seen
> /usr/{,local}/{include,lib}/{,db4} in the wild. What I chose to do was
> to test for the logical places and then allow the user to specify
> --with-db--{includes,libraries} if my guesses weren't good enough.

You missed a real obvious one.  The tarball you get directly from
Sleepycat installs itself here by default:
	/usr/local/BerkeleyDB.4.[x]

Where [x] is the minor rev.

I sorta like there naming scheme.  It allows you to install and test a
new version before commiting to it.

So your rule can become:

/usr/{,local,BerkeleyDB.4.?}/{include,lib}/{,db4}
