"Eric S. Raymond" <esr at snark.thyrsus.com> writes:
> * RTCM104. I'm now convinced this is worth supporting, but have
>   no desire to write the RTCM104 decoder part myself.

I can donate both a homegrown RTCM decoder and encoder.  The decoder
is a bit simpler than John Sager's* and it uses no globals so one can
decode multiple streams at once.  Ditto for the encoder.  At one point
I was going to use this to unwrap all available streams, average and
make a composite RTCM feed.

Right now it only has code for type-1 and type-9 msgs, but these are
the only ones consumer GPS's care about anyway.

        http://www.wsrcc.com/wolfgang/private/rtcmencode.tar.gz

It comes with some tests, and it looks like they both work, but it was
never deployed, so for all I know it might have a few bugs.  I just
touched it up to compile under FC4/amd64.

If deemed useful I'll slap a GPL on it.
