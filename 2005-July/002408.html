The gpsd driver structure now supports a new capability.  For GPSes
such as the iTalk that allow it, there can be a method that changes
the GPS cycle time (thus, its sampling rate).  

I have attempted to implement this for the iTrax using $PSFT,FIXRATE.
There are other GPSes for which it should also be possible.  See:

1. The updated documentation for the C command.
2. The Hacking Guide section on "Throughput computation for baud rate changes"
