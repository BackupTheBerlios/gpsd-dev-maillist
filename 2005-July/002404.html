I've been off most of the weekend at a live-action roleplaying game -- playing
a Prussian mad scientist in a steampunk version of 1874 London with both
Babbage engines and magic -- but I found time to code up a first cut
at support for iTrax NMEA and iTalk binary.

The code could be completely wrong; I have no test hardware and was
working solely from the manuals.  However, if I haven't screwed up,
here's where we are:

1. The extended iTrax dialect of NMEA is fully supported.

2. There's a packet getter and a stub driver, italk.c, for the iTalk
   binary protocol.

Oleg Gusev has indicated he's ready to write the decoder.  Go to it, Oleg!
