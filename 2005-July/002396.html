There is a way how to probe for EverMore chipset. I can send binary packet:
10 02 04 81 13 94 10 03

and evermore will reply with something like this:
*10 *02 *0D *20 E1 00 00 *00 0A 00 1E 00 32 00 5B *10 *03

Only bytes mareked with star are fixed, others could vary. This is a reqest to
read "LogConfig" (message ID 0x81). Reply is the status of internal data
buffer used to store logging information.

My question is, if this request will not triger some action on others GPSs. Is
it save to send such request? 

This command could be send even when EverMore GPS works in NMEA mode.
