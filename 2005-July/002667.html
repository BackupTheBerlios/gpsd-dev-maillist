It works again, thanks.  I did look around a bit but I could not spot an
obvious error.

However, what Andreas wrote about packet_names.h is true.
When running a make on a cleanly downloaded version, the make process
stops on a missing packet_names.h.
When "make packet_names.h" is entered it is created, and a plain "make"
then continues, but it is not picked up automatically.

I think this is because the makefile is not very clean.  Lots of things
are listed as depending on all sources, instead of well defined
dependencies for each individual file.  This also shows as a lot of
unnecessary compilations when only one .c file is modified.  You would
only expect that with .h files.

No idea if that automake tool can produce better makefiles.  I know for
sure that "in the old days" I wrote makefiles that were a lot simpler and
worked a lot better than this 1400-line piece of %^$#&%@^%#&...
