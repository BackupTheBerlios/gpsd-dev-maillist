I had a bug list item that read:

*** First packet from each device may be getting junked

Rob Janssen reports
> As the code is now (and has been as long as I have been hacking drivers)
> the first packet received is junked somewhere before the parser is called.

I didn't think this was true in general, and it isn't.  To
demonstrate, run "gpsfake -p test/garmin17n.log: and note that the
first line, 

$GPRMC,093802,A,5213.1439,N,02100.6511,E,000.0,226.0,160305,004.2,E,D*15

produces an O notification as expected:

GPSD,O=GGA 1110965882.00 0.005 52.219065 21.010852  137.20        ?
? 226.0000    0.000  0.000

Now, it can *look* like the first line or packet is being skipped if it
reports no fix.  

Rob, if you're still seing this it has to be a driver-specific
problem.  Can you produce a log with the property that (a) the
first line reports a fix, and (b) it is ignored?
