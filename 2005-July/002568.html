Rob Janssen <gpsd at xs4all.nl>:
> Are you still looking for the UTC offset?  That is located in the 50bps
> data (almanac/ephemeris).  Maybe your receiver can send it on request.
> The almanac is a very complex piece of data, because details of the
> modulation scheme are visible in the data returned from the receiver, and
> you have to do many checks and handling yourself.  For example, the data
> is sometimes inverted (I presume for spectral reasons, after modulation)
> and this has to be undone.

The code to extract leap-year offset has been broken out of sirf.c and now
lives in libgpsd_core.c, specifically so EverMore and other drivers can use it.
 
> The SiRF driver has code to extract the UTC offset from it but I never got
> it working on my receiver.  Others have got data but found strange bugs in
> the data from the receiver.

It appears to work for me.  However, the code is sufficiently nasty 
that I'm not at all sure I trust it.
