On Thu, 24 Mar 2005, Rob Janssen wrote:

> What is the format of the version string returned by typical SiRF units?

ASCII free-text. I've a receiver that says 231.000.000ES, and some that say
2.3.2-GSW2-2.05.024-C1Prod1.3

> My old uBlox gps-ms1 returns: 1.3.2SCKR344

That's a SiRFstar I product.
http://www.u-blox.com/products/ms1.html

> As I see the code does an atof() on it, this probably differs from normal?
> Right now, gpsd considers it "less than 231" and tries to throw it in NMEA
> mode, which fails, and then only complains about unexpected NMEA messages.

I think sending message 135 (Set protocol) might be useful here. uBlox says
in their 2.x manuals that this message is obsolete. SiRF marketing insisted
on a bunch of backward compatibility features in the pinout and protocol, so
there's a good chance that this message will do what we hope...

see the pdf below for documentation of the uBlox extensions.
http://www.linkwave.co.uk/assets/GPS.G2-X-01003-E1.pdf

> I think this receiver will have to be a special case (with its own flag)
> as it also uses a different message type (0x62) and layout for the
> extended fix information.  (see sirfmon.c)

Again, see the pdf at linkwave for documentation of the uBlox extensions.

> However, I'm not sure if this is just a very old SiRF release (132) or if
> there is some different numbering convention involved.

Very old firmware indeed, running on an old chipset.
