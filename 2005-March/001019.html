On Thu, 24 Mar 2005, Eric S. Raymond wrote:

> Newer SiRF chips have UTC time in packet 41 -- unfortunately this isn't
> reliable at firmware level 231 which is what everybody has.

Nevermind "isn't reliable" - that data is nonexistent. :(

> Ah.  But I think I can *poll* for message 8.  Or do I have to wait for
> a satellite to ship one?

Message 8 is generated as the data is received. It is not buffered on the
chip. So when you enable message 8, you'll get on subframe every 6 seconds.
Of the data received, the almanac and ephemeris are buffered and stored,
so you can query them at will. Alas, the time parameters are not stored,
which is really lame, as the utc-gps correction changes 1 second every few
years. Maybe.

>> The receiver stores almanac data in backed-up memory so it is usually less
>> of a problem, but I don't think there is a command to read it from there.
>
> What a botch!

Yes, you can poll for the almanac and the ephemeris, both for a specific
PRN and for the whole constellation.
