Rob Janssen <gpsd at xs4all.nl>:
> What is the format of the version string returned by typical SiRF units?

I'm not sure.  All I know is that all the ones I have return

	231.000.000ES-01-39

I've heard references to 232 firmware, so I assume that in newer units the
first three digits become 232.

> My old uBlox gps-ms1 returns: 1.3.2SCKR344
> 
> As I see the code does an atof() on it, this probably differs from normal?
> Right now, gpsd considers it "less than 231" and tries to throw it in NMEA
> mode, which fails, and then only complains about unexpected NMEA messages.
> 
> I think this receiver will have to be a special case (with its own flag)
> as it also uses a different message type (0x62) and layout for the
> extended fix information.  (see sirfmon.c)
> 
> However, I'm not sure if this is just a very old SiRF release (132) or if
> there is some different numbering convention involved.

What do you mean by "its own flag"?  The driver type option is gone and
it's not coming back.  Instead, we can put a check for this format of
version number in the handler for 0x06 responses.

If you want to write an 0x62 case into the packet parser, feel free.
Send me a patch, I'll incorporate it.
