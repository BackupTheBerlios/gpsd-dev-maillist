On Mon, 14 Mar 2005, Eric S. Raymond at BerliOS wrote:

> Log:
> NTP interfacing is a todo item.

I have tried syncing NTP to GPS in the past.  If I have any sort of
internet conenction at all I can get 20mS, or even 1mS accuracy with
ntp.  When I sync NTP to NMEA I am lucky to get 500mS and it jumps
around badly.

After some research I have found that unless you have a PPS signal
from your GPS that you can not expect sub-second accuracy and you are
fooling yourself to use this as an input to the high precision ntp.

If all we want to do it set the system clock to GPS time then an easier
way can be found.  Only when no internet conenctivity is an option
would this be of any use, and then would only add a teeny bit of extra
acuracy.
