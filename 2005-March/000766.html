I shipped 2.15 a couple hours ago so real users would get to try the
new spiffy autobauding code.   So, of course, Gary found a bug and
committed a fix within an hour of the release.

The wildly fluctuating vertical speeds from SiRFs and Garmins *were* a
coding problem.  My bad.  He fixed it.  Congratulations.

This means we now have fully working SiRF binary support, with autoprobing.
As soon as Gary gets the Garmin probe function working, I'm going to
nuke the -T and -s options and ship 2.16.

Chris, the strategy I ended up using was to ignore everything in SiRF
packet 41 *except* the geodetic altitude.   Everything else comes from
packet 2.  On firmware versions less than 231, the SirF binary driver
says "no go!" and drops back to SiRF NMEA.
