Indeed, if you have a valid fix the time is normally OK.

But when you coldstart a receiver, it usually finds one or two sats
quickly, starts outputting GPS time, receives almanac data and hunts for
more sats, and when it has received the almanac it can jump to UTC.

This is especially true in NMEA mode.  In binary mode it is usually known
that you get GPS or there is some indication if UTC is valid.

I assume the people here use modern packaged receivers.  I have 3 bare OEM
modules (small printed circuit boards) that require external battery
backup power or optional goldcaps to keep the data valid during powerdown.
 My time reference receiver does not have backup power, so powercycling
the thing almost immediately results in a coldstart.
