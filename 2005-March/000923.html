On Wed, Mar 16, 2005 at 08:58:09AM +1100, James Cameron wrote:
> Ideally it should cause an interrupt and be delivered by the kernel to
> gpsd as an event.  If it's that important to have synchronised time,
> then latencies induced by frequent polling become significant.

I've done some more research (without yet reading any followups on this
thread), and found the TIOCMIWAIT ioctl(), which allows a program to be
woken (scheduled) when any modem status register line changes state.

I've not tested what kind of latency is experienced, nor whether the
typical PC AT serial port has interrupt on change capability that would
help the driver achieve low latency.

Reference:
"Linux Device Drivers, Third Edition, by Jonathan Corbet, Alessandro
Rubini, and Greg Kroah-Hartman. Copyright 2005 O'Reilly Media, Inc.,
0-596-00590-3." chapter 18, TTY Drivers, page 565, ioctls.
