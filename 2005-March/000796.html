I can see this WAAS satellite too, although it is useless in Europe.
There is a sirf message 170 that controls the SBAS behaviour,
but it is undocumented. sirf FAQ says

: 110. If a receiver has been up and navigating for a long period of time 
: (i.e., more than 12.5 minutes), and it is commanded to process SBAS
: corrections, which SBAS satellites will it search for, and if it searches
: for a subset of satellites that it looked for from a cold start, how does it
: know that it should search for that subset?
: The engine searches for the best SBAS satellite based on its longitude in
: the
: subset and if it fails to lock it goes to the next best satellite in the
: subset list. The subset is determined by the SBAS almanac.  Note that this
: is not always the best solution.  For example, a receiver located in western
: Europe may find the US WAAS satellite before it finds the European EGNOS
: satellite, and as a result may have poor DOP values due to receiving
: corrections only on satellites to its west.  To avoid this, users may direct
: the system to search for specific satellites.  Contact your technical
: support representative to find the way to do this based on your software
: version.
