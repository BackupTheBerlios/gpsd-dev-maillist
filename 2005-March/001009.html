I touched on this issue in one of the messages I sent you last week, maybe
you archived them.
There I warned against taking a timestamp from a GPS receiver and blindly
assuming it is UTC.  Look in the packet description if it actually should
be UTC, or may be "GPS time".

In my uBlox, the time in message 0x02 is GPS time, and the extended
message 0x62 has UTC time.

The offset (the figure 13) should appear in the Almanac data.  You can
enable Almanac messages (message 0x08) and get it from there, but it can
take a couple of minutes before it comes by, so this is not very useful. 
The receiver stores almanac data in backed-up memory so it is usually less
of a problem, but I don't think there is a command to read it from there.
