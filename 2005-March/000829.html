[Eric S. Raymond]
> There is an alphabet soup of supplementary systems intended to reduce
> the GPS error budget.  DGPS.  WAAS.  EGNOS.  SBAS.  I don't clearly
> understand the exact distinctions among them.

I do not know the latter, but I do know the first three.  DGPS is
differencial GPS, where one typically have "out-of-band" communication
with a service providing GPS signal correction information to make the
GPS positioning more accurate.  WAAS and EGNOS is almost identical
systems, WAAS being the system covering USA, and EGNOS being the
system covering Europe.  They provide "DGPS" with in-band
communication, having geo-stationary satellites broadcasting GPS
signal correction information on the same frequency/format as the GPS
satellites.  The latter make sure the GPS receiver can receive and
process this information if it has support for WAAS and EGNOS.  The
WAAS/EGNOS system will make GPSes more accurate, and also add
integrity checks making it possible to detect when the GPS location is
totally wrong.

> I don't think gpsd users should have to care about any of this. gpsd
> should autoconfigure itself to do the right thing.  Now, can anyone
> tell me what the right thing is?

For WAAS and EGNOS, I believe the GPS receiver will handle the
information automatically and just provide higher accuracy and
integrity.  To use DGPS one need to handle the DGPS protocols to be
able to fetch information from the external source and send it to the
GPS or process it in software.

> The implementation goal should be:
>
> (a) For gpsd to choose sensible defaults
>
> (b) For non-default settings to be optionally tweakable through some
>     kind of configurator that talks to the GPS.

For WAAS and EGNOS, I suspect the only sensible default is to enable
it, and I can see no reason to have it disabled.
