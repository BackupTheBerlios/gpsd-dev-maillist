On Fri, 11 Mar 2005, Eric S. Raymond wrote:
> Remco Treffkorn <remco at rvt.com>:
>> For me DGPS is still an important feature. The west coast is covered
>> by DGPS stations transmitting between typically 150 to 300kHz. A
>> DGPS receiver costs about 200 bucks, if you look. Some GPS have a
>> serial input port for the data, some accept it on the NMEA serial
>> port. For those support in gpsd is still critical.
>
> Sorry, I don't understand how.  The DGPS support in gpsd has nothing
> to do with feeding corrections from a radio or other local equipment
> into a serial port.  It's all about fetching RTCM packets from a
> remote server on the Internet and feeding *those* into the port.
>
> Or am I missing something basic here?

Ooooooo! something I just realized - SiRF message 17 is RTCM differential
messages. So if your receiver can't speak WAAS, but it can speak RTCM, you
might be able to use these RTCM / DGPS / WAAS messages. So perhaps we could
use SiRF GPS receivers to make DGPS messages more widely available.
