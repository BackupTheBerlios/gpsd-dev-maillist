On Tue, 15 Mar 2005, Eric S. Raymond wrote:

> It's even possible that with some thread programming we could set up
> a lightweight process to record time-of-day on each PPS pulse.

The PPS is part of the NTP code.  No need to reinvent that wheel.

They use DCD for the the PPS pin.

The NTP folks have gone to quite some effort to squeeze every possible
smidgen of accuracy out of the PPS signal.  They already have kernel
patches that grab the local timestamp just as the PPS is detected.
That way any later latency can be ignored.  There is even a standard
(RFC-2783) for how the PPSAPI should look like.

The NTP folks claim to be able to get accuracies of tens of microseconds.

The only purpose I would see to a gpsd/ntpd conenction would be to
get one second accuracy when you are off net.
