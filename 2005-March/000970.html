On Wednesday 23 March 2005 15:06, Eric S. Raymond wrote:
> Presently gpsd is issued under the GPL.  I propose to replace this
> with LGPL.
>
> GPL made sense before I factored the code into a library and a daemon
> core.  Now, both daemon core and library are GPLed.  The deployment
> problems with GPLed libraries are well known; many projects, especially
> those under BSD and Apache licenses, won't touch them.
>
> My own policy is to use GPL as a default for standalone code, and BSD
> for linkable libraries.  If the list consensus were to move to BSD I
> would welcome that, but the LGPL is suitable.
>
> Since I wrote libgps.c entire, I would be within my rights to change
> the license on that piece unilaterally.  However, the rest of library
> also contains work by other developers, including Remco Trefforn,
> Derrick Brashears, and Russ Nelson.  Thus, the entirety should not be
> relicensed if any of you have substantive objections.
>
> If nobody yells about it before I ship 2.19, I'm going to replace the
> COPYING file (presently a copy of the GPL) with a copy of the LGPL.
>
> Any objections?  (It would be nice to have this on record in the list
> archives.)

Fine by me. I would be happy with BSD for the lib.
