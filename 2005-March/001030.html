Yo Chris!

I guess I did not make all the steps clear.  The leap seconds are in
the current ephemeris.  If I do not have the current ephemeris the
GPS can not calculate the curent location.  If I can calcualte the current
location then I have a current ephemeris.  If I have a current ephemeris
I have the leap seconds.  Ergo, if I have the current location I have
the leap seconds.

Sure I do nt NEED the leap seconds to compute the current location, but
given that certain things are bundled together in the GPS system I can
make that coenction.

> Let's say I suck back a big huge joint and decide to arbitrarily redefine
> today to be Christmas. That doesn't change anything about certain universal
> constants, like the charge of an electron or the speed of light. So long as
> the system uses consistent measurements (ie. gps time all over the place),
> then the UTC conversion doesn't really matter. You're measuring distance
> from 4 known points to position yourself in 3-space. As soon as your receiver
> has code lock, it has a pretty good idea of what gps time it is, and can
> begin to start calculating a fix.
>
> It's just a convenience that the system remains fairly close to UTC, and
> uses the same length seconds as UTC and can be fairly easily converted to
> UTC (which seems to make more sense to more people...)
