GPSD Folks,

> switching to ISO8601

I don't mind switching but I haven't figured out a good transistion plan unless we change the "time" token in JSON to something else.

I also like the fact that we are explictily showing the "Z".  Everytime I need to look in the docs to figure out if the time is UTC or GPS time.

In the Perl lib, I'd like to keep "time" as epoch and "datetime" as an object but add something new like "timestamp", "timez", "ftime", or "fixtime".  I could support all three with 100% backward compatibility if we rename "time".  

But, if we reuse "time" to hold the new ISO datetime then we might break clients as I would not want my "time" method to be different than the JSON "time" data element.

Of course, I could release an entirely new object in an entirely new namespace like Net::GPSD4 or whatever the new protocol number will be but that just sounds like more work and would "force" people to change their Perl code for v3.
