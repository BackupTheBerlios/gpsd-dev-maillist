On Sun, 2 Jan 2011, tz wrote:

> I noticed the floating point numbers are quoted so are a string and
> not a number and with 3 digit exponent and are between 0.1 and 1.0.
> Is that intentional?

Yes, and no.  I'm not sure how JSON handles numbers, if JSON floats
are not supposed to be in quotes then I'll skip them.  As for the
exponents, that is the way I the online almanacs I can find do it, so
doing it that way makes it a lot easier to validate the numbers.

> There are other chips too, in particular, I use the Venus 634lpx chip
> (10hz, SBAS, AGPS) and the docs are available for all the commands as
> is a 7-day AGPS binary blob download from an FTP site, however the
> internal format is NOT documented.

Well, that is sort of a show stopper.  I don't have the time to
reverse engineer when there are lots of well supported chips.

> Is all this which you are putting in going to work cleanly with the
> other chipssets?

I am just dumping the 50bps subframe data.  Any chip that outputs the
50bps data without munging it should be easy to support.  Anything
else is out of scope for this task.
