Should these LOG_... symbols become a formal part of the libgpsd API? 
Currently, the documentation (libgpsd.xml) says only this:

    <function>gpsd_report()</function>.
    The library will use this to issue ordinary status messages.  Use
    first argument of 0 for errors, 1 for ordinary status messages,
    and 2 or higher for debugging messages.</para>

which means users of libgpsd can't write a gpsd_report like what now 
exists in gpsctl.c and gpsd.c, not without relying on undocumented 
symbols. (I ran into this very problem a couple weeks ago when trying to 
translate gpsd_report's errorlevel values into my system's logging 
facility...)
