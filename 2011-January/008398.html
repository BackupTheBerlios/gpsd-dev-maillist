> models.  How does this look for a proposed output:
> 
> {class:"ALMANAC","ID":02,"Health":000,"Eccentricity":"0.9887218475E-002","Time_of_Applicability":"147456.0000","Orbital_Inclination":"0.9387741089","Rate_of_Right_Ascen":"-0.8170900401E-008","SQRT(A)":"5153.618164","Right_Ascen_at_Week":"0.1624360561E+001","Argument_of_Perigee":"-3.111318946","Mean_Anom":"-0.1650987267E+001","Af0":"0.3128051758E-003","Af1":"0.3637978807E-011","week":593}

> The key names are a bit verbose, but that tends to make them self
> documenting.  Comments?  Ideas?

Up to now I've generally made the JSON member names identical to the C structure
names.  I think that consistency is valuable for both documentation purposes 
and maintainability.

There may be some exceptions to this rule in older parts of the report 
structure, but if so I view those as mistakes rather than precedents.
