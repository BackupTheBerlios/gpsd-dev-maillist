> Michael Cook <mcook at bbn.com>:
>> On 1/5/2011 10:17 AM, esr at BerliOS wrote:
>>> +    while (read(context->dsock, buf, sizeof(buf) - 1) == -1) {
>>> +	if (errno == EINTR || errno == EAGAIN)
>>> +	    continue;
>>> +	gpsd_report(LOG_ERROR, "ntrip stream read error %d on fd %d\n", errno,
>>> +		    context->dsock);
>>>  	goto close;
>>
>> If read() ever did result in EAGAIN, this would turn into a
>> busy/poll loop, repeatedly calling read() as fast as possible until
>> data is ready. That could be bad, no?
> 
> You're right, it appears this change from Beat Bolli is incorrect.  I
> have backed it out. Perhaps he can explain what was intended here.

I kept getting errors with the caster I mentioned, so I added reporting
to see what the problem was. errno was EAGAIN, so I added it as an
exception as I also had to in ntrip_sourcetable_parse(). I'm not sure
what the intention was regarding EINTR, but since the socket is
non-blocking, I thought it made sense to repeat the read()/recv() in
this case. And with the swisstopo caster, GPSd managed to sync to the
Ntrip stream with this change.
