"Chris Kuethe" <chris.kuethe at gmail.com> writes:

> On 3/27/07, Michael R. Davis <mrdvt92 at yahoo.com> wrote:
>> > > I think localhost is a good default, and will satisfy 99% of the
>> > > use cases, and probably 99.999% of the use cases
>>
>> "localhost" on fedora 6 (fresh install) is now ::1 not 127.0.0.1.  Be
>> careful what you ask for.
>
> that's the name "localhost"...  probably resolved from /etc/hosts. My
> hosts file also starts with ::1. Note that the proposed patch doesn't
> do a name lookup for localhost, it uses the IPv4 constant
> INADDR_LOOPBACK.

Really gpsd should support IPv6 also.   But the resolution to ::1
should not cause a problem:

  > telnet localhost smtp
  Trying ::1...
  telnet: connect to address ::1: Connection refused
  Trying 127.0.0.1...
  Connected to localhost.

If Fedora 6 doesn't also return 127.0.0.1 for localhost, it's broken.

>> The default most likely should be unix domain sockets.
>
> I'm not convinced of that. You can already have a unix domain socket
> with -F. This is about restricting access to the TCP socket.

I'd also lean towards AF_UNIX as default in a new design, but given
that many (most) existing clients use TCP to localhost, it seems
unfriendly to remove that.
