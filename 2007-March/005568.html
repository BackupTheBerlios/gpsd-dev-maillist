"Chris Kuethe" <chris.kuethe at gmail.com> writes:

> gpsd -> /usr
> CFLAGS="-I/usr/include -pthread"
> LDFLAGS="-L/usr/lib -lgps"
>
> gpsd->/usr/local
> CFLAGS="-I/usr/local/include -pthread"
> LDFLAGS="-L/usr/local/lib -lgps"

This isn't right on systems that need rpath because packages are
installed in a place not in the default search path.  

I think the right answer here is probably to use pkgconfig, which
provides the "how do I compile with this library" interface.  NetBSD
pkgsrc usually patches .pc files to add -R if configure doesn't put
that there and then everything else works ok.

As for a .h file, it would seem to declare the procedures that a user
of the library might call, and the library should provide an
unchanging interface, subject to normal shlib versioning rules.  But
somehow this seems messier than that, and I don't follow why.
