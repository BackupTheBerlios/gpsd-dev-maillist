> Anyone presently working on this or have an idea where I can start?

How does it attach? does it pretend to be a usb tty or a generic
device? What vendor and product id numbers does it use? Half the
problem will be getting the device to attach meaningfully under
!windows.

I think that you get to do some bad-ass reverse engineering. Looking
at the UBX protocol reference, data logging does not seem to be built
in to the stock firmware or protocol, but there is a mostly-undefined
"USER" class of messages to support user apps in firmware.

If you can attach a serial monitor or usb monitor to the device while
downloading the logs, you can see what's being sent and see how its
packet format corresponds with other UBX messages. Mmmmmm sweet sweet
sniffers...

This is why I dual boot my laptop to windows - so i can run things
like usbsnoop and billserialmonitor.
