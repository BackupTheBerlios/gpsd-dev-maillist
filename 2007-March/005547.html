> where X is killed too.

I've heard this can happen, but I've never been able to put my finger
on it. How are you getting data from gpsd:
    connect ; while(1){ read , process, sleep }
or
    while(1) { connect, query, process, disconnect, sleep}

If I had to guess (and i have no evidence to back this up), I'd say we
may have an fd leak that is exposed by many short connections, or we
may be mis-handling / not detecting a closed socket. I say that
because a number of the client apps used to spin and burn 100% cpu
when the server went away.
