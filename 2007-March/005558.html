> Easily worked around; the file wants to be declared in the RPM spec.
> But I don't like the fact that we're exporting this.  It should be
> pfor internal library use only.

I export that because there are bits in gpsd.h that depend on what was
detected at configure time. When users write their own apps, they link
against libgps.so/.a but need the proper header to do so. By including
gpsd_config.h, we ensure the user gets all the right bits.
