Hi,

Here are two patches, to make libgpsmm work with the new semantics of
libgps.  It appeared to be completely non-working; I tried to keep it so
that it would conditionally compile with pre-major-version-5 libgps, but
I'm not sure if that's at all useful (in fact I don't find libgpsmm to be
at all useful, but it's needed for another program I want to compile).

In the process, I noticed that gps_read()'s return-value is not consistent
with the man-page (although the man-page is a little unclear on this and
seems to contain some cruft from previous versions).  I tried to make the
return code a little more consistent with the API docs.

Cheers,
