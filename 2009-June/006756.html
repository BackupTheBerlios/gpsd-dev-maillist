Hi

I can't get Garmin's eTrex Vista HCx (USB device) working with gpsd. I
simply don't get any NMEA data out of this device.

I'm using openSUSE 11.1-i586, i.e. Kernel 2.6.27, gpsd 2.37 (build
with "--enable-dbus --enable-garmintxt").

The kernel module 'garmin_gps' is loaded. /proc/bus/usb is mounted. I
followed the INSTALL file (section 1) and apparently this command
already fails (started as root of course):

# stty -F /dev/ttyXXX ispeed 4800
stty: /dev/ttyUSB0: unable to perform all requested operations

Nevertheless I tried running gpsd (options: "-N -n -D2"). When
connecting via telnet to port 2947, I can see in gpsd output, that my
commands 'f=/dev/ttyUSB0' and 'wr' succeed ("gpsd: opening GPS data
source at '/dev/ttyUSB0'","gpsd: turned on watching\ngpsd: turned on
raw mode"), but still I don't get any NMEA data (section 4-6 in
INSTALL file). Of course any other gpsd clients don't work as
well. That is what I've tried first.

Any ideas (how to continue investigating this)? Thanks.
