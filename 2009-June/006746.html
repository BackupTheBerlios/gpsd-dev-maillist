(Apologies if you already know this, but it took me by surprise when I
learned it) Check your serial port's settings using the stty command,
such as:

stty -F /dev/ttyS0 -a

There are a number of settings you can read about on the man page,
such as icrnl, "translate carriage return to newline"; igncr, "ignore
carriage return"; and inlcr, "translate newline to carriage return".

Programs like gpsd know they're going to be using a serial port, so
they can turn those options off; programs like cat just treat the port
like a file and don't change the configuration at all, which might be
the cause of your problems.
