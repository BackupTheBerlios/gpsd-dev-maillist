what is this "tgpsd" you speak of?

html versions of the manpages (as well as developer documentation) are
linked off gpsd.berlios.de.

i run gpsd like this:
#on my time server
gpsd -n -b /dev/cua01

#on my laptop
gpsd -n /dev/cuaU0 /dev/cuaU1 /dev/cuaU2 /dev/cuaU3

people using linux and bluetooth have been known to do something like this:

gpsd -b /dev/rfcomm0
