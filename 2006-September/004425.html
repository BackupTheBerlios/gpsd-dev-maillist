I am workong on getting garmin binary streaming working with the serial
driver (not usb driver) in gpsd.  Getting it working with the USB
will be the last thing I do, so hang tight.

The SVN code can now force a garmin serial GPS into garmin serial binary
mode and then recognize the packets as garmin serial binary.  Next I'll
modify the packet parser to understand the garmin serial binary data.

BTW, as I was reading the Garmin doc again.:

"there is the possibility that the device will transmit a few extra bytes at the end of the data, [...] you may safely ignore the extra bytes."

It is easy to ignore the extra bytes if they are at the end of a USB
packet (i.e. Garmin USB binary) or framed in <DLE> (i.e. Garmin straming
binary).  OTOH, with no reliable way to detect packet boundaries (i.e. mashing
Garmin USB packets together) there is a real problem.

So that is another reason you can not concatenate Garmin USB binary packets!
