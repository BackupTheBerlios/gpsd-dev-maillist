Gentlemen,

I see two issues:

1) PGRME message generated by gpsd is in wrong format. Message
$PGRME,0.00,0.00,28.80*4D is invalid, correct message looks like 
$PGRME,15.0,M,45.0,M,25.0,M*22.

2) nmea parser of PGRME message in gpsd is very optimistic (simple and
stupid). It ignores units and it has no check if message format is correct.

With regards,
Petr
