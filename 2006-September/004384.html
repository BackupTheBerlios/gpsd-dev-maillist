On Fri, 22 Sep 2006, Petr Slansky wrote:

> O message uses format "%.2f", Y message "%f". I guess that Y message is

> be formated in the same way, "%.2f". Please, check gpsd.c file.

Some GPS, like the Pharos GPS-360, provide time to 3 decimal places past
the second.  So I would vote for $.3f.  Clients should be prepared to
handle any kind of float.
