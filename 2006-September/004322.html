I have a Nokia LD-1W, a SirfStarIIe/LP based bludetooth GPS unit. In
theory it should support DGPS, but in practice it doesn't seem to. At
least it seems to totally ignore the RTCM data I'm sending to it. Of
course I've set the DGPS source to the external RTCM source in stead of
SBAS, but that doesn't seem to make any difference.  might be wrong, but
it seems that the DGPS corrections have to come in through the second
serial port, which apparently isn't connected on my GPS receiver.

After I spent time implementing the Ntrip support for gpsd, I can tell
you it was quite a disappointment to notice it had all been in vain.

I've now bitten the bullet and started working on doing the position
calculations inside gpsd. I'm just going to read the raw pseudorange
measurements,clock corrections and IS-GPS-200 navigation messages from
the GPS receiver and do everything else in the daemon. This way I at
least can apply the correction data to the measurements and hopefully
get the accurate position estimates I need for my master's thesis project.

If there is interest for this feature, I can probably post some code in
a week or two.
