I keep a minimal chroot in a tar.gz of about 70 MB, unpackaged about 200
MB. The build dependencies get installed at build time. This way, I
always have a clean build environment and can check the completeness of
my build dependencies. You should have a package mirror close by, or a
good caching proxy, to install packages at build time. Or keep the
minimal chroot + everything you need to build gpsd, which takes less
than 100 MB in the tar.gz und about 300 MB unpacked. So Eric, if the
leftover 10% of your disk are bigger than 1G, you have more than
enough...

I'm using pbuilder[0] to create the chroot and build the packages;
unfortunately, it is Debian-specific, but maybe someone on the list
knows something similar for other distributions.

Another possibility is using LVM snapshots for your chroots: Have a
clean master copy, make a snapshot in a matter of seconds, build
packages in the snapshot and mess it up at will (good for trying new
tweaks to the system, too!), throw it away when you are done.

[0] <http://packages.debian.org/unstable/devel/pbuilder>
