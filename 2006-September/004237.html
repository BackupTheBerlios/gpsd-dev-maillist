On Tue, 2006-09-05 at 22:25 -0400, Eric S. Raymond wrote:
> And here's a service question: does anybody who's listening know how
> to build 32-bit RPMS on a 64-bit machine?

Here's what I do for Debian packages; the same should work for RPMS as
well:

I have a clean 32bit installation in some subdirectory,
e.g. /tmp/chroot, and change into it for package building:

# linux32 chroot /tmp/chroot

The linux32 command takes care that the kernel identifies itself as
32bit, and the chroot command gets me into a pure 32bit environment:

mother:~# uname -a
Linux mother 2.6.16-2-amd64-k8-smp #1 SMP Fri Aug 18 21:10:33 CEST 2006 x86_64 GNU/Linux
mother:~# gcc -v
Using built-in specs.
Target: x86_64-linux-gnu
[...]
mother:~# linux32 chroot /tmp/chroot
mother:/# uname -a
Linux mother 2.6.16-2-amd64-k8-smp #1 SMP Fri Aug 18 21:10:33 CEST 2006 i686 GNU/Linux
mother:/# gcc -v
Using built-in specs.
Target: i486-linux-gnu
[...]
