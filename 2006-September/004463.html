Gentlemen,

I have one small topic for discussion. Here are several O and Y messages from
gpsd:

GPSD,O=EID2 1159555346.000 0.005 50.047538 14.443747  278.04 42.40 56.80  
0.0000    0.000  0.000 164.6271 72.80        ? 3
GPSD,O=EID4 1159555346.000 0.005 50.047538 14.443747  278.04 42.40 56.80  
0.0000    0.000  0.000 164.6271 72.80        ? 3
GPSD,Y=EID6 1159555346.000 9:7 84 320 15 1:21 68 71 23 1:16 59 248 0 0:18 39
131 15 1:3 36 288 0 0:31 31 287 0 0:6 23 102 18 1:22 18 167 0 0:19 9 278 14
0:

I noticed that Y message separates fields with one space, O message is more
like fixed length field format. From my point of view we can seve some code
space and space occupied by log files when O message will use just one space
to separate fields. What is your point of view? Is there any rule that forces
O message to use fixed length of fields?

Current O message format:

GPSD,O=EID2 1159555346.000 0.005 50.047538 14.443747  278.04 42.40 56.80  
0.0000    0.000  0.000 164.6271 72.80        ? 3

O message after reformat:

GPSD,O=EID2 1159555346.000 0.005 50.047538 14.443747 278.04 42.40 56.80  
0.0000 0.000 0.000 164.6271 72.80 ? 3

What do you think about it? Maybe that the same rule can be applied to other
messages as well...
