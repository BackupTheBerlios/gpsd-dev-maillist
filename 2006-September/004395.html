On 9/24/06, Petr Slansky <slansky at usa.net> wrote:
> Hello,
>
> I noticed that Sirf GPS sometimes produces messages like "GPSD,$=MID52 ..." I
> can enter "$" command from telnet session to port 2947. But this "$" command
> and message are not described in man page for gpsd (gpsd.8). '$' is not listed
> in 'l' command response too. Why is '$' kept in secret?

Yeah, I got hit by that today too.

It's not so much a secret as it provides no useful information to the
end user. It's used within gpsprof for performance testing - ie. to
figure out how long it takes for a message to be transmitted,
processed, etc.
