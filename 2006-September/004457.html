On 9/24/06, Gary E. Miller <gem at rellim.com> wrote:
> On Sun, 24 Sep 2006, Chris Kuethe wrote:
> > I was going to say that perhaps '@' by itself is special, causing gpsd
> > to escalate to v4, but then I remembered the 'L' command. Clients
> > wishing to use protocol 4 features must send 'L' first, to see if the
> > daemon support v4. From there, the v4 parser can be probed for which
> > commands it supports.
>
> I like it.

RTFM on a bunch of chipsets that use proprietary NMEA, it seems the
trend is to allow one command per line. Large commands (eg. ephemeris
loads) are either broken up into multiple commands, one per SV, or
they are just sent as one big huge command. Either way, it's one per
line.

Given that, I think we can go one command per line. That also makes
error detection and reporting a bit simpler.
