I suggest to extend structure nmea_phrase in nmea_parse.c file with column
like "minimum number of fields of message". Parser of NMEA message (like
processGPGLL) will be called only in the case that message has enougth fields.
This will avoid calls on messages that are not complete/broken. In the case of
GPGLL message, parser should be called only when message has at least 7 fields
(count > 7). Minimum number of fields for PGRME is 6, etc

Several parsers already have similar check before they start to process
fields. Lets make it more system way...
