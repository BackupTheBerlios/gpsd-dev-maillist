I've appended the full build log (from before your %zd fix).  The
warning flags are the default ones.

The problem is that NetBSD defines isalpha etc. as

  #define isalpha(c)      ((int)((_ctype_ + 1)[(c)] & (_U|_L)))

isalpha is defined to take an int:

  http://www.opengroup.org/onlinepubs/9699919799/functions/isalpha.html

and it's required to have a value which is either EOF or representable
as an unsigned char.

It is still unclear to me if using isalpha with char is wrong, or if the
isalpha implementation above is wrong.
