There is a udev rule in the in the cvs that calls the
gpsd.hotplug.wrapper script to add/remove gps devices. The script is
necessary because I have not found a way to test $DEVLINKS in a udev
rule. It does not appear to be valid until you start the script.
What we are testing is if the symlink is a GPS symlink and doing
nothing if it is not. The bad part is that it will create a symlink
for some serial converters that are used for USB GPS devices, as
well as other things.
