> - PATH is not honored, this avoids ccache 
> - gpsd.h is not installed
> - man pages for python programs are not installed

I've merged the PATH change, but what do you mean by "avoids ccache"?

It's not clear we should be installing gpsd.h. That's a header full of
internals that GPSD clients shouldn't see; putting it where third-party
code might rely on it makes me nervous.  I might revert this.

You say "libgps.so and libgpsd.so symlinks are not created on installation".
This probably requires a code addition to InstallVersionedSharedLibrary()
in SConstruct.
