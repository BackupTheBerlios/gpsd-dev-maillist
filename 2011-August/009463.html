Yes, with 3.0 and 2.96 the -N option is needed to have the root
permissions to open the chrony socket.

With the latest git it works without -N too. Also, the path to the
socket has changed to /var/run/chrony.ttyS0.sock.

One thing that might be not obvious is that chrony needs to be started
before gpsd. There is a bug in chrony that it exits before the socket
is actually created, so a short sleep before gpsd is started might be
necessary. I'm trying to get that fixed.
