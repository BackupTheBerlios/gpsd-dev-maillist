> I tried to connect Chrony with GPSD with the SOCKET method to get ns instead of us accuracy
>        refclock SOCK /tmp/chrony.ttyS0.sock
> (this is according to the chrony manual, the gpsd manual states "/dev/pps0 refclock SOCK /tmp/chrony.ttyXX.sock" but that gives an error.
> However, this doesn't work, chrony does not receive any data over that socket...
> I upgraded gpsd to 3.0 (the 2.96 version I was running did not mention this option in the man pages, 3.0 does, but no change :(
>
> Any idea how to get Chrony talking to GPSD using the SOCKET method?

I think you need to get GPSD connecting with your GPS in PPS mode
it also needs to be started so it connects and keeps connected -N  I believe
you might have to run in the foreground for the moment with the last
release, or you could try the current development which fixed a
permissions bug for opening the chrony  socket.

when you start gpsd you can use verbose to see where the chrony socket
was put, you will have to use that exact path in chrony.

I know there are several others who have set this up, and would be
able to help even more on the list.
