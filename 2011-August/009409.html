One way to wedge ccache into your workflow is to place a
ccache-wrapper script named gcc earlier in your personal PATH than the
systemwide gcc binary.  This is not the most common approach, but it's
what I've used because I use tools which notice CC overrides and
include non-default CC values in build directory names encoding
compiler, CPU, and OS, and I didn't want to invalidate all my
in-progress build directories, nor did I want longer build directory
names from CC='ccache gcc'.  It was also ugly in configure output, and
difficult to make accessible to other users of the system.  I use a
'gcc' script which defaults to a shared ccache directory.  This is not
appropriate if all users are not trusted, that is, this configuration
allows anyone to subvert the cache.

/usr/local/bin/gcc:
#! /bin/sh

#
# 'gcc' invocation script to use ccache without the
# side effects of requring CC='ccache gcc', such as
# NTP's build script wanting to include it in the
# A.* build directory name.
#
# set CCACHE_DISABLE to cause ccache to invoke gcc
# without any caching.
#

export CCACHE_BASEDIR=/home
export CCACHE_COMPRESS=1
export CCACHE_DIR="${CCACHE_DIR-/var/cache/ccache}"
export CCACHE_UMASK="${CCACHE_UMASK-002}"
export CCACHE_NOLINK=1
unset CCACHE_HARDLINK
ccache /usr/bin/`basename $0` "$@"
