Petr Slansky <slansky at usa.net>:
> I guess you cannot get current configuration from many devices to store it
> before you apply your changes.

Yes.  There's no way to query a generic NMEA device about, for example,
how often it sends GGA.  But devices where you *can* change that do have
a corresponding query.  

In principle, a judicious combination of queries and storing that changes gpsd
itself makes should make restore-on-exit possible.  It would be something of
a pain in the butt to do it, though, which is why we haven't yet.

It would be better to do that than have a runtime no-configure switch.

> What about a switch for gpsctl that will switch GPS device to some
> default configuration. Some GPS (like EverMore) has command to reset
> device to default mode (NMEA, 4800 N 8 for Evermore, command
> 0x95). You can define sequence of commands to configure every GPS to
> some default state, probably NMEA if possible.

How would this be different than "gpsctl -n -s 4800"?
