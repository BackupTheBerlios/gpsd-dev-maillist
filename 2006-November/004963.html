I have implemented interleaved probing, and it seems to be a big win.  
gpsctrl appears to be working reliably on SiRFs now.  Awaiting confirmation
from Chris on this.

I've fixed 'E' reporting myself; it now emits '?' for DOP components
not available, separately for each of the fields.  I've also fixed
libgps and gpsd(8) to reflect this.  Petr Slansky, since you reported
E was buggy please check the new behavior,

On Chris's say-so I have deep-sixed itraxctl.c and its manual.  sirfctl.c
and its manual are now redundant, but I don't plan to remove them without
Chris's nod.

Mick Durkin's driver-writing how-to is in the repository.

Remaining issues:

1. Gary informs me it's the Garmin USB driver using mode 1 (serial emulation)
instead of mode 0 (packet mode). that's busted.  

2. Once Gary fixes his timing problem in the driver, garminctl needs 
to be merged into gpsctl.

3. The True North driver remains broken.  This is holding up Jeff Francis's
rewrite of cgps.  I'm going to follow up today with the biz-dev director 
at True North Technologies and try to get us an eval unit.

4. PPS support is still broken.  Gary is working on this.
