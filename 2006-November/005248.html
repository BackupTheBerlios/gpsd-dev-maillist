On 11/14/06, Petr Slansky <slansky at usa.net> wrote:
> Seriously, I think you have valueble point and I don't see reason why to
> introduce revert calls to gpsd once gpsctl ultility was designed.

Because it's rude to not clean up after yourself.

Random users should not be expected to know about the protocol their
receiver is running in. They expect that they plug it in, and gpsd
tells them where they are. Maybe maybe gpsd decided that it wanted to
use the binary protocol... now they reboot to windows and run some
mapping program and their gps doesn't work any more. That's rude.

If you are like me, and have your receiver set up *just so* you really
don't appreciate having the settings changed. A better thing to do
might be to add an option to gpsctl to set up the receiver the way
gpsd would, and then exit.
