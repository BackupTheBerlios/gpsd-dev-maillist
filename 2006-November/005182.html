Gentlemen,

I send a patch to add support for "EverMore NMEA" driver.

It enables to run EverMore chipset in NMEA mode and to use EverMore commands
to change baudrate. It configures GPS to send GPGSV message every 5 seconds.
It is possible to switch to EverMore binary mode and back (it didn't work well
in the past, gpsd pushed EverMore to binary mode).

I see just small problem with this patch now. It shows firware number in 
NMEA mode "EverMore NMEA 5.42" after return from binary mode:

# telnet localhost 2947
Trying 127.0.0.1...
Connected to localhost (127.0.0.1).
Escape character is '^]'.
i
GPSD,I=Generic NMEA
i
GPSD,I=EverMore NMEA
i
GPSD,I=EverMore NMEA
n1
GPSD,N=0
i
GPSD,I=EverMore binary 5.42
n0
GPSD,N=0
i
GPSD,I=EverMore NMEA 5.42
i
GPSD,I=EverMore NMEA 5.42
Connection closed by foreign host.
