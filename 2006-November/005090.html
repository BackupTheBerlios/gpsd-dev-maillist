>And how many OS X users know how to use Fink?  Very few in my experience.

I'm not sure I understand your objection here.  If they're not using fink,
they're installing from a source tarball.  If they're installing from a source 
tarball, they're techies.  If they're techies, the one-time step to install
xsltproc is not a big deal, whether via fink or otherwise.  So where's 
the problem?

In general, you're mixing arguments from convenience to users with arguments 
about what techies might have to do in odd edge cases in a way that doesn't
make any sense to me.

>Why is it you feel the need to maintain a zillion packages instead of just
>one good tar?  What possibly could be the win here?

Ensuring that I never get a patch against generated man markup, for starters.
This used to happen to me, and I hated it.  Yes, I'd rather maintain 
a dozen spec files than do this, because that has other benefits.

>What about the other 100+++ Linux distros?

Frankly, I don't care.  Firstly, everything other than Fedora, Debian,
and Ubuntu is statistical noise.  Secondly, everything other than Fedora, 
Debian, and Ubuntu is statistical noise used by people who are quite
competent to do a one-time install of xsltproc on their own. after
which they no longer have a problem.

>                                         Why this need to go to all
> this extra work when one simple automated fix solves (and had solved)

Because it didn't solve *my* problem, which is that I never want to see
a patch against generated markup ever again.
