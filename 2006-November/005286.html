> http://xml.coverpages.org/nvml.html 

Most people that I know are working with GML as their geo-XML base. There is a temporal and dynamic feature structure. My problem with that XML structure is that the XML is above the sensor level. In other words, the system must know what the sensor relates to in order to send the data across the wire. 

I briefly looked into SensorML but I never found (not that it not in there) a structure that supports geo-tracking where the system does not know in what vehicle (or object) it is installed.
