> now using gpsd, instead of being stand alone.

Your premises continue to be mistaken.  You should read the gpsctl
manual page more carefully.

First, the new gpsctl has a "low-level" mode that doesn't use gpsd.  
You can force this mode even when the daemon is running.

Second, even the high-level mode is capable of leaving gpsd in a "twiddled"
state.  B, N, and C commands will not be undone on exit.
