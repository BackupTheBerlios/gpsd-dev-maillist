> Three, like a command-line option, a config file is an easy out for
> lazy programmers.  As I pointed out to Chris Kuethe some time back, 
> no-reconfigure just addresses a symptom -- the truly right thing
> would be to have gpsd store its config changes and revert them on
> exit.  But the second I implement a runtime no-reconfigure, the

I guess you cannot get current configuration from many devices to store it
before you apply your changes.

What about a switch for gpsctl that will switch GPS device to some default
configuration. Some GPS (like EverMore) has command to reset device to default
mode (NMEA, 4800 N 8 for Evermore, command 0x95). You can define sequence of
commands to configure every GPS to some default state, probably NMEA if
possible.
