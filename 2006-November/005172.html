> > What about a switch for gpsctl that will switch GPS device to some
> > default configuration. Some GPS (like EverMore) has command to reset
> > device to default mode (NMEA, 4800 N 8 for Evermore, command
> > 0x95). You can define sequence of commands to configure every GPS to
> > some default state, probably NMEA if possible.
> 
> How would this be different than "gpsctl -n -s 4800"?

This command is expected to set NMEA mode and baud rate to 4800, and that is.
But you want to change other parameters to default too. Why? For example it
could be useful when you get a device with unknown configuration. EverMore can
configure pinning, power saving, datum, DOP mask, etc. From my point of view
every driver should have a call to configure GPS to some default state.
