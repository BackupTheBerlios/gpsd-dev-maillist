Yo Andreas!

On Sat, 4 Nov 2006, Andreas Stricker wrote:

> > xmlto uses openjade and the DSSSL stylesheets.
>
> Funny, mine uses xsltproc (yes I'm sure).

Digging a bit more, I see mine does too.  But it also depends on tetex
and tetex depends on some verion of jade.  So it depends on how it was
installed and what sort of documents you are converting too.  Looking at
Gentoo I see they removed the tetex dependency, so maybe other distros
have too.  That rips a lot of functionality out of xmlto.

Now that I know xmlto can be installed without openjade that will save
me some effort in the future.

> >> I know this become the hard part. There should be a catalog in
> >> /etc/xml/ named docbook that points to the right installations.
> >> xsltproc should somehow be able to get the path from there.
> >
> > There should be, but it rarely is setup up properly.  Plus there are
> > so many versions of DocBook that having the right one installed is rare.
>
> You're right. But I don't need to know where the catalog is (/etc/xml,
> /var/db/xmlcatalog or somewhere else). Important is that xsltproc
> knows it. The prerequisite is that docbook xsl is installed correctly,
> as it should be if the docbook packages are from the distribution.

I always end up mucking with the catalog to make it work.  YMMV.  Given
the coplaints we have had about it in the past my situation is not
uncommon.

Slackware certainly does not include xmlto or set things up that an
xmlto install is easy.
