Marr <marr99 at highstream.net>:
> I haven't looked at the code for 'cgps', but I've spent some time over the 
> last few days working on a Gtk2-based replacement for 'xgps'. This is a task 
> which has been on my mind since we discussed this back in Feb 2006 and one 
> which became more important as I'm hearing about and experiencing more 
> GNU/Linux distributions releasing without Motif (OpenMotif, that is).

Fortunately, they *do* ship lesstif, which is good enough for us.

> If things work out the way I'm hoping, my Gtk2-based version will be
> better in some ways than the current version of 'xgps', rather than
> a simple clone.

I'll welcome such a client, but it can't replace xgps, unfortunately.

The reason is not technical, it's political.  If we introduce a
gtk2-based test client and drop the motif/lesstif client without also
introducing a qt-based client, we look like we're taking sides in the
GNOME-vs.-KDE war.

I do *not* want to do that.
