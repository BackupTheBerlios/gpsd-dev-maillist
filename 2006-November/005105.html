> I'm not a "configure monkey".  However, I like to keep my system clean.

> Looks like I spoke out of place for Mac users - my system has  
> xsltproc installed, and I didn't install it.  It must be a part of  
> Apples coding tools.

I'm not surprised.  Mac OS X went to all XML all the time for its config
files -- makes sense that they'd have an XML tool chain preinstalled.

> Now, what does it require of me, to install documentation for gpsd?   
> Is there a separate tarball?  Is it documented & clear, what I have  
> to do, to install the documentation?

No, there's no separate tarball.  There's a step you need xsltproc for
that translates the XML masters in the source distribution to to man
page sources.  Then the normal man page install compiles them into
/usrt/share/man locations, or whatever the Mac OS equivalent of that
location is.

Does your configure notice that xsltproc is installed?  If so, xsltproc
should be run automatically when you 'make' to build the binaries.  If it
doesn't, we may need to tell configure something about the MacOS XML
catalog location or some other bit of configuration arcana.  

But since xsltproc is preinstalled, we should be able to get the Mac
install from SVN or tarball smooth enough that even Gary won't bitch
about it.

> Ps.  I've never had to download or build documentation separately,  
> for ANY other software that I've built from tarballs.....

And you won't this time either.
