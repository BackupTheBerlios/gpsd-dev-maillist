Gary E. Miller <gem at rellim.com>:
> I am tracking back the ntp PPS.  It looks like gpsd is poking the
> ntp shared memory each second, but ntpd does not see it.  You change
> anything around the ntpshm stuff?

I changed a variable name from shmTime to shmindex because it confusingly
had the same name as a member in a different structure.  Relevant rev
is 3718 and possibly 3717.  I bet if you revert to 3718 ntpshm will
work again.

The actual goal was to eliminate a core dump in gpsct[r]l from the
initialization of session structures being tied too closely to the
initialization of the ntpshm stuff.  Whoever wrote that code wasn't
being careful about the libgpsd(3) layer boundaries.
