On Mon, 2006-07-24 at 09:50 +0200, Rob Janssen wrote:
> I have my GPS receivers on normal serial ports, so I need such a script.
> However, I don't know how common this still is.
> 
> What I have now is very simple and hardwired for my config, but a
> distribution version would of course need a corresponding
> /etc/sysconfig/gpsd file with variables for "start gpsd y/n", commandline
> options (-n in my case) and serial devicename(s).
> 
> It may not be worth the trouble.  Any idea how many users still have RS232
> ports instead of USB?

I still have a receiver on a serial port, and I occasionally get reports
by Debian users who do as well, but I guess that's the minority
nowadays.

That's why the Debian package still ships /etc/init.d/gpsd, which is
disabled by default, but can easily be activated (run
'# dpkg-reconfigure gpsd' and answer some questions, or
edit /etc/default/gpsd[0]).

Cheers, Til


[0] /etc/default/gpsd:

# Default settings for gpsd. This file is sourced from
# /etc/init.d/gpsd.
START_DAEMON="false"
DAEMON_OPTS=""
DEVICES=""
