> I'm inclined to agree (and nearly mentiioned this point myself), but there
> is something to be said for collecting real numbers and making as few

OTOH: Having any numbers from any platform beats having none from one specific 
one.

Regardless where the profiling is done, it is beneficial.
You will always find special cases, where exact platform choice matters, but 
we can get most things taken care of easily.

BTW, if there was any hope that gpsd would run under Windows, it's gone now 
with the fd limiting change. Under Windows file descriptors are like random 
numbers.

Assuming fd's are in any way even slightly consecutive are not born out by any 
standard. I grant you, there are *implementations* where this is almost 
always true. It's in no way guaranteed, though, and can change tomorrow.

I got to believe that select(2) was not a good choice and have gone to use 
poll(2) instead for performance reasons and Windows compatibility.
