> Rob,
> So, for Fedora, RH, SUSE, you and I can agree that we need
> /etc/sysconfig/gpsd and /etc/init.d/gpsd files that default to hotplug.
>  I think we should also default to "-n" fewer issues from new users.

That is reasonable, at least for a serial-port connected receiver.
(where power consumption is usually not an issue)

> But, what serial port???
>
> /etc/sysconfig/gpsd
> SERIAL=/dev/ttyS0  # is the the most common non-usb serial port?

Don't set a default serial port.  The user has to select one.
Starting a daemon that opens some random serial port is a real debugging
nightmare.

The user will have to edit the config file anyway.
On SUSE there is a "sysconfig editor" that can be used for this, or one
could write some controlfile for YaST to have a more friendly dialog
screen.
I presume RedHat has similar tools.
But a textfile with some comments in it should not be too hard to
understand, compared to setting up a completely fresh init.d script.
(which needs quite some special entries these days)

Rob
