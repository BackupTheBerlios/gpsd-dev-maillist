>> The user will have to edit the config file anyway.
>
> No need for a config file just for the serial port.  How about opening
> /dev/gps, which if it exists is a link to the serial port to open.  Then
> no need for a config file.

This used to be a good solution.  ntpd uses it for receivers on serial
ports as well.

But now, major distros are switching to devfs (udev).  This means a
manually made symbolic link in /dev does not survive a reboot.

You need to figure out how to re-create the link at boottime, and that
again involves editing some config file.  Which of course again varies
between versions.  That does not really solve the problem.
