> Probably not.  I used to ship something like this, but the FC5 RPM
> already does something better, It installs a hotplug script that
> launches gpsd whenever a serial device of a type that might be a

I have my GPS receivers on normal serial ports, so I need such a script.
However, I don't know how common this still is.

What I have now is very simple and hardwired for my config, but a
distribution version would of course need a corresponding
/etc/sysconfig/gpsd file with variables for "start gpsd y/n", commandline
options (-n in my case) and serial devicename(s).

It may not be worth the trouble.  Any idea how many users still have RS232
ports instead of USB?
