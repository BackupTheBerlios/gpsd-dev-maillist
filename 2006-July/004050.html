I think Mike is asking what his client should do with the data and  
not what GPSd should be doing.

The answer depends on what the client needs the data for. If you need  
low latency raw data you take what you get when you get it. If you  
need a consistent data set and can afford to delay a second or two  
you can combine reports with the same time stamp. If you need a full  
data set for a specific time you will have to write your own  
interpolation or extrapolation engine. If you need to know what time  
it is so you can match the GPS data with reality you are on your own.
