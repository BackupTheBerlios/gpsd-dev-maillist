You are right, Rob.  OK, this is a policy decision:

1. As a large and increasing percentage of our user base is using USB
GPSes rather than serial, the default gpsd installation will remain
centered on autoconfiguring USB devices via hotplug.

2. init scripts for different distros will be carried in the contrib
directory or somewhere similar, if someone is willing to be
responsible for maintaining each of them.

I expect this will be acceptable to all parties.
