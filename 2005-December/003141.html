Hi Gary,

I just recall an earlier message on this list that this patch is
necessary in recent Slackware versions (hmm, here we go, Dave Cottingham
on gpsd-users, 02 Oct 2005 10:28:05 -0400, required on Slackware 10.1);
can't test myself.

Xm/Xm.h and Xm/XmStrDefs.h are both part of lesstif/motif. If one of
them is around, the other one will be as well. I guess the former used
to be pulled in implicitly somewhere along the line. But there shouldn't
be any harm in including both header files; if lesstif/motif is not
installed, xgpsspeed will not build anyway. xgps.c includes Xm/Xm.h as
well.

Cheers, Til
