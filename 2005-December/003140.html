Yo Tilman!

Well, I have a problem with your first patch:

- --- xgpsspeed.c.orig    2005-11-13 20:02:18.749331651 +0100
+++ xgpsspeed.c 2005-11-13 20:02:37.875855463 +0100
@@ -9,6 +9,7 @@
 #include <X11/Shell.h>
 #include <X11/Xaw/Label.h>
 #include <X11/Xaw/Paned.h>
+#include <Xm/Xm.h>
 #include <Xm/XmStrDefs.h>
 #include <Tachometer.h>


I have my Slackware 10.0 running X 6.9 RC2 and I do not have any Xm/Xm.h
include file in that X tree.  I do have one in my /usr/local/include but
that should not be counted on.  In any case I do not need your patch
to compile, so it may do more damage than good.  My guess is that this
should be part of the configure process to include only if it is known
to be available.  Not sure how we could know when it is really needed or
not.

The other patch looks OK.  If your linker is not set up to include -lm
there is likely see no harm in forcing it.  ANyone else have an opinion
on that?
