Marr <marr at flex.com>:
> I doubt (but haven't ever tested and certainly cannot prove) that any GPS 
> which allows dynamic configuration of the NMEA output stream would cause any 
> given NMEA sentence to be partially completed (i.e. cut off).

I agree.  This strikes me as so frantically unlikely that I think we
can design on the assumption it never happens.
 
> So, for my "two cents", we don't need to worry about NMEA sentence
> "cut-offs".  However, we still need to make sure that we don't
> (re-)design 'gpsd' such that it does something completely
> intolerable if the NMEA sentence stream is altered (added to and/or
> subtracted from) on-the-fly.

Why?  The only time gpsd ever hacks the sentence mix is in the driver's 
init string.  That's only going to get shipped once, at device-open time.
