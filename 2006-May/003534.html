>The scenario I'm worried about is this:  suppose we get a GPS that
>ships garbage rather than NaN -- something I think I've actually *seen*
>from a SiRF chip.  Then it could look like this:
>
>15000', 3D
>14500', 3D
>3728', 2D
>5496, 2D
>13000', 3D
>
>Better to report NaN than let this happen.

Agreed... so that's the question we need to resolve: which do devices 
really send with a 2D fix - something almost completely random (say, 
100+ times greater uncertainty than normal), or something which is 
probably close but not assured (like a DR lat/long fix - more like 
5-10 times greater uncertainty)?
