On 5/19/06, Eric S. Raymond <esr at thyrsus.com> wrote:
> This is the first I've heard of GPSes returning estimated altitudes
> when mode is 2D.  Having a nasty suspicious nature and some awareness
> of the sorts of corners low-end units tend to cut, my reflex would be
> to treat altitude as garbage unless the GPS affirms it's valid.
>
> Acting as a neutral agent is important, yes, but not passing the user
> bad data is even more important.  Can anyone address these questions:
>
> (a) What does NMEA 0183 say about this case?
>
> (b) What do the GPSes of your experience actually do?

SiRF receivers use an altitude hold, either from the last good 3d fix
or from user-specified initialization data.

The navigation solver is capable of providing a 2d position estimate
either using the least squares method or the kalman filter.

In the case that altitude is invalid coming out of the solver, the UI
(the part of the code that generates SiRF or NMEA messages) can delete
that data, and set a flag that says "there was garbage here, so i
cleaned this message up for you".

I'll have to put together a little test jig to obscure enough sky that
only a 2d fix is possible, and then see what the NMEA says.
