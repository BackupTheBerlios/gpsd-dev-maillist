We may get some interesting bits of this discussion coming through 
when the berlios listserver starts working again... but in the mean 
time:

I've just done some testing in a marginal sky view environment 
(indoors, inside a glass window and an aluminium mesh cyclone 
screen), and I found that - in this case - the altitudes reported 
with 3D modes were less accurate and more widely scattered than those 
with 2D modes.

Mode
====
2D	avg	62.9
	max	81.7
	min	60.9
	count	134

3D	avg	89.9
	max	161.28
	min	55.1
	count	172

These were all taken without moving the GPS unit at all, and the mode 
varied between 2D and 3D and back again.

Incidentally, the ACTUAL elevation here is supposed to be 10m, so all 
of those are well above what should be the correct value... but the 
real point of the comparison is that - at least for this sample - 
altitude data reported with a 2D mode appears to have been no more 
random than that reported with a 3D mode.

In the absence of evidence of random-garbageness, I'm still inclined 
to run with the "1+2" option from my earlier message - i.e. my 
existing patch plus possibly some additional status-handling work (if 
we decide that P= should be similarly filtered to A=).

Anyone else done some data gathering to either support or refute the 
above findings?
