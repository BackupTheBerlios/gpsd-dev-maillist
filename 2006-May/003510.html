Marr <marr at flex.com>:
> First, it's true that 'gpsd' doesn't currently have the facilities
> (other than via initialization) to send a command to the GPS
> receiver. But... will that always be the case? If so, there's
> probably no point in discussing this further.

Actually, it isn't the case now.  The N and B commands, among others,
may send control strings to the GPS.

I'm not pointing this out just to nitpick.  I'm fine with sending
control strings to the GPS, as long as it's done by gpsd itself so
that *gpsd knows when the sentence mix changes*.

Under that assumption, we never have to deal with the sentence mix
changing unexpectedly.  Though, actually, the way the framework is written
would make it pretty robust that way even now.
