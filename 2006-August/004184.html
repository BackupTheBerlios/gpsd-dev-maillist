The attached patch for revision 3417 (which should also work as-is on 
3418) builds and tests with correct behaviour on all client programs 
here, inasmuch as I can test it from a desktop machine with a very 
limited sky view from the only window.  I'll let folks that already 
know the regression testing process try it out on that, but I'm 
pretty sure this provides the functionality sought while allowing 
administrators of embedded systems to keep the footprint minimal.

Configure switches supported:

--enable/disable-casoc - sets DEFAULT buffering logic - 
default=enable (i.e. clear-at-start-of-cycle, hyper-correct but may 
cause jitter)

--enable-wired-policy - "wires-in" the above default setting: minimal 
footprint for embedded systems, but individual clients may not choose 
their own buffering logic - default=disable (i.e. above setting is 
only the default - individual clients may select logic to use)

Feedback & testing welcome!
