On Mon, 21 Aug 2006, Raphael Jacquot wrote:

> I mean, my receiver is proprely outputting NMEA and GPSD sees it
> correctly. however, it appears GPSD is not analysing it properly however
> as it sends out a triplet of analysed fixes every 3 seconds. it appears
> there are some buffering issues at work here...

I use a Garmin GPS-18 in NMEA mode constaanly and I get fixes every
second.  So something is subtly wrong here.  Can you send a sample of
the NMEA?

> the way I see it, gpsd should analyse the incoming nmea sentences,
> starting with GPRMC, and note which sentence comes just before it.
> then, as it sees this sentence, send one fix, before waiting for the
> next GPRMC.

This is a different issue.  If gpsd waits till later to send new info
as it is receives then latency is added that can be bad for some
users.  There is experimental support in the SVN version of gpsd
to enable/disable buffering at runtime.
