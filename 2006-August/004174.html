I have withdrawn the J patch. It seems to produce good live data, but
there is bad interaction I don't understand that breaks the regression
tests.  I'll continue working on this, but I don't want to let it 
block the upcoming release.  (Others can experiment by configuring with
--disable-wired-policy),

I've addressed Dave Hylands's problem by (a) rewriting the NMEA parser
so it doesn't need to know in advance how many channels a GPS
supports, and (b) increasing MAXCHANNELS to 20.  This, in turn,
required that the library major number be incremented.

The only other release blocker I know of is the build bug in
gpxlogger.  I'm awaiting a patch for this.
