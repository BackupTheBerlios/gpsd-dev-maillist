For the embedded users, here are some performance numbers from "slow"
machines. My test system for this is a Soekris NET4801, running
OpenBSD 4.0-beta. I'm running the latest-and-greatest code, compiled
with gcc 3.3.5, CFLAGS="-pg -g -O2 -static", so I could run profiling.

I've noticed that with 3 clients - sirfmon, xgps and cgps - gpsd seems
to pull about 4% cpu while syncing up, and then 3.5 - 5.0% while
serving the clients. After about 7500 seconds of wall time, I have
accumulated about 380s of cpu time. See attached output from gprof.

Soekris:
cpu0: Geode(TM) Integrated Processor by National Semi ("Geode by NSC"
586-class) 267 MHz

To give you an idea of the performance of this system, here's the
output of "openssl speed rc4 blowfish aes-128-cbc"
