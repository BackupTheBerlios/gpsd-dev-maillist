That is too minimal for many distributions, I think.  I am amazed that it
still works for Fedora or RedHat.   or do you make symlinks for this (in
rc3.d and rc5.d) manually?

Normally, there has to be a header like this:

### BEGIN INIT INFO
# Provides:       gpsd
# Required-Start: xntpd
# Required-Stop:
# Default-Start:  2 3 5
# Default-Stop:
# Description:    Start gpsd for clock sync
### END INIT INFO

This provides info for the runlevel manager to allow it to be
enabled/disabled from the distribution-specific control panel, and to
determine the name to be used for the symlink so that it will be started
at the correct point in the boot sequence.
