Like you, I don't think code should be debugged by tweaking it until
no more faults are apparent.  That usually is a long process and does
not produce good quality code.
With pointers, you need to understand what is happening and make sure
it is OK also in all boundary- and error conditions.  Else the problems
will come back to haunt you for a very long time.

It often helps to draw a schematic diagram of storage cells and how
they are linked together (i.e. what points where, and is the where not
removed before making sure nothing points there anymore?)
