Are you aware of postgis?  It's an extension to PostgreSQL to enable
position as a database column, so you can do efficient queries for
"near", "overlaps" etc.  It also supports coordinate transforms within
the database via proj.

  http://postgis.refractions.net/

I realize gpsd isn't trying to be a database, but knowing about
postgis might be helpful in thinking about data that's essentially "at
this place and time, foo was bar".
