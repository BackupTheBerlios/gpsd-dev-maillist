My vision is this: gpsd's job is to arrange rendezvous between a time-varying 
set of data sources and a time-varying set of subscribers.  

The data sources present as byte streams of unknown type, either from
hotplugged local serial devices or TCP/IP links such as NTRIP servers.

gpsd's first job is to deduce the protocol type of each stream.  Most
streams will just be GPS data.  These streams need to be made available to 
subscribers on port 2947.

Some streams will be DGPS correction data.  These streams need to be (a)
fed to attached GPSes, and (b) made available to subscribers on port 2101.
