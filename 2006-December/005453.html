On Sat, Dec 16, 2006 at 02:34:31AM -0500, Eric S. Raymond wrote:
> Robin H. Johnson <robbat2 at gentoo.org>:
> > As a point of reference, there is a way to do it without having the pain
> > of other commits.
> > 
> > svn copy -r ${REV} \
> > 	trunk tags/release-${VERSION} \
> > 	-m "Tagged for external release ${VERSION}"
> > 
> > Where REV is the SVN revision you want to make the branch with.
> So this triggers its own commit?
Sorry, I overly shortened it beyond being usable - lacking sleep last
night.

Form 1:
# svn copy -r ${REV}  \
	$URL/trunk \
	$URL/tags/release-${VERSION} \
	-m "Tagged for external release ${VERSION}"

Where URL is the SVN URL to the repo.

Form 2:
If you use relative paths, use the original command, leave out the -m,
and do a separate svn commit:

# svn copy -r ${REV}  \
	trunk \
	tags/release-${VERSION}
# svn ci -m "Tagged for external release ${VERSION}"
