I just pushed a minor change to Makefile.am that gets the distinction
between srcdir and abs_builddir correct in a few more cases.
conventional 'make check' (after ./configure in the srcdir, so srcdir ==
abs_builddir) was successful, and 'make distcheck', which

  makes a distribution
  unpacks it in a new directory, read-only
  creates a _build dir in the new srcdir
  in the _build dir
    runs $(srcdir)/configure
    runs make
    runs make check

now gets all the way to make check.
