For consistancy the DEVICE->activated timestamp needs to be in W3C format instead of unix epoch.

This issue is that the protocol says "0" means not activated.  

I propose making activated 0|1 or JSON ture/false then adding a timestamp property if activated.

Or make the activated false value null not null and true would be the W3C timestamp.
