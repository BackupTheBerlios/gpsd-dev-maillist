> to clean up VPATH builds (where one runs configure from a directory
> other than the source directory).
>
> I've had one report of failure - Chris Kuethe says that on OpenBSD it
> fails, and the build log shows that abs_builddir seems to be empty
> instead of being the path to the build directory (which is the source
> directory if you simply type ./configure).

The problem turned out to be using autoconf 1.9.x instead of something
newer.  With 1.10.3, things work fine on OpenBSD.  1.10 was released
over 4 years ago, so it seems reasonable to expect >=1.10.

Thanks to Chris for helping me debug this.
