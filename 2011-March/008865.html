> GPSD Folks,
> The TPV class has a time property.

> 
> The POLL class has a timestamp property.
> The POLL class calls the TPV array "fixes".
> The POLL class calls the SKY array "skyviews".
> 
> I propose that we change all "time" properties to "timestamp" when we use the new W3C format.

FYI:

I've already done timestamp->time in POLL; I considered the
non-uniformity a bug, and we're changing the timestamp format anyway
this release.  In 3.0 either all time fields will be "time" or
all will be "timestamp"; mixing is not acceptable.

I have skyviews -> sky and fixes -> tpv queued up in a stash for
right after 2.96. Good idea.  I should have thought of it when I
designed the POLL response.
