Bookends sound good, but technically it is validation or coherency
(checking), not synchronization.

In effect the data is coherent if the bookends match (on a forward
read with backward write).

I'm not opposed to bookend synchronization, but bookend coherency is a
little more precise.

>> If write copies the whole structure in reverse, with the last byte

>> with the first byte being the start sentinel, then you can't have read

>> intervening write occurs during any part of the copy.
