> Shared memory segment plus mutex, using mutex_trylock() to avoid blocking gpsd?

AFAIK, mutex_trylock() only works in threads in the same parent process.

The problem here is between gpsd and a client or clients.

Worse yet, if a thread dies while holding the mutex() there is little
hope of recovery.
