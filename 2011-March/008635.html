I'm trying to be disciplined about not doing things that might destablize
core code.  So there is a feature implemented in my repo that hasn't 
been pushed to the public one yet; I plan to merge it after we ship.

There is a new policy parameter, loglevel, in the WATCH comand.  Defaulting
to 0, it sets the maximum level of daemon log messages that will be shipped
to user sessions as they are issued.  

Each log message will be shipped as a LOGMESSAGE object.  For example:

{"class":"LOGMESSAGE",
   "level":1,
   "message":"LOG_WARN:speed less than 38,400 may cause data lag and loss of functionality\n"}

Of course, it is entirely up to clients what they do with this.
