Bernd Zeimetz <bernd at bzed.de> writes:

> I'd like to replace AC_CHECK_LIB and similar calls in configure.ac by
> PKG_CHECK_MODULES / pkg-config. So the question here is: is pkg-config
> available on all our supported architectures/operating systems?

I'm a trailing-edge type for requiring new tools, but I'm in solidly
favor of using pkg-config (and have been for several years).

pkgsrc has quarterly stable branches (without cute names, but they are
basically releases), and pkg-config 0.19 was first included in 2005Q3.
So by the time we add pkg-config, it will have had it for 5.5 years.

For others: pkg-config is basically a replacement for each program
having a foo-config program that you call with --libs and --cflags to
find out how to link against that program's libraries, and in the
process can replace checking for libraries.  It's based on data files so
it's cross friendly.

gpsd has been shipping .pc config files for a while, which is a separate
issue.
