On 10/03/2011 17:18, Bernd Zeimetz wrote:
> I give clang a try regulary. Works just fine.
> Would be nice if we could use dietlibc instead of libc, though.

If you check the latest uclibc (ie git) you should see a superb libc
with nptl support for a number of architectures.  The project doesn't
seem to be creating as much buzz about this latest pre-release as I
think it should do, but with uclibc you get very close to a full glibc
environment!

I haven't re-tested recent gpsd on uclibc, but previous versions have
worked without a problem.

(I have had a few issues with dietlibc not working with my hardened gcc
compiler in recent versions.  Not debugged things, but definitely some
issues there.  I only use it on one x86 architecture to link to
linux-vserver though so hardly testing it thoroughly...)

Good luck
