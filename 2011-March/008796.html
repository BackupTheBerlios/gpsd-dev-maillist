It is something I would remove or bypass

make gpsd seems to work - it may have to configure for all the other
stuff, but if I don't need them I don't have to build them.  make
install-sbinPROGRAMS looks like it would put it where it thinks it
should go, but I usually just move the gpsd in .libs (after screaming
in annoyance the first time at fraudulent gpsd in the root of the
build).

Or simply wait overnight.

Another thing which I havent tried is smuggling in the docs built on a
big machine.  This might sound silly, but I don't think the HTML or
groff or any other versions are ever any different across platforms.
There is no variant as to libs, integer type width, or endianness.  So
I theoretically generate them as part of the tarball and touch them so
as to prevent make from redoing them not unlike I'm supposed to run
./configure instead of ./autogen.sh

Not that I would have any docs on my chumby, but that would avoid the
redundant build, and I suspect a diff -r on the results on your Sparc
and your x86 woudn't show a difference in the relevant directories.
