Let me also suggest another optoin - I mentioned that X,Y,ZDoP are
being calculated, and it suggests "climb" would be too.

This really should be per-device, but I would prefer a
"calculate"=true/false option.  Most clients would not need the
calculated values but if more than one did it could turn them on.

Alternately we could add the calculations to one of the libraries or
something similar so that if they get the data they could call it to
fill in the computed fields.

Also, as to the "fork or not", NOFLOAT is a very fundamental change.
For many files, the first line will be #ifdef NOFLOAT, then a
different set, then #else, then the original, then #endif.

This will be a fork, but we can do it either explicitly, or have the
replacements inline in the code causing each file to get much bigger
until NOFLOAT becomes the base.
