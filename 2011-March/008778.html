Try running it on a zaurus, nokia tablet, or chumby - you might not
tend to notice it because by the time you get your coffee it will be
done on a netbook.

The advantage with Python is that it is a lot faster than bash, so
even if it was the same length it would run in less time.

And even better, I think I would be able to modify it for other (e.g.
embedded) platforms.

Editing one recipe in one place even if it is large will be a lot
easier than trying to get autogen.sh to create the next set when it
doesn't want to, then hand editing the ./configure, then hacking the
makefile anyway.
