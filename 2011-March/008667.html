> I would also note the Linux kernel uses none of this, and manages to

The linux kernel seems rather nonportable.  I've never heard of anyone
building it on a *build machine* which is not itself Linux.  The
autoconf test things are largely about OS/headers, rather than cpu.

For restricted environments, I think the real issue is not about
requiring libtool/pkg-config on the build host, but rather about making
the cross build product small (and not need python, which I think it
might not right now, but python is major bloat on the embedded scale).
