I was not offended.  You might be interested to know that one other
unusual measure we take against memory-management problems is that
malloc() is banned from the daemon code.  We get away with this because
gpsd processes what are in essence datagrams of strictly bounded length.
The benefit is that we can *guarantee* never having a memory leak.
