On Fri, 11 Mar 2011, Eric Raymond wrote:

> > Four easy examples:
> > 	Garmin USB
> > 	Garmin Serial Binary
> > 	ntpd
> > 	chronyd

> Are any still unverified?

They are all working.

And I am losing hope for any simple solution to the chrony permissions
problem.  gpsd drops root priviledges very early, just after the arguments
have been parsed.  The problem is in gpsd_ppsmonitor() which needs a
populated *session to decide which chrony files to open.  A lot happens
after dropping priviledges before *session exist.

At a minimum what has to happen is to identify the GPS device(s) on the
command line, open the chrony sockets, and save the file descriptors.

Ideally the code that creates the *sessions and saves the device names
in them would be moved just after parameter parsing, the chrony sockets
could be opened, saved in *sessions, and then root dropped.

That would require some serious hacking not appropriate this late in a
release for a new feature.  Unless someone can easily see a way to do it.

So I'll probably change the doc to note that opening chrony sockets as
root can only happen in foreground (-N) mode and allowing that in
normal mode is still a TODO.
