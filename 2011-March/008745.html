GPSD Folks,

> Since we aren't going to do plugins, it makes
> sense to have in addition to the ala-carte settings, a set
> of build options for light, normal, everything.

Compile time options do nothing for the 99% of the people out there who want there distro to install their GPS device correctly.

This is where I feel we (gpsd-dev folks) need more work.  The concept of plug-ins would resolve much of this headache as we could put plug-ins in different packages (with lesser supported ones out of the main stream build).  Let's face it; GPS chips (location aware chips) are going to be in the next laptop we need to be ready for that kind of pervasiveness.

We also need to integrate these plug-ins into the package management system so if I have a new "GPS(BU-303)" all I need to do is "yum install 'GPS(BU-303)'" and I'm up and running.

We, also, should be driving the interface and protocol that that GPS chips should be using in this future.  I'd like to suggest that serial is not the way to go.  And I think efforts like the "fanout" driver (http://www.linuxtoys.org/fanout/fanout.html) may be good building blocks.
