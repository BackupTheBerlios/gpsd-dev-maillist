Greg Troxel <gdt at ir.bbn.com>:
> I thought we were in feature freeze heading to a relaase.  I realize we
> had a regression failure and thus backed off to some other color where
> perhaps non-bugfixes are allowed, but I don't see how that makes it make
> sense to start doing things other than fix bugs.  We already are over
> 1000 commits since the last release (a nice side effect of git
> describe).

I'm more relaxed about modifications that don't involve logic changes. 
Especially when they enable our auditing tools to work better.

In this case, the benefit from double -> timstamp_t is that the static
checking in splint and cppcheck can catch invariant violations that
would otherwise be hard to notice.  And, in fact, I caught two very
minor bugs by doing this (as I suspected I might, because I noticed a
bad smell in some code).

Think of it as fumigation.  That's two bugs - one bad printf(3)
format specifier and one questionable assignment across types - that
we won't ship in released code.
