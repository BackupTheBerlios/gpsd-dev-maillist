OpenBSD doesn't like all these silly, unobvious components. I've
package gpsd thus:

-x11 (gui, requires X libraries and -main)
-php (webgui, requires php-gd)
-main (gpsd, curses clients, libraries)

Only in rare cases do we make a separate -dev package. It has to have
a relatively large volume of headers and static libraries. 50M of user
binaries shared libs and 50M of headers and static libs would very
likely get split. 500KB (gpsd on amd64) isn't worth it.
