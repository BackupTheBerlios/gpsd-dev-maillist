One other quick note - It is not merely wireless security.  If I can
compromise a Windows XP machine somewhere inside the network so as to
create a tunnel, gpsd has to be sufficiently secure (maybe a mode
disabling UDP, maybe other things) that I can't use it to get a shell
on the system where I can start looking around, either directly or
indirectly.  Knowing it is Linux, BSD, MacOSX or something else is
rather helpful to the hackers, and it bypasses any attempts at making
the TCP stack not disclose the system type or if some kind of proxy is
used to mask it.
