> First, both support multiple devices, but what is the best way to

Seems to me like the natural thing to do would be to implement a tab stack,
one tab per device.  This would have the advantage that tabs for AIS and
RTCM devices could easily have a different layout than GPS tabs without
either unduly complicating the other.

>  For maps, if there is more than one device, which one do I use
> (webgpsd checks the DoP each second and has a "best" which is always
> used for the web pages).

Map display as part of the per-device tab?  Pardon me if that's a naive
suggestion, I don't know the Google Maps API at all.
 
> So would this be a good idea to add to gpsd-ng?  Could it or an
> enhanced version replace some of the other client utilities?

It sounds far more capable than the PHP script we currently have, but
I'm not going to deep-six that unless Chris Kuethe (the author) agrees
that it is and doesn't raise serious dependency issues. In any case we'd
probably carry both for a while as yours gets polished and documented.
