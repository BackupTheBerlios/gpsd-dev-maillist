If I may suggest something here:

Why not have

    ?WATCH={"enable":true,"json":true,"classes":["TPV",...]}

and

    ?POLL={"classes":["TPV",...]}

? The list of requested classes should be a JSON list, not an object. If
the "classes": part is missing, default to all classes. This way, the
filtering logic could be the same for both requests.
