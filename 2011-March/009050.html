> Naive question : if you inline finctions in a dynamic lib, how is the compiler supposed to generate the code ? The compilation step should even be possible without having the .so on the compiling machine. I think it would be pretty bizarre if a compiler actually did inlining on dynamic lib as it would break the possibility of upgrading the lib and having the inlined functions upgraded.

The inlines are special cased.  You can see it in
/usr/include/string.h on linux systems - if various DEFINEs are set
right, it uses parts from architecture dependent areas - asm or inline
highly optimized functions.  Or just inline the lib routines.

There is even a special -m option for gcc on MIPS that controls this.

The short answer is that you wouldn't upgrade such a trivial function.
 It doesn't inline for debugging or if all optimization is turned off.
