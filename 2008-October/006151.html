Hi!

I'm a user of the current DBUS support (receiving fixes) and from my
point of view, there's a shortcoming if you have more than one GPS
receiver connected to gpsd. In this case, all GPS receiver's fixes
will be send to the DBUS and there's no way to separate them.

THat's not much of a problem if they're all next to each other, but
when they're several meters apart, that's not too funny.

I'd like wo tork on a patch fixing this. There seem to be wo possible
routes: (a) create a new DBUS message ("named-fix" instead of "fix"
additional containing eg. the receiver's serial interface name, or
hooking up another command to the socket interface to allow to choose
the device which is taken for DBUS fixes, ignoring all other devices.

What do you suggest in this case?
