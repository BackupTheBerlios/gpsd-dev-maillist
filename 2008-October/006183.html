Christoph,
Maybe yes and maybe no.  The Net::GPSD::get method opens up a telnet connection to the gpsd deamon and requests a position with an "o" request and then closes the connection.  In some versions of the gpsd with some gps receivers the behavior you describe is a known bug.  This is however not a bug in Net::GPSD.

The watcher mode is much better supported interface by the gpsd deamon.

http://search.cpan.org/~mrdvt/Net-GPSD-0.36/lib/Net/GPSD.pm#subscribe
