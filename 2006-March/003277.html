On Fri, 3 Mar 2006, Russell Nelson wrote:

> Yep.  xml is not a Unixy data format.  Unix programs are REALLLLLY
> good at handling newline-delimited data.  XML data can be written
> without a single newline.

I used to agree you with.  Then I learned xsltproc.  Now I love xml and
find it a great format to store data in.  With a simple .xsd file I
can validate any xml file.  With other .xsl template files I can convert
a .xml file to just about anything my heart desires.  Custom reports
are just a few key strokes away.

Now with a xsltproc (or the xml in PHP) and can convert just about anything
to just about any thing in a few lines of code.  Searches, sorts, extracts
are a snap.
