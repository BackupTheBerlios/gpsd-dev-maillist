On Tue, 28 Mar 2006, Chris Kuethe wrote:

> I'm not sure I like an XML protocol. Too hard to parse in shell
> script... but you're welcome to try sell me on it.

I used to feel as you do about XML files, then I learned about xlt
files.  With an xlt template you can convert XML to just about anything.
With XPath I can search an XML file for just about anything.

I now keep a lot of data sets in .xml.  Then to generate .csv, .html,
etc I just need to apply a template.  Also a lot of languages, like PHP,
have built in functions for reading .xml.  In PHP I can read, parse,
extract and print data from an XML file in less than 10 lines of code.

Even then, it would be best if the .xml output is optional, sorta like
the way nmap does it.  Nmap outputs plain text by default and .xml after
you set a command line switch (-oX).
