to the TODO list?
There is a discussion in may 2005 but it describes a slightly different
problem: drivers that support TIOCMIWAIT with receivers that do not
support it leave the thread hanging when the device is closed.  I fixed
that by only starting the PPS thread when the -n option is given to gpsd
(meaning the line is never closed before exit).  PPS only makes sense when
gpsd is running with the -n option anyway.  That patch was accepted.

I seem to remember the problem I recently described was also occurring,
and it is more or less what the TODO entry describes, but I cannot find
the mail message that brought it up.
