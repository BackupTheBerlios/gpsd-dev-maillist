On Jul 5, 2010, at 2:46 PM, Gary E. Miller wrote:

> pkg-config for OS X is trivially available from MacPorts.  hard to  
> believe
> anyone compiling on OS X does not have MacPorts installed.

Call me old fashioned. When something doesn't absolutely require  
something to build then I find no reason to install it. If the project  
managers wish to flat out require pkg-config, which in a way they have  
it is documented, that's fine. I'm simply appealing to the notion that  
if it's only required because you want to build something in QT should  
it still be required if you wish to not build the QT lib?

Therefore I find no reason why if someone on Mac OS X wants to  
download the source tree and build it should go through the hassle of  
installing pkg-config from MacPorts or from scratch just to build gpsd  
when for some cases it isn't absolutely required.
