On Thu, Jul 22, 2010 at 03:38:23PM -0700, Gary E. Miller wrote:
> Eric has set the  goal for gpsd to be as close to zeroconfig as
> possible.  So maybe just try to open gpsd.chrony.sock{0,1}  in
> all the obvious places, use it if found, and just skip it if
> not found.  Unless gpsd is the creator, in which case I would just
> fix the location.  /var/run or /var/run/gpsd sounds  good.

The socket is created by chronyd and the path is specified in
chrony.conf for each SOCK refclock. If the documentation said chrony
needs to be configured to create /var/run/gpsd.chrony.sock0 for
message decoded timestamps and /var/run/gpsd.chrony.sock1 for PPS
timestamps, I think it would work nicely.

> Got a link to how that socket is used?

Here is an example:
http://mlichvar.fedorapeople.org/chrony/ntp-chrony.patch

It's a patch for ntpd which disables everything except refclock
support and adds the SOCK support to pass the samples from ntpd
refclock drivers to chronyd.
