On 07/04/2010 08:37 PM, Michael R. Davis wrote:
>>> There's a BSD specific section to the
>>> gpsd.hotplug.wrapper "if [ -r /etc/default/gpsd ]; then"
>>> what should we do about that?
>>
>> Thats not BSD specific, but thats what Debian and Ubuntu
>> uses. Is there any /etc/default replacement in your distro?
> 
> $ cat /etc/sysconfig/gpsd 
> OPTIONS="-n"
> DEVICE="/dev/ttyUSB0"
> 
> Do we need to align the environment variables like GPSD_OPTIONS vs OPTIONS?

Would definitely make sense. Especially when you want to use the example init
script, not sure how compatble it is to other distributions, though. I would
prefer not to change them (again) in Debian. Until some time ago there was not
an extra option for the gpsd socket, but I've enhanced the debconf question a
lot and I'm abusing OPTIONS vs. GPSD_OPTIONS to distinguish between the old and
new stuff. Not nice, but ti works.

> Should we add GPSD_OPTIONS, GPSD_SOCKET and USBAUTO to packaging/gpsd.sysconfig?

Yeah, and then we should tell the hotplug wrapper to read the proper file, or at
least check for both. I doubt there is a distribution which ships both of them.
Also you might want to modify the init script to use the options as I've done
for Debian. (I *think* the last version is in git, if not - see
http://git.recluse.de/?p=debian/pkg-gpsd.git;a=blob;f=debian/gpsd.init;h=1c123b8714d2c2abcb1cd9c3fd80a86a1cc8fcba;hb=HEAD

> I think since we have not been back porting the packaging they have diverged quite a bit.  If we keep it up, we can keep them better aligned.  

That would definitely make sense.

> Should we have different folders in packaging for the different distros?

Not sure if that is necessary, the whole debian stuff is not in git anyway, but
the hotplug wrapper and the etc/default/gpsd file should be in git to be used by
other distributions.
