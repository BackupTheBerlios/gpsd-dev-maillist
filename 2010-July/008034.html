> If I understand the autofoo documentation right, shipping aclocal.m4

'shipping' is unclear.  Basically I think the situation is:

  aclocal.m4 should not be checked in to git.  people building from git
  need pkg-config installed

  when making a distribution tarball, not only is aclocal.m4 created,
  but also configure.  configure at run-time uses pkg-config to check
  for versions of other things, and they are deemed not found if
  pkg-config isn't there.


I think the only problem is that someone wanted to be able to build from
git without pkg-config.  One can't do that, just as one can't build from
git without autoconf/automake installed - and IMHO that's ok.
