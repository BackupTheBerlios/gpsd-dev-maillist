> > setup.py

> 
> No idea. -s should work well, I cound imagine that --no-print-directory prints
> too much in weird cases.

Not sure if this specific to GNU make, but -s doesn't suppress the
directory printing if it was called from another make (i.e. MAKELEVEL
variable is set) which doesn't use -s.

Try this Makefile:

version3:
	@make version2
version2:
	@echo --- make -s ---
	@make -s version
	@echo ---------------
	@echo --- make --no-print-directory ---
	@make --no-print-directory version
	@echo ---------------------------------
version:
	@echo 1.1

"make version2" and "make version3" have different output.
