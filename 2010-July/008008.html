"Gary E. Miller" <gem at rellim.com> writes:

> On Mon, 5 Jul 2010, Bernd Zeimetz wrote:
>
>> > Do you think we could entertain the idea of not requiring pkg-config to
>>
>> Is there no pkg-config on OSX at all?
>
> pkg-config for OS X is trivially available from MacPorts.  hard to believe
> anyone compiling on OS X does not have MacPorts installed.


As for MacPorts (formerly darwinports?), there are several packaging
systems for OS X.  There is also Fink and pkgsrc.  I use pkgsrc
exclusively, but indeed pkg-config is readily available.

pkg-config is a normal part of the world and it seems totally ok to
require it.  But I think that pkg-config itself has a 'check for
pkg-config' test and that will then cause all the rest to just fail.
Looking at the m4 sources, it indeed has such a check.


So the real question is:

  does gpsd, when autogen is run on a system with pkg-config, work ok
  w/o pkg-config

if yes, that's fine.

It seems entirely reasonable to require pkg-config on developer systems,
just as it's reasonable to require autoconf.
