Bernd Zeimetz <bernd at bzed.de> writes:

> On 07/13/2010 01:04 PM, Greg Troxel wrote:
>
>> huh?  It seems libtool/etc. think that installing .la is normal, so the
>> normal build should probably continue to install them.  (On my (pkgsrc)
>> system I have many .la files in /usr/pkg/lib.)
>
> In short: they're the reason for a lot of problems in distributions.
> See http://wiki.debian.org/ReleaseGoals/LAFileRemoval for (links to)
> the gory details. I know that Fedora dropped them since a longer time,
> not sure about other distributions.
> http://blog.flameeyes.eu/tag/libtool comes with some more details on
> the problem.  The only point where la files are useful is for static
> linking, but pkg-config should provide the same information, so the la
> files could be droppped.

So some (most?) Linux distributions won't be using these - that's fine.
There are other operating systems, and at least some of them haven't
gone on a ban-la-files binge.  Our build system uses them internally,
and there is not a special rule to install them.

So having packaging for Linux distributions that have no-la rules
respect those rules sounds just fine.  But there isn't consensus among
the entire free software community to remove them - if so the default
libtool/automake support wouldn't install them.
