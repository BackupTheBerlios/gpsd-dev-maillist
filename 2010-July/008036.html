well, in my tarball aclocal.m4 is there.  But really the distfile
creation is seriousy broken.

I wonder if it is because of

# Exclude version-control state from the tarball.  Needed because we
# grab the test directory wholesale. Hack appropriately when and if we
# change version control systems.
# \bug This is non-portable and depends on GNU tar beyond-POSIX extensions.
am__tar += --exclude '.git' --exclude '.gitignore'

and because there's no attempt to deal with finding and using gnu tar.
(My system has gnu tar as gtar.)  BSD tar (pax) pretty much complies
with POSIX, and doesn't have --exclude.  We really should be trying to
remove this kludge.

But, there's something else funky - my tarball has .git in it, which is
in the top-level directory, not test.  So I don't know how it got there.

Could others look at the tarball made by make dist and see if it's
excessively including things?
