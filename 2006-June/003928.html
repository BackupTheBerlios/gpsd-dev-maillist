I think a branch should be created whenever experimental developments  
are started. It provides both protection for the release branch and a  
stable base for the developments. When the developments have been  
tested and are deemed ready for prime time they can be merged as a  
single update to the main branch which then provides a coarse  
revision history that can have links to the development branch with  
the fine revision details.
