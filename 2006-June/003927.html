> This doesn't look right. Is the ITALK protocol working at all? And if
> so, is it dropping every other packet?

The ITALK support is just some first attempt to see if the protocol could
be recognized.  The "driver" is just a stub that logs every packet as
"unknown id".

IMHO it is not a good idea to include such code in a release version. 
People could think that there is support for something, while actually
there isn't.

For such a small project we would not want to keep separate "development"
and "stable" branches, but the least that could be done is to disable it
by default, and include notes in files like TODO and README that indicate
that one should not expect anything from this driver...
