Who said that there will be records with the same timestamp and different
data?

I think nobody suggested that.  The timestamp should be kept at the value
from the receiver.  I.e. gpsd stores a timestamp and the data valid for
that timestamp.  A client can poll it as often as it likes, and will
receive that same data until newer data (with a different timestamp)
becomes available.

There could either be double-buffering so you have an entire fix dataset
for a given timestamp, or a separate timestamp for each (group of) data
commonly sent in a single timestamped message.

There will be no confusing or contradictory data, there could only be some
data that is up to a second old.
