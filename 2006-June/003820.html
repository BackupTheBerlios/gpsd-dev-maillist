> closes the connection. even better if it's xml. sounds like it's wishful

I have suggested using XML before, but it did not seem very welcome.
I agree to the standpoint that XML usually just means a lot of overhead
(both in code size and data packet size) and buys you very little when
using the gpsd protocol only locally between the server and clients
compiled with a special-purpose library (like libgps).

Of course, it would reduce the discussions about message compatability
between older and newer versions, and it could maybe solve some of the
buffering issues, because it would be simpler to ship only those data
items that are actually known, without having to build a lot of knowledge
about the receiver protocol into the client protocol.

Maybe we could build optional XML message support (as a compile-time
option) first, and then work from there.  It could at least be used to
evaluate what the bloat really is, and if it really poses a problem.
