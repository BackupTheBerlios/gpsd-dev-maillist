Charles Curley <charlescurley at charlescurley.com>:
> I hit a dependency glitch while installing on a Fedora Core 5 box
> (upgraded from FC4, rather than a fresh installation). In installed
> the source RPM as usual:
> 
> rpm -ivh gpsd-2.32-1.src.rpm
> 
> I started to build:
> 
> rpmbuild SPECS/gpsd.spec
> 
> This complained about a lack of the openmotif-devel package, so I
> installed it:
> 
> yum install openmotif-devel
> 
> That brought in a slew of related packages.
> 
> Compilation then failed for lack of /X11/Xaw/Label.h. I was able to
> determine that that file belongs to libXaw-devel. I installed that,
> and compilation concluded sucessfully.
> 
> You might want to add that package to the dependency check. On the
> other tentacle, it could be a glitch in the openmotif devel package
> dependencies.

If you look in the spec file, you'll find that gpsd-clients does
in fact have "BuildRequires: openmotif-devel".
