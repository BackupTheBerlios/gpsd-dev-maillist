On Jun 10, 2006, at 3:53 PM, Eric S. Raymond wrote:

> Now that's weird.  Here's the code that does the hunting:
>
>     static unsigned int rates[] = {0, 4800, 9600, 19200, 38400,  
> 57600};
>
> 	if (session->baudindex++ >= (unsigned int)(sizeof(rates)/sizeof 
> (rates[0]))) {
> 	    session->baudindex = 0;
>
> How the heck can it get those nonsense speeds?

Post increment off by one error.

If rates[session->baudindex] is referencing the last speed in the  
list then session->baudindex == 5.
sizeof(rates)/sizeof(rates[0])) == 6.
5 >= 6 is false.
Incrementing session->baudindex to 6 and it now references an element  
off the end of the table.
