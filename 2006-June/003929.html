All...

After a couple days of [seemingly unnecessarily] tracing the packet
state machine, I've finally made gpsd work with my DeLorme Earthmate
(pre-2003 Zodiac Chipset).  My problem was that gpsd could never
complete the initialization process because it wasn't sending back to
the GPS the required "EARTHA" string.  My fix (i.e. kludge) is to add
to state EARTHA_5 (around line 283 in packet.c):

/* Why isn't earthmate_initializer (drivers.c) called automatically? */
write(session->gpsdata.gps_fd, "EARTHA\r\n", 8);

I suppose the next question to answer is why this isn't done
automatically.  Furthermore, am I the only one now using the Earthmate
since it last definitively worked (v2.25, circa-mid-2005, I think)?

If anyone is interested, I'm currently working on a Solaris 10 (SPARC
and x86) port.  Has anyone else already done this?

-- Gary
