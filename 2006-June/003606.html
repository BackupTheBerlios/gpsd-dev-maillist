Gary E. Miller wrote:

Hi Gary!

> Not sure how to test the ntrip.  I assume that ntrip just passes the corrections
> on to the client?  I wonder how I would go about using that to improve the
> accuracy of a generic NMEA GPS?

Ntrip works like Wolfgang's dgps-ip passing DGPS correction data to the
client (i.e. gpsd). For the correction data to be of any use gpsd needs
to relay it to a DGPS enabled GPS receiver. The receiver will then apply
the corrections to the GPS signal to produce a better estimate for its
position.
