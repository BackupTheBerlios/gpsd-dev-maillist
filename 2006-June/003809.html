>I have to say that I don't even properly *understand* that second line.

The way I read it, it means that it is not unreasonable to make 
something that requires the very latest version of free software in 
order to work, because you're not insisting that someone with an 
otherwise-functional older version "buy" the newer one.

It's a bit like saying it's not unreasonable to develop web pages 
that will only work on standards-compliant browsers: all current such 
browsers are free, so your visitors can't really cry poor when they 
say they want to stick with IE5.5 or Netscape 4.7.

That said, the kludged software would still stand to benefit from the 
server supplying M instead of O=?, even if it still works with old 
daemons.  Here's how one of my pieces of software currently works:

watcher mode on
report received=>
=> if mode=0, send "M" query to check if it's really 0 or 1
=> then process data

If I connect to a newer daemon that supplies M=1 rather than O=?, I 
save that extra query every cycle.  If I connect to an older daemon, 
it still works.

Incidentally, I'm not sure I've got the patch working (I may have 
made the Y reports disappear), so I'm not trying to sell anything 
I've made yet.
