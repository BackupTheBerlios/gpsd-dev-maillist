The new version includes M as part of the O report when M>1 (O=? is 
returned when M<=1).

>I wouldn't mind if a return like "GPSD,O=xxx,Y=xxx,M=xxx" was an option
>to the W command. "W=1 OMYX\n"  Why are the O and Y watcher returns on
>different lines anyway?

Watcher mode is effectively a parsed data push method, not a timed 
client pull method.  When it receives position data from the GPS, it 
sends an O report.  When it receives satellite data from the GPS, it 
sends a Y report.  This is why you generally only get one Y report 
for several O reports (satellite data is provided by most GPS units 
less often than position data).

If you want to query the GPS instead, that's certainly an option - 
indeed, I'm currently working on a client program that uses watcher 
mode AND occasionally queries for more information (an M command when 
O=? is received to distinguish between M=0 and M=1, since position 
sentences with either of those modes cause an O=? report)... the 
logic is simply 'if fix.mode=0 qps_query("M\n")'
