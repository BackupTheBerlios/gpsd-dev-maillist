> Rob Janssen <gpsd at xs4all.nl>:

>> > Can you capture a log from the device that will reliably cause gpsd to
>> > lock to 7N2 within gsfake?
>>
>> Is there a record of receivers that really need this 7N2 mode and won't
>> work in 8N1?
>
> Yes.  The San Jose Navigation FV-18.
>
> The NMEA standard allows either 8N1 or 7N2.

Ok.  Then my suggestion is to change the packet.c statemachine so that it
recognizes the NMEA leader with high bit set ('$'|0x80 etc) and switch to
7N2 mode only when that matches.

Binary modes that use the full byte cannot be 7N2 anyway, and although
there is a binary mode that uses 7 bits, we don't currently support it.
(Sony)
It can be similarly detected and handled.

This should speed-up detection and avoid nasty hangs.
