>Robin L Darroch <robin at spade-men.com>:
>>  why the opposition to implementing the simple fix?
>
>Because it relieves the pressure to do the right thing.

I guess my concern is that the extension of this (i.e. per-client) 
may turn out *not* to be the right thing, and the only way to find 
out one way or the other is to put the compile-time fix out there.

Otherwise, we're stuck trying to fix a theoretical argument instead 
of a real problem.

Here's something of a summary:

Data-logger client + binary GPS source
Current version: Works
Compile-time patch: Works
Per-client concept: Works

Query-based client + binary GPS source
Current version: Works
Compile-time patch: Works
Per-client concept: Works

Simple watcher client + binary GPS source
Current version: Works
Compile-time patch: Works
Per-client concept: Works

Mix of data logger and query-based clients + binary GPS source
Current version: Works
Compile-time patch: Works
Per-client concept: Works

Data-logger client + NMEA GPS source
Current version: Works
Compile-time patch: Works
Per-client concept: Works

Query-based client + NMEA GPS source
Current version: FAILS
Compile-time patch: Works
Per-client concept: Works

Simple watcher client + NMEA GPS source
Current version: FAILS
Compile-time patch: Works
Per-client concept: Works

Mix of data logger and query-based clients + NMEA GPS source
Current version: FAILS
Compile-time patch: May not work (? - still no proof that these are broken)
Per-client concept: Works


So we're holding back something that fixes three real failure 
scenarios for one, remotely possible failure that doesn't currently 
work anyway.

If it can be shown that there is a similarly simple fix that allows 
per-client buffering logic, then I'm more than happy to see it 
implemented along with a compile time option (because the server 
administrator should be allowed to choose which logic they want as 
the default)... but I don't think per-client is a simple fix, and I'm 
really not sure it's worth it based on the above.  If the above is 
not fair in terms of what works and what doesn't work, then I'd be 
happy to see corrections.
