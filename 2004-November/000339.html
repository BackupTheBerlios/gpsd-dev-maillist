To avoid error messages, the file should be created only 
when  /etc/udev/rules.d exists:
...
if [ -d  /etc/udev/rules.d ]; then
   ....
fi
...
I dont know when udev re-reads config file, but it is cooperating with hotplug 
and should be able to create new USB device on the fly.
For standard serial port, maybe create both udev entry and a symlink (if it 
does not already exist). Then the symlink will work until reboot.
