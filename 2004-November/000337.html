Last night I managed to buld gpsd by changing a specfile line
--------------------
%configure
--------------------
to
---------------------
LIBS=-lm ./configure --prefix=/usr
---------------------
but I don't know if it will be needed for other distributions (or even future 
versions of FC)

Another problem with Fedora Core 3 comes from udev filesystem.
The /dev filesystem is created from scratch on every boot and /dev/gps link is 
erased. I had similar problem with softmodem which also required a symlink. 
There are two solutions:

1) use only real serial port name (must be configurable)
         gpsd 'cat /etc/gpsd.conf'
    (gpsd.conf  should contain a line like "-p /dev/ttyS0")

2) tell udev to re-create the link. 
If I am correct, the file /etc/udev/rules.d/gps.rules containing a line
KERNEL="ttyS0", SYMLINK="gps"
must be created (by some kind of install script for convinence?)
