Solving the no-GPGSA problem without a new state variable turned out
to be trivial, just a matter of removing one line of code.  It turns
out that the no-fix value of mode is 1 rather than 0; I was setting it
to that at gps_t initialization time.

I just checked in a change that removes that line and adds a check for
mode == 0 in the logic that forms protocol replies; the effect is that

(1) M will now return ? if no GPGSA has yet been seen, and

(2) the "gpsd is confused" test will not fire until GPGSA has yet been seen.
