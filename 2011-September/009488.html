Ed,
It took me a long time to "get" SCons but you don't use the prefix at all.
 
> Hi, I was having a bit of a stab at converting the gentoo packaging to
> use the new SCONS system and gpsd3.x and I'm not sure of the 
> correct way to install into a prefix directory?
 
export DESTDIR=$RPM_BUILD_ROOT
scons install
 
Have a look at the RPM Spec in the gpsd/packaging/rpm/gpsd.spec.in
 
Once you complete your spec-equivalent you might want to contribute it back to the project.
 
To build the RPMs from the tar ball, you only need to `rpmbuild -ta gpsd-3.0.tar.gz` since we have the spec file in the project.
 
Thanks,
Mike
 
 
 
%build
#Note: prefix must be set to / since the current scons build only supports relative paths
scons %{_smp_mflags}         \
   prefix=/                  \
   bindir=%{_bindir}         \
   includedir=%{_includedir} \
   libdir=%{_libdir}         \
   pkgconfigdir=%{_libdir}/pkgconfig   \
   sbindir=%{_sbindir}       \
   mandir=%{_mandir}         \
   docdir=%{_docdir}         \
   PYTHONPATH=%{python_sitearch}
scons build
%install
rm -rf $RPM_BUILD_ROOT
export DESTDIR=$RPM_BUILD_ROOT
scons install
