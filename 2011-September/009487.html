> Hi, I was having a bit of a stab at converting the gentoo packaging to
> use the new SCONS system and gpsd3.x and I'm not sure of the correct way
> to install into a prefix directory?

> Background: Gentoo runs the installer in a sandbox and ensures that the
> install script doesn't touch anything outside of a temp dir - what is in
> that temp dir at the end of the install is then recorded and applied to
> the real filesystem. 
>
> I tried "scons prefix="/var/tmp/portage/sci-geosciences/gpsd-3.1/image/"
> install" on gpsd3.1 and my packaging system chokes with the errors below:
>
> Am I an idiot and doing something wrong, or is the installer missing an
> instance where it should apply a prefix?

I don't know about gentoo but pkgsrc does something similar.  There are
two concepts that need to be kept separate:

  prefix: where the code will be when it is run

  destdir: where the code will be installed by 'make install' or
  equivalent.

So in pkgsrc it is usually "--prefix=/usr/pkg" (autoconf0 or
"prefix=/usr/pkg" (scons).  And then there is an install wrapper script
that puts things in
/usr/pkgsrc/category/name/work/.destdir/usr/pkg/bin/foo instead of
/usr/pkg/bin/foo.

So I suspect you want a DESTDIR at install time, not a different prefix.

I have no idea if scons can do that.
