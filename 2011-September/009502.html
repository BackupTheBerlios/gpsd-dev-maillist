On 09/09/2011 01:03, Eric Raymond wrote:
> Ed W <lists at wildgooses.com>:
>> Note, I examined other build recipes in gentoo and they all use this idiom:
>>
>>     scons prefix="/" install DESTDIR="${D}"
>>
>> This means that gpsd is unusual in needing:
>>
>>     DESTDIR="{D}" scons prefix="/" install
>>
>> I don't know enough about scons to comment either way, but can someone
>> consider whether it's common/normal/suggested to support the top idiom
>> in gpsd?
> This could be done, but it would be difficult to support both forms with
> predictable behavior.  Is there any functional reason to switch?

As I say, it's just the preference of the gentoo build system - it has a
"debug" flag and it uses that to decide to strip or not.  Gentoo tries
to give uniform configuration across all builds, so I can turn on the
debug knob in one place and any subsequent builds then have debugging
enabled.  So stripping is done defacto by the "install" system and
doesn't need to be done by the build
