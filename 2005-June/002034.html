> 4. Enable gpsd to set the system timezone
>
> David Wheeler (the "Why OSS/FS? Look at the Numbers!" person)
> suggested this at FISL.  If we're going to give gpsd the capability
> to set system time via ntpd, why not let it set timezone as well?
> A good thing for hackers travelling with laptops!
>
> The major issue here is that I have not yet found code, or a
> database, that would allow mapping from lon/lat to timezone.

I think this is very difficult.  As already explained, not only the
timezone borders are complicated (it would be simple when it was a "15
degrees 1 hour" system but in reality there are many exceptions.  We are
at 5 degrees east but we do not use the -7.5/+7.5 timezone (UTC), we are 1
hour + DST ahead.

Furthermore, setting the timezone requires a reboot (in practice).

I think this functionality does not belong in gpsd anyway.
Those who want it can code a CLIENT application that uses gpsd to find the
position, and sets the timezone.  NTP is not involved in this, all NTP
time handling is done in UTC.
