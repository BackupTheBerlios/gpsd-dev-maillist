"Eric S. Raymond" <esr at thyrsus.com> writes:
> I need to understand better what the dgpsip server actually does.

Is this referring to the dgpsip server at dgps.wsrcc.com? 

That server shouldn't be doing anything surprising.  It just has your
standard poll loop looking for input from the DGPS radio and sending
it to all the TCP sockets that can accept data.  It is just your
standard user-mode program.  

The machine the server is running on is tracking openbsd-current with
~1 week granularity.  It is entirely possible that some brand-new tcp
bug is causing funny interoperability problems where the remote client
wedges in a read.  Maybe a full tcpdump along these lines is called
for???

   tcpdump -w tcpdump.out -i EXTERNAL_IF -s 1500

Then using ethereal ("ethereal tcpdump.out") look for funnies around
the time of the hang.
