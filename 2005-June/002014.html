I'm still waiting to find out from Andreas Brauchli if the last macro
change (reverting to rev 2560) fixed his known bugs in the Sirf-binary code.

I believe the Garmin, Zodiac and NMEA drivers are currently working OK.
If there are any known bugs, please speak up.

I don't know the state of the TSIP driver.

splint has revealed four bugs (now fixed) one pending problem in the
framework code (long-int confusion).  This is not serious, but is a
potential port bug on odd architectures.  The framework code is
otherwise in excellent shape, with no known bugs.

splint currently generates 78 warnings.  These are now almost all
about potential memory-allocation problems and leaks.

Since I started trying to splint-clean the code we have had exactly
one (1) unrelated bug report, a build problem when three drivers 
are configured out.  This has been fixed.

I'm looking at factoring out the SirF/TSIP byte-extraction macros into
a module so they can have their own unit test and be re-used by future
drivers.
