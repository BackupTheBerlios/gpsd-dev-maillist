There is an existing bug that I believed I had fixed some time ago, but
yesterday it happened again:

When gpsd is running and I start a backup of my system, which starts and
mounts a normally idle disk and does a tar cvzf to it, the load of the
system is quite high and gpsd seems to get out of sync with the received
data from the SiRF receiver.
It then gets stuck in a tight loop.  As gpsd is running at nice --10, this
further increases the load and slows down the backup to a crawl.
Killing gpsd makes the backup finish and then it can be started again.

While gpsd is spinning, the message parsing is stuck but the PPS handling
operates normally (so my clock remains locked).  I think the cause is that
the state machine has detected an error and does not recover.
I hoped my fixes to the "discard 1 character" routine a while ago had
fixed it, but they haven't.
