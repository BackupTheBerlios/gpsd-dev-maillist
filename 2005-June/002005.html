On Mon, 6 Jun 2005, Remco Treffkorn wrote:

> What I suggested in an earlier message was that we create a stable branch
> based on an earlier release, that is maintained only with critical fixes,
> while the development branch evolves.

I also like the stable/unstable branch concept.  That helps casual users
of the project to pick a good version for their needs.  We can not expect
every distro to follow this project and therefore know which golden
version to ship to their users for the next 12 months.

We need not maintain two branches in CVS for this, just a note on the
website of the most recent version that has no serious bugs in it and has
been tested in the wild for a week.
