The attached patch to gpsd.spec.in uses the %makeinstall macro to
package gpsd.  The resulting gpsd and gpsd-devel packages contain the
exact same file lists, plus the following two additions to the -devel
package:
	sirf.h - If this is not supposed to be distributed, it should be
	         removed from include_HEADERS in Makefile.am.
	libgps.a - This is typically included in devel packages.

The resulting packages fix my original problem: I can now link against
libgps.
