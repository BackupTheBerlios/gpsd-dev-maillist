Yo Eric!

I have managed to subscribe to the SVN change email and check out
the current code.

There are only 2 things I would like to see before the garmin code
is ready for 2.13.

1. The GPGSV sentences emited from libgpsd_core.c are corrupted.  YOu
can not take the NMEA from garmin USB driver and feed it back in with
gpsfake without a core dump.  I am working on a patch to fix this.

2. Do a once over of the PI issues.

Since I mostly have the day off I am working on these now and should
have a fix shortly.

After a working 2.13 is out we need to revisit the init code.  I notice
that the current init code assumes success.  There is no way to
return a failure code.  So I am not sure what the garmin usb init should
do if it fails to find the garmin driver or the a garmin device?  It
could just abort right there.  If there is a momentary USB disconnect
this is not the right thing to do.  Ideas?
