On Mon, 21 Feb 2005, Eric S. Raymond wrote:

> Gary, have you got any Garmin changes you want in?

The present trunk code for the garmin USB interface seems to be working
well when things are working.  It should also be suitable to Gentoo
although I have no way to test that.  So this would be a good point for
a 2.13.

As soon as you have a 2.13 release.  I can check in some error checking code
for the garmin.  At the momemnt the garmin driver will hang on
these conditions:
	1. no garmin_usb driver in kernel
	2. no garmin USB device attached to expected port
	3. garmin device unplugged after gpsd starts.

First pass, all three need to emit a suitable error message and die.

Second pass it would be nice if the driver had a retry strategy, that
way the user could start gpsd at boot time then conect and disconnect
the garmin usb device as required.

> I'm still interested in having probes for FV-18 and Garmin

My python skills are atrocious and I have no desire to improve them.

The current garmin init polls the linux driver for version, then polls the
garmin usb device for version.  I could pull those our in to a subroutine.
Then maybe someone could figure how to call that from python?
