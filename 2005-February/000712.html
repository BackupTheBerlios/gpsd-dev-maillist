Hi Eric,

my fix looked somewhat different. I was using short options (-s instead
of --speedunits and -a instead of --altunits); this way, they could be
handled by getopt() and be compatible with future command line tools
that don't link against libXt (e.g. gpslogger, still in the making...).

And I was defining in gps.h wich units to use by default
(DEFAULT_SPEEDUNITS, DEFAULT_ALTUNITS), to make it easy to switch from
imperial to metric to nautical units at compile time.

Still interested?
