GPSD, the GPS service daemon, has recently added support for the
USB Garmin driver.  You can read more about this project at
<http://gpsd.berlios.de>.

I am trying to make GPSD totally autoconfigure itself once it is 
given a device file to read.  For serial and USB devices, it knows
how to hunt through plausible baud rates looking for GPS packets.

I am in the process of implementing a state machine that will reliably
detect NMEA, SiRF-II and Zodiac data packets so the daemon can
automatically select the right driver type by sampling the input stream.

The one missing piece before can throw away the option for specifying
a driver is a way to detect that the device file given to GPSD is
attached to the Garmin driver.  

How do you recommend that I do this?  Should I test for major number, 
and if so for what major number?
