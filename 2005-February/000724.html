Yo Eric!

On Fri, 25 Feb 2005, Eric S. Raymond wrote:

> The one missing piece before can throw away the option for specifying
> a driver is a way to detect that the device file given to GPSD is
> attached to the Garmin driver.

The Garmin USB driver just shows up as /dev/ttyUSB0.  No way to tell
by looking at the device what it is.  Even if you can tell if the
garmin_usb driver is in the kernel, no way to tell if it is connected to
a Garmin USB device w/o querying it.

The query code is in the garmin_init.  It sends a binary packet and
expects a binary response back.  You could try that.

You could narrow things down a bit with some rules.  AFAIK, the garmin_usb
driver only works on recent Linux 2.6 kernels.  So if you detect anything
else you could rule out the garmin.  If you have 2.6 then you could
play in the /sys tree looking for clues.
