I see.  One of the things really at issue here is whether the site
design should be optimized for first-time users or for repeat
visitors.

Repeat users are probably mainly interested in the the Download and
News sections.  New visitors need to see the Screenshots, and
Clients parts.  You left out the section on other GPSDs, but that is also 
probably mainly interesting to new users.

Documentation links and mailing lists should be interesting to both groups.
History and the Guide to GPS Hacking are already separate pages.

How about if we group this stuff by who will want to see it?  So,

* Home page contains About followed by News, followed by Documentation, 
  followed by Mailing Lists.

* The new user's page includes Screenshots, Clients, and Other GPSDs.

Menu looks like this:

    Home
    For New Users
    Compatible Hardware
    Project History
