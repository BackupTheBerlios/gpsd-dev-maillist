Gary E. Miller <gem at rellim.com>:
> Can it handle devices like the EC-10X that only output NMEA data
> every 2 to 8 seconds?  EC-10X and others need that.

Try it and see.  

The difference in the new code is that it can skip any amount of baud barf 
and still resync properly when it sees a well-formed packet.  So there don't
have to be delays after a baud-rate change; the state machine just chews 
up the garbage that the UART emits while it's settling and spits it out.

I suspect this change will actually make devices with funky timing
work *better*.
