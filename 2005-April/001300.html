> The current version of gpsd.hotplug does not query udevinfo to get the proper
> serial device name.  On my machine (gentoo) udev maps a usb-serial converter
> (tty class device) to /dev/tts/USB[0-9].  The existing hotplug script relies
> on a hard coded device name.  This patch will automatically query udefinfo for
> the proper device name given to the new device and update gpsd.  I suppose if
> you were not intending to work with udev and hotplug, then you would not want
> to apply this patch.

This looks like a good idea, but it's against a version of the hotplug
script that has been obsoleted by the big security fix in SVN head.  Would
you please grab SVN head and send a patch against that?
