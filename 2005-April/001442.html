Eric S. Raymond wrote:
> Petter Reinholdtsen <pere at hungry.com>:
> 
>>Any way to drop this design choice, and make it possible to run gpsd
>>without root access?  I would feel a lot safer it is ran as another
>>user.
> 
> 
> So would I.  Unfortunately, gpsd has to open and reopen tty devices,
> which are owned by root.  Because it has to do so at unpredictable 
> times as hotplug events come in, that privilege can't be dropped 
> after startup.
> 
> If all tty devices were owned by a single group (say, uucp) then gpsd
> could be non-root but sgid to that group.  This may be a little hard to
> arrange for hotplug devices.

not necessarily.
in fact, you can specify the mode the devices are created with

in /etc/udev/rules.d
you can add your own file (for instance, 10-local.rules ) next to the
standard 50-udev.rules file

in it, you have a line like this

BUS="usb", SYSFS{product}="Crystalfontz CFA-635 USB LCD", NAME="%k",
SYMLINK="serial_lcd", MODE="0666"

see, the MODE specification ? this will create the device node with 666
so that anyone can read/write this device, and, at the same time, create
a symlink with a fixed name (USB serial devices like to change number
when plugged/unplugged...)
