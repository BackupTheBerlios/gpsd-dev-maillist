OK, this is *exactly* the same pattern of results we got from Gene Imes.
I think it tells us the following things:

1. This is a not a client problem.  That's established by your first test.

2. The problem is specific to binary mode.

3. This almost certainly isn't a gpsd problem either.  If gpsd were garbaging
the speeds it got from the chipset, it would trash them indifferently whether
or not the static-nav bit was on and whether or not the GPS was in motion.

I already believed 1 and 2 from Gene's report; you're helping confirm 3.
You've also banished the outside possibility that Gene's report was user
error.  This is all good.

I think this is a SiRF-chipset problem.  There must be some speed- or track 
filtering that is enabled in NMEA mode but not in binary.  What we need to
do is find the right bonary-mode control bit to enable it.  

I'll bug SiRF tech support about this.
