This is quite common amongst "technical" users.
They want untranslated text (because local translations often are
confusing), but they want to use the local conventions.
It is like installing an English version of Windows but setting
the national settings to your own country.

Unfortunately it is not that simple in many Linux distributions.
I use SuSE Linux 9.2 (a German development...) but it allows only
the selection of "language" in its graphical system management
program.  Based on the selection it sets these variables:

LANG=en_US
LC_CTYPE="en_US"
LC_NUMERIC="en_US"
LC_TIME="en_US"
LC_COLLATE="en_US"
LC_MONETARY="en_US"
LC_MESSAGES="en_US"
LC_PAPER="en_US"
LC_NAME="en_US"
LC_ADDRESS="en_US"
LC_TELEPHONE="en_US"
LC_MEASUREMENT="en_US"
LC_IDENTIFICATION="en_US"
LC_ALL=

It is possible to override some of them in an "advanced" mode
in this program, but not LC_MEASUREMENT.  Of course you can override
it from many places but it should not be expected that the average
user would do that.

IMHO:
1. there should be only a single switch.  or is there a need for
expressing speeds in mph while measuring distance in meters?

2. it should be controllable by a gui widget in xgps.  probably the last
state would not be saved because that is difficult in Linux, but at
least it could be changed by a user that does not know about commandline
options or X resources (how many users know about X resources??)

If course I would vote for defaulting to metric units (that is what
the GPS outputs) but apparently we have to live with the fact that the
whole world knows how to convert feet to meters but the US people don't
know how to convert meters to feet.
