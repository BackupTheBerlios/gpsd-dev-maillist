Howdy,

A few changes in my copy of the specfile might want to swim upstream.

http://www.greenkey.net/~curtis/rpm/src/SPECPATCH/gpsd.spec.patch.html

You'll see that I spun off the client apps package to keep the X 
dependencies out; for those of us running daemon-only on leaner machines.

Also, I added a couple missing build deps and perms fixes.

You probably don't want profiling on by default, though...

Are you aware of the /proc/tty/driver/ perms conundrum? Since 
garmin_probe() attempts to read /proc/tty/driver/usbserial but cannot as a 
normal user since the kernel restricts /proc/tty/driver/ to root only.

The ugly workaround is simply to chmod go+rx /proc/tty/driver/ as root 
before starting gpsd as a normal user, but I read this can have unpleasant 
security ramifications.

http://www.cve.mitre.org/cgi-bin/cvename.cgi?name=CAN-2003-0461
