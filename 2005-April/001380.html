> Related issue: currently, once a hotplug script has invoked gpsd, it
> never goes away.  I've thought about implementing a time-to-die option
> so that if there have been no client requests in (say) three hours gpsd
> would quietly exit.  Any time-to-die setting of less than thirty days
> would
> as a side effect completely prevent stale leap-second corrections.

Some people may want to use gpsd to provide time sync and may rarely start
any client (e.g. in a fixed installation).
I think the default should be to not unexpectedly die after a while,
because it can have very surprising results.
Or at least, the -n flag should turn this feature off.
