Andreas Brauchli <a.brauchli at elementarea.net>:
> > That you don't see this when using sirfmon is interesting.  If you telnet
> > to port 2947 and do 'w' to enable watcher mode, do the speeds in the 'O'
> > report look reasonable?

> gpsd output from an immobile BU-303 unit: (watch how it switches
> from 0 to 8-12 and then gets back to normal)

Yes, I see that from my BU-303 too.  On mine, it goes away when I use 
sirfmon to set the static-navigation bit ('c' in sirfmon).  

Gene Imes, on the other hand, reports that while setting static
navigation suppresses speed noise when stationary but not when he's 
in his car.

So I'm not certain that the small fluctuations you see when stationary
have the same cause as the very large spurious speeds you see when moving.

Please try the following experiments.

1. Watch the w output with static navigation off while moving.  Do
   you see crazy speeds?

2. Watch the w output with static navigation *on* while moving.  Do
   you see crazy speeds?

3. Switch to NMEA mode with N=0 and then watch the w output. Do
   you see crazy speeds?

The point of watching w output is to try to rule out the client as a source 
of bugs.
