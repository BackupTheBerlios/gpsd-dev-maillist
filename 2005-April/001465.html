> > I think I like the idea of having gpsd drop privileges after creating the
> > SHM segment.
> 
> It would also solve problems with opening the serial port.
> Just do not close it after you dropped privileges.  When you want to
> shutdown the port, it is usually possible to set it to 0 baud and keep
> it open.  But this may need some testing.
> Of course it will either need configuration (like the -u username) or a
> hardwired username that is assumed to be always existing or to be created
> during installation...

Nahh, it's much simpler than that.  The logic is in SVN head as of just now.

First it looks for the owning group of the device given on the command
line (or /dev/gps if none is given).  If there is no such device, it
assumes that the owning group of all tty devices will be the same and looks
at /dev/ttyS0 to get it.  If both these checks fail, it keeps group root.

No command-line option, no config file.  No configuration.  Keeping it simple!
