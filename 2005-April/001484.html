Yo Eric!

On Fri, 29 Apr 2005, Eric S. Raymond wrote:

> Will this approach fail if the Garmin is only plugged in after gpsd starts?
> That is, can we count on the Garmin driver being loaded at boot time?  If
> not, life gets rather more complicated.

I never use modules if I can avoid.  That puts me in a smal minority.
Most people will use the garmin_gps as a module to be loaded when they
insert the garmin gps.  So, if gpsd is started at boot-time and the
garmin inserted later, this will fail.

> We might even have to re-elevate to root on each device open.  That can
> be done under Linux, but portability could be a problem -- I wouldn't be
> surprised if Mac OS X bit us in the butt on this one.

I think what sshd does is run a helper program with root perms and
everything else has no priveldge.  These days so many things are running
as nobody that they decided to run as their own user.

Alternactively, if this is the only root issue after startup we could
just look for the /sys/ entry.

Here is what I found on the ntpshm.  Looks like the shared memory is
opened 0700, as root, then later used as nobody.  I tyied moving the
ntpshm_init() until after the drop priveledges but same results.  Dunno
what the implications of opening the shared memory as 0666 instead of
0700.  BTW, shmget() ignores the 0111 bits.
