i'm one of the guys wondering why the americans haven't adoptend the
international metric units (SI).. but i'm not here to flame about it.
instead i tried solving my xgps problem of always having to add the
arguments -speedunits kph -altunits meters.. the result is a patch that
sets those units to the local default... metric if LC_ALL or
LC_MEASUREMENT doesn't conaint "_US" and imperial if the substring is
found.

the patch also fixes the xgps -h output where "ft" instead of "feet" is
shown..

and since i'm quite a C-newbie you'd better look over the code ;)
i've tested it with quite some combinations (locales/args) and it
worked..
