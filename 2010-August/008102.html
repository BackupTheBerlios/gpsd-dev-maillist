> gpsd: unknown sentence:

> In a GSV sentence, an empty SNR field should be interpreted as zero.

Gpsd does it well with sat 29's SNR (last one) in this example:

gpsd: <= GPS: $GPGSV,1,1,04,30,73,046,27,12,40,082,32,02,27,054,22,29,73,222,*7A\x0d\x0a
gpsd: GSV: Satellite data OK (1 of 1).
gpsd: => client(0):
{"class":"SKY","tag":"GSV","device":"/dev/rfcomm0","satellites":[{"PRN":30,"el":73,"az":46,"ss":27,"used":false},{"PRN":12,"el":40,"az":82,"ss":32,"used":false},{"PRN":2,"el":27,"az":54,"ss":22,"used":false},{"PRN":29,"el":73,"az":222,"ss":0,"used":false}]}\x0d\x0a

Thus i suppose the problem only is with sentences where all SNRs
fields are null.
