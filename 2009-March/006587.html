> Second thing I was about to say that libgps is more lightweight than
> dbus, but at least from the shared library size this isn't true: The
> dbus library is smaller than the gps library (both stripped), which is
> probably caused by linking in all gpsd stuff into libgps.

That would be easily solved by breaking up libgpsd into one (very
small) library for clients and another (somewhat larger) library for
tools and the daemon.

I haven't done this only because in the typical case both client and
daemon are running on the same host, so all that code has to be in VM
anyway and there is little point in breaking it into smaller pieces.
