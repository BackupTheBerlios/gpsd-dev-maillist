Well, having looked into this a bit, I have the following to report:

1) There is *some* documentation for DeLorme's protocol to talk to the
EarthMate PN-20 and PN-40:

http://forum.delorme.com/viewtopic.php?p=50029#50029
http://forum.delorme.com/viewtopic.php?t=13846

The documentation is a little out of date - newer versions of firmware
have added some functions that aren't documented in the file that is
available above.  Also, I'm unsure what the intellectual property
status of the protocol description is, here's the first couple of
paragraphs from the doc that talks about that:

> The DeLorme Binary format - also known as DeLBin - is a proprietary
> bi-directional messaging format developed for DeLorme GPS
> receivers. It is used to send and receive various types of data,
> including navigation, waypoint, route, track, and mapping
> data. Currently, the Earthmate PN-20 is the only GPS receiver in
> DeLorme’s product line to support the DeLorme Binary format.
>
> The DeLorme Binary format compresses the transmitted data as much as
> possible without compromising resolution. Choosing a binary method
> over ASCII reduces bandwidth and increases transfer efficiency over
> limited bandwidth connections. Byte ordering is little endian (least
> significant byte first).
>
> Notes:
>     • DeLorme permits the use of the DeLorme Binary GPS Format on an
>       as is, where is basis, and without any representations or
>       warranties of any kind. This is an accomodation to skilled
>       users, and is not supported by existing DeLorme customer service
>       arrangements. Users who undertake to use the protocol do so at
>       their own risk.

2) I no longer have access to a PN-40 - I decided to return it for a
refund and purchase a GPSr from a different manufacturer.

Since I don't have a PN-40 anymore I won't be working on GPSD support
for it, but I thought that I'd add these notes for the archives for
the next person to come along.
