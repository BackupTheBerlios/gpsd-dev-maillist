"Gary E. Miller" <gem at rellim.com> writes:

> Interesting:
>
> # xsltproc --nonet http://docbook.sourceforge.net/release/xsl/current/manpages/docbook.xsl gps.xml
> warning: failed to load external entity "http://docbook.sourceforge.net/release/xsl/current/manpages/docbook.xsl"
> cannot parse http://docbook.sourceforge.net/release/xsl/current/manpages/docbook.xsl

I agree that we should have --nonet; having a build access a website is
at best a privacy violation.

I think what's going on is that libxslt is supposed to have files that
map the above URI to a local file, and an xslt installation is not
enough.

gdt 15 ~ > find /usr/pkg/share/xsl -type f |xargs egrep http://docbook.sourceforge.net/release/xsl/current/
/usr/pkg/share/xsl/docbook/catalog.xml:  <rewriteURI uriStartString="http://docbook.sourceforge.net/release/xsl/current/" rewritePrefix="./"/>
/usr/pkg/share/xsl/docbook/catalog.xml:  <rewriteSystem systemIdStartString="http://docbook.sourceforge.net/release/xsl/current/" rewritePrefix="./"/>
/usr/pkg/share/xsl/docbook/manpages/table.xsl:  <xsl:import href="http://docbook.sourceforge.net/release/xsl/current/html/docbook.xsl"/>
/usr/pkg/share/xsl/docbook/website/website-targets.xsl:         doctype-system="http://docbook.sourceforge.net/release/xsl/current/common/targetdatabase.dtd"/>

So this finds

  /usr/pkg/share/xsl/docbook/manpages/docbook.xsl

which is the needed style file.

On my (NetBSD pkgsrc) system, libxslt has xsltproc but not the above.

The docbook-xsl-1.74.0nb1 package contains 1585 files under
/usr/pkg/share/xsl.

So perhaps you need to find/install a docbook-xsl package of some kind.
