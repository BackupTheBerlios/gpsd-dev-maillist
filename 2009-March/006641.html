From the long expired rfc draft:

      if (scanf("%9lu",&len) < 1) barf();  /* >999999999 bytes is bad */
      if (getchar() != ':') barf();
      buf = malloc(len + 1);       /* malloc(0) is not portable */
      if (!buf) barf();
      if (fread(buf,1,len,stdin) < len) barf();
      if (getchar() != ',') barf();

You know what a "Sonntagsfahrer" is. This code is the same, it assumes 
everything is always fine. If not, it's over.

Once DJB starts working in the real world, things start looking a lot less 
elegant.

One of the unspoken premesis is that the amount of data available is unknown.
My particular beef is with: 
      if (fread(buf,1,len,stdin) < len) barf();
Once you make this work in the real world, any gains from using this scheme 
are lost.

I always say: it's hard to argue with success!
The opposite is also true ;-)
