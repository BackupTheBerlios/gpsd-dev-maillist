"Michael R. Davis" <mrdvt92 at yahoo.com> writes:

>> If we have a preference how the daemon should be started, we should communicate that. 
>
> I think we have.
>
> RPM
> PIDFILE=/var/run/gpsd.pid
> CONTROL_SOCKET=/var/run/gpsd.sock
> OPTIONS="-n -F $CONTROL_SOCKET -P $PIDFILE"
>
> PKG
> DAEMON_OPTS="-F /var/run/gpsd.sock"
>
> HOTPLUG
> CONTROL_SOCKET = "/var/run/gpsd.sock"
> gpsdcmd = "gpsd -F " + CONTROL_SOCKET
>
>
>> Fedora has local policy (settings) in /etc/sysconfig
>
> The RPM does not currently support this but it could.

(I acknowledge Eric's earlier point that we are talking about a default,
expecting that it is not right for all situations.  I had internallized
my impression of the 'there shall be no configuration' dogma so strongly
that I didn't get that we were talking about configuration.)

From the 'all the world is not Linux or rpm' camp:

After we decide on 'default behavior', then I think running 'gpsd
/dev/gps' should follow that default behavior.  The notion that 'gpsd
/dev/gps' has non-default behavior and every environment is supposed to
set up to do 'gpsd -n /dev/gps' seems like trouble waiting to happen.

pkgsrc has the same notions and could install a start script with the
necesary options to start with default behavior, but people start things
in various ways, and this seems to lead to inconsistency.
