Attached patches:

* use-correct-make-command.patch (attempt #2)

setup.py was hardcoded to always use 'make', even if the top-level make
was something different, like 'gmake'.  This patch causes setup.py to
use the proper make command.

* restore-python24-support.patch

The 'foo = value1 if condition else value2' syntax was introduced in
Python 2.5.  This patch restores Python 2.4 support by using a different
syntax.
