Ugh.  Unpleasantly heavyweight.  Typical of a bad XML design specified by
bureaucrats.
 
> I don't think gpsd should provide this service necessarily but it
> could provide all of the data so that an XML service could easily
> layer on top of GPSD.

And that will happen.  The AIVDM decoder is working, as you can verify by 
running an AIVDM log through gpsdecode.  Basically it just dumps the payload
fields in a CSV format, recovering string data as well as numeric. But that's 
good enough.  You could massage this into that XML trivially, if we had a
spec for it.

> It won't be long until we will have a location service protocol akin
