That's a braindaed behaviour of the distribution then. Debian and Ubuntu have
sane ways for such a transition, and I'd expect other distributions to have a
sane way of dependencies at least, so you can't install a new verison of libgps
without updating the packages which depend on it at the same time.

> 
> Well, if I take the glibc and X libraries as example... I can take a
> xterm binary 7 years old and it will run fine with a current
> installation. If that is possible for this libraries (which are a little
> bit more complex), it should be also possible for libgps.

Glibc uses symbol versioning, that's the reason why it was possible to avoid a
soname bump for a longer time. But libc7 will be there soon... For a project
like gpsd it doesn't make sense to use symbol versioning and the necessary overhead.
