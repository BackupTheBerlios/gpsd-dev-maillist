Hi,

Remco Treffkorn:
> He fails to label the container. Hence the feature of packaging one or
> more of these into one of these is useless unless you start making
> unspecified assumptions.

I disagree, because any real-world protocol will specify those
assumptions (OK, any _good_ real-world protocol).

As an example, some SMTP requests are followed by dot-escaped data, some
are not. The protocol spec needs to state which is which; the line
protocol just uses semi-arbitrary four-letter commands.

And that applies to almost any protocol in existence.

Even XML, which nobody in their right mind would accuse of not labeling
things, doesn't denote e.g. whether a text string is a string, or a
base-10 representation of a number.
