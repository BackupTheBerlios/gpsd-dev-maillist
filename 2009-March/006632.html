Matthias Urlichs <smurf at smurf.noris.de>:
> Another possibility would be Bernstein's len:BYTES; scheme for responses.
> (Thus the message "This is a test" would be encoded as "14:This is a test;"
> and a three-message response would look like "21:4:zero;3:two;5:three;;".)
> 
> Advantages:
> * can carry arbitrary binary data if required
>   * thus we could e.g. monitor the RTCM message stream
> * we can add CRLF filler after the semicolon for readability
> * can be nested
>   * no need to mark and process multiline responses in any special way

The Bernstein scheme is very elegant but has one killer disadvantage.
It's a pain in the ass to generate by hand.
