Hi,

Michael R. Davis:
> Lastly, I don't know how to solve this RPM management annoyance but
> gpsd-2.39dev-1.i386.rpm is "greater than" gpsd-2.39-1.i386.rpm
> rpm-wise.  So, in order to Update from a "dev" or "rc" version to the
> production version you have to erase old version then install the
> released version rather than just Update (e.g. rpm -Uhv gpsd*.rpm).
> Does anyone have any insight on this annoyance?
> 
Debian's DPKG solves this with a tilde, i.e. foo~dev is always sorted
before foo. Unfortunately, that's not particularly helpful in an RPM
context. :-P

I'd suggest the traditional even/odd scheme, i.e. the next release gets
to be numbered gpsd-2.40.

Another possibility is to name the release RPM gpsd-2.39rel-1.*
(dev < rc < rel(ease)), but IMHO that's somewhat ugly.
