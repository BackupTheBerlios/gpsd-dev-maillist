Yo Petr!

On Sat, 2 Jun 2007, Petr Slansky wrote:

> Current garmin.c does not probe for a GPS, it probably expects that GPS sends
> data again and again.

Not exactly, garmin.c actively probes for Garmin Binary on USB conencted
units.  The serial config part for Garmin Serial Binary got moved to
garminctl.  Inserting the Garmin Binary in the base gpsd was confusing
too many other GPS units.

> Do you know how to force gpsd to expect garmin data packet during detect call?

Nope.  A basic design goal of gpsd is zero config.  So far the Garmin
detection has worked pretty well.  If it is failing on yours we need to
find out why.  Garmin documentation on the Etrex would help.  Otherwise you
could up the logging to -D6 and try to spot where the parser gets lots.
