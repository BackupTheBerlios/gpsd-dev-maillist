On 6/6/07, Jon Schlueter <jon.schlueter at gmail.com> wrote:
> Hello,
>
> I just got a Ashtech G8 GPS Sensor that I would like to use GPSD to
> interface with.

Nifty. I've been working on getting an AC12 or CP14...

> gpsd: selecting Generic NMEA driver...
> gpsd: => GPS: $PSRF100,0,4800,8,1,0*0F\x0d

This is a SiRF probe.

> gpsd: => GPS: $PGRMCE*0E\x0d

This is a Garmin probe.

> gpsd: => GPS: $PFEC,GPint*58\x0d

This is a Furuno probe.

> gpsd: => GPS: $PFST*11\x0d

This is a FastraX probe.

> I have found the messages you need to send it to probe and identify the device
>
> $PASHQ,RID\r\n
>
> and then you can send the following to turn on RMC (or something very similar)
> $PASHS,NME,RMC
>
> What is my next step in getting this device to work with GPSD and be
> able to collect data from this GPS?

Have a look at how the aforementioned probes work. Basically we know
that a "Vendor X" receiver responds in a certain way to a "$PVNDX"
message. If we send $PVNDX and we get the right reply back, we know
that the device is a Vendor X receiver. From there we can then send
more specific Vendor X initialization configuration string. That may
be turning on a few more NMEA sentences, or that may be flipping it
over to binary mode, or whatever else...
