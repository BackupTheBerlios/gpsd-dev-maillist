OK, I've now committed my pure-C version of gpxlogger. It compiles
cleanly for me with "-ansi -Wall -pedantic" and seems to run properly.
It's completely self-contained, relying on the magic of strsep,
sscanf, and sockets. No PCRE, DBUS or libgps needed.

I chose to poll because no one message existed that gives position,
time, svs and HDOP all in one go.

Anyway, GPX-users, please test and comment. If I missed something that
causes your GPX applications to break, please let me know so I can fix
it.

And once more for the record, all of my BSD machines ship with perl
installed by default and python is an extra to be added from ports.
Which means on some BSDs, python is the extra runtime dependency. ;)
