Yes, it would probably be an improvement. But I'm so disgusted with
autotools that I am doing only the minimum necessary to keep our setup
working.  The next time we have a serious problem my plan is to say
"Screw it..." and move to scons.

OTOH, Python is not required for production use on the target, and my policy
is that it won't be.  I'm more than happy to use it for build tools,
test tools, and (platform-independent) code generators, but I am
mindful that a significant number of our customers are running on 
constrained systems where the overhead of a Python interpreter would
be too large.
