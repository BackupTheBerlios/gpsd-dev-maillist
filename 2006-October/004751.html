Petr Slansky <slansky at usa.net>:
> cgps has support for environment variable GPSD_UNITS. xgps ignores it. Could
> be this added to xgps?

I wouldn't object, but I have lots of other higher priority things to work on.

I think the reason this hasn't been done is that xgps was designed to have
its preferences set the X way, with X resources.
 
> It will be nice to have the same switches in both tools to handle
> units. Now they have very different command line switches to handle
> units. Like they come from two different sources. Aren't these part
> of the same package?

Yes, but they were written years apart by two different people.

I've actually been thinking recently that it would be nice to merge
xgps and cgps into a single test client that chooses its display method
depending on whether or not it's running under X.  

This would be a good small project for someone who wants to contribute
without getting deep into the grubby guts of the daemon.  I think I'll
add it as a to-do item.
