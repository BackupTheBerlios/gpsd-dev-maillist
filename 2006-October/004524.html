this is code from libgps.c to parse O message:

                        char eph[20], epv[20], track[20],speed[20],
climb[20];
                        char epd[20], eps[20], epc[20], mode[2];
                        int st = sscanf(sp+2,
                               "%8s %lf %lf %lf %lf %s %s %s %s %s %s %s %s %s
%s",
                                tag, &nf.time, &nf.ept,
                                &nf.latitude, &nf.longitude,
                                alt, eph, epv, track, speed, climb,
                                epd, eps, epc, mode);

Is there a safer way to parse O message? In theory current code is a way to

overflow.
