Yo Rob Jensen!

I have been asking about the TSIP and not getting anyone that knows the
code.  So I look at the code and see your name on it.  Have you been
listening here to the TSIP problems?  I hate to touch your code, but
there seems to be no other way to resolve this problem unless you have
a fix.

It seems like the TSIP packet recognizer is too easy to claim a packet.
Do you have any thoughts on how to improve this?

After looking at the code I was thinking maybe we could limit the TSIP
recognizer to just a few TSIP packets.  How about we check for these
packet IDs and matching lengths:
	0x41, GPS time, length 10
	0x42, Single Precision Fix, length 16
	0x43, Velocity Fix, length 20

All 3 are output every second by default.  Does this look good to you?
It should be simple to code in packet.c and I can whip it up unless have
a better idea.
