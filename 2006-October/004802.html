I don't mind using python as part of the build process (even though I
don't like the language). I run "make", some ugly shell, m4, python
and c does its thing, and at the end I get some small fast binaries
that do what I need them to.

I care less than half the nucleus of a hydrogen atom about our extra
development tools on the target systems though. While test tools are
extremely critical on the development machines, gpsprof, gpsfake and
valgrind mean precisely jack on some remote, sensor-like device.

Thinking like a user: why do I have to waste extra disk on stuff I'll
never use? If I need it, I'll just install it from the cd or wherever.

Gpspipe is 10K on disk, Python is at least 23MB, and that's before
installing any other packages.

Gpspipe uses 664K of memory, Python starts at 1.7M, just sitting there
doing nothing.
