So I don't see gpsd rejecting this sentence: it would say so (as you know).

Continuing on

gpsd: <= GPS: $PTNTHTM,285.1,N,-2.4,N,10.7,N,59.1,3844*24
gpsd: GPS has a fix (status=1, mode=2).
gpsd: => client(7): GPSD,O=PTNTHTM          ? 0.005          ?          ?   59.1
0        ?        ? 285.1000   10.700 -2.400             ?        ?        ? 2

All in all, this is to be expected: cgps makes the assumption that
you're reading from a gps.

gpsd altitude = TNT Dip Angle (59.1)
gpsd course = TNT True Heading (285.1)
gpsd speed = TNT Roll Angle (10.7)
gpsd climb = TNT Pitch Angle (-2.4)
gpsd mode = 2, because we've parsed as much as we can from this device.

So gpsd is handling this device as best it can. What you want is for
cgps to try display this?
