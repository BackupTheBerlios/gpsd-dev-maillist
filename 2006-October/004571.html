We already have GPSD_UNITS - it makes sense to pass in "none" if you
don't want libgps to do any unit conversion and just pass on whatever
the receiver sends. I think that should get implemented on general
principles. We can do some pretty printing if you like, but we'll also
stop and just pass you raw data if you know what you're after.

As for a client flag, that also means hacking xgps, and the various
loggers. In a way this strongly suggests that mapping non-gps data
into a gps message is the Wrong Thing to do. But cgps is your display
engine - I'll leave it to you to decide if you want to implement
additional user data transforms.

Just my two bits. More discussion would be good.
