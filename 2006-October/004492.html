> PRN numbers... Can gpsd can use it. Chris highligthed several dayes ago, taht
> gpsd uses 3
> SBAS PRN ids for SBAS satellites (xgps for example). I think that this
> translation should be used for GPGSA messages generated by gpsd at least...
> Mapping is simple, it is:
> NMEA PRN = SBAS PRN - 87;

To the user, the main identifier is of a GPS transmitter is PRN, not
SVN or any other arbitrary serial number. If a vendor wants to remap
PRNs fine, but we want the mapping so we can undo it. That way our
observations can be correlated against other receivers. It is my
feeling that if we know what the real PRN is, we should use it. Xgps
and cgps use the 3 digit PRN because that's the real PRN.

Page 7 (pdf page 21) of IS-GPS-200D specifically states: "PRN
sequences 33 through 37 are reserved for other uses (e.g. ground
transmitters)." Given that the people who assign PRNs *didn't*
allocate PRN 33 .. 37 to various SBAS applications, Evermore's NMEA
PRNs are utter bullshit, and need to be fixed. If anything this map
should be used to convert Evermore PRNs to the actual prn being
transmitted. I think garmin has some similar insanity going on
(Gary?).

Don't take my tone to mean I'm angry at you - I'm not. I am totally
incredulous that a vendor would violate the spec like that. I just
blows my mind. *sputters incoherently*
