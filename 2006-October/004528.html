> On 10/11/06, Petr Slansky <slansky at usa.net> wrote:
> > Is there a safer way to parse O message? In theory current code is a way to
> > buffer overflow, isnt't it?

I use a regular expression "/\s+/" but the Y lines are colon delimited vice white space delimited.
From: http://search.cpan.org/~mrdvt/Net-GPSD-0.25/lib/Net/GPSD.pm
 
  my @line=split(/[,\n\r]/, $line);  
  foreach (@line) {
    if (m/(.*)=(.*)/) {
      if ($1 eq 'Y') {
        $data{$1}=[split(/:/, $2)]; #Y is : delimited
      } else {
        $data{$1}=[map {$_ eq '?' ? undef() : $_} split(/\s+/, $2)];
      }
    }
  }
