My point of view is that it makes source code more compact. Problem of that
part that confused Eric was probably state variable, "ok", it was a little bit
confusing. But in many cases you can use that pseudo loop code to do nice job
without state variable, like here:

do {
  if (test1) break;
  if (test2) break;
  if (test3) break;
  if (test4) break;
  {do something interesting here as winner takes it all ;-) }
} while(0);

it is like 

if (!(test1 && test2 && test3 &&(test4))
{do something interesting};

or

if (!test1) 
  if (!test2) 
    if (!test3)
      if (!test4) 
      {do something interesting}

As this is not readable when test conditions are not easy, many use goto
statament in similar situations. I started with BASIC and after that wrote for
several years in assembly language and I still don't like goto statament too
much ;-) I am sorry for confussion.

It is possible that whitespaces will be removed by email agent and result will
be ugly. In that case I am sorry; I formated code examples in some way.
