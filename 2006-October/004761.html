> If this were 1994 and Linux had only static linking, I'd agree with you.

> with dynamic linking.  We have two major use cases:
>
> 1. Tiny systems that can't support X at all.

Like everyone running on WRT54G's. Like the Gumstix crowd.

I'd like to say "like my soekris" except I need to have a few X
libraries for the GD PHP module... :(

> 2. PCs where X applications are running essentially all the time.

Like my laptop. Vim/gvim does that... if X is up it'll run in its own
window. If not, it just does the console thing.

> About the only time we could potentially have an issue is if somebody
> happened to be running the merged client on a virtual console with
> X shut down,  Even then, modern VM systems probably wouldn't pull the
> X libs into the working set unless one of the entry pointers got chased.

I bet that if you link against an X library and then don't call any X
function, ld.so is probably still going to try map the library into
address space. Or at least it'll try find the library and save the
symbol resolution for later. This may give you some pain.
