On Tue, 31 Oct 2006, Eric S. Raymond wrote:

> You do weird shit like putting extra spaces around parens,

I'll cop to that. It is a bit odd.  Chinese programmers got me doing that.

> putting
> constants in comparisons with function returns *before* the comprison
> operators

Now this is one thing that I will defend strongly!  It seems to me this
is not a "style" issue (something that indent can handle) but rather
a very useful idiom.

Think about these two:

	if ( variable == 1 )
and
	if ( 1 == variable )

You are arguing for the former and you would be serious wrong. :-)

Think of the most common error in typing either of these, missing the second
"=".  I am sure you see this type of typos all the time. gpsd has suffered
from them occasionally.

Here is how the error would look:

	if ( variable = 1 )
	if ( 1 = variable )

One of those, your choice, will give you an unexpected surprise at run time.
The other, my choice, will fail to compile.  Which is going to be
easier to debug?  Which is easier, on its face, to know that no assignment
was ever intended?
