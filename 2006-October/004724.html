The spec requires "libXaw-devel" which is not available except on FC5
and FC6.  Should we separate out gpsd-clients-x?  Not sure why "make"
builds without any of that.  When I removed the libXaw-devel
requirement it would NOT build on my system.

Does anyone know a better way to test rpmbuild from the SVN other than?

mv trunk/ gpsd-2.34
tar cvfz gpsd-2.34.tar.gz gpsd-2.34/
mv gpsd-2.34.tar.gz /usr/src/redhat/SOURCES/
cp gpsd-2.34/gpsd.spec /usr/src/redhat/SPECS/
cd /usr/src/redhat/SPECS/
rpmbuild -ba gpsd.spec
