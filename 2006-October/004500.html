Other interesting feature of gpsd is different handling of binary and NMEA
GPSs receivers. In the case that receiver talks in binary protocol, gpsd has
to generate NMEA messages (for example for gpspipe -r output). In the case
that NMEA GPS is connected, gpsd just retransmit NMEA messages it receives. I
think that gpsd should work in the same way for all GPS receivers, it has to
regenerate NMEA messages from data it receives. Or at least some option can
control this behavior. In the case that connected NMEA GPS receiver has some
special behavior, gpsd will not hide it (different frequency of messages send,
etc). Possibly gpsd should send only special messages that gpsd don't
understand but basic NMEA messages (GPGGA, GPGSV, GPRMC, etc) should be build
from the ground using gpsd code. Without this we cannot talk about something
like "GPSD NMEA" compatible" applications ;-)
