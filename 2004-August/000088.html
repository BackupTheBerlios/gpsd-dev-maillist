Hi all

I tried to build a rpm for gpsd today and ran in some problems with
unclean paths, missing mkinstalldir and configure problems.
I were annoyed by the code and therefore converted gpsd to a gnu
autoconf/-make compliant environment.
Attached is a patch agains svn trunk.
The changes and advantages:
- it uses libtool
- it builds libgps as shared library
- no difficult makefile hacks needed anymore
- configure and Makefile.in are created by autogen.sh, which checks for
proper tool versions and then calls autoheader, automake and autoconf
- for a release you can do "make distcheck" which creates and then builds
a release tar.gz
- you can do "make release" which creates proper tar.gz and tar.bz2
archives

It would be nice if you would apply it.

Ciao,
	Jens
