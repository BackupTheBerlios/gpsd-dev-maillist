The automake-based build machinery is in the repo. There are some
loose ends, however:

1. gpsd.spec needs to have the machinery in it to install shared libs.

2. gpsd.spec should probaby be generated from a gpsd.spec.in template
   so the ciurrent version can be plugged in.

3. I see libgps.la and libgps.lo, but libgpsd_core.lo.  Should there
   be a pair of libgpsd.la and libgpsd.lo files instead?

4. autogen.sh spews a lot of obnoxious "underquoted symbol" messages.
   Is there some way to suppress these?

When we wrap these up I'll ship 1.96.
