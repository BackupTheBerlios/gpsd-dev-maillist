That's sort of lame. Either configure should test for them, or they should 
be non-fatal when they fail. Note that it's not just non-Linux that lacks 
these. I can install gpsd as not-root somewhere, and if I got the serial 
device chowned to me as console user, I can even run it (2947 isn't a 
privileged port, so, that's it). And I still have Linux machines in 
service which use /etc/rc.d/init.d, and don't have /etc/init.d.

Likewise, uninstall:
         rm -f /etc/init.d/gpsd
         /sbin/chkconfig --del gpsd

We sucked for not having install targets before, but this is lame the 
other way.
