Two comments re. gpsd 1.95:

- better advertize default command line values?
'gpsd -h' gives the available options. It would be nice if default 
values were indicated on the -h usage summary.
Specifically for -p,-S,-s,-D



- "GPS is confused about whether it has a fix"
Using -D3 I see the following message (gps antenna indoors, I can see
1-2 sats but it has trouble locking on..):

gpsd: <= GPS: $GPGSA,A,1,,,,,,,,,,,,,,,*1E
gpsd: GPGSA sets mode 1
gpsd: GPS has no fix (status=0, mode=1).
...
gpsd: <= GPS: $GPGLL,,,,,040657,*50
gpsd: GPGLL sets status 1
gpsd: GPS is confused about whether it has a fix (status=1, mode=1).

....
gpsd: <= GPS: $GPGGA,041819,,,,,0,00,,,M,,M,,*63
gpsd: GPGGA sets status 0
gpsd: GPS has no fix (status=0, mode=1).
gpsd: GPS has no fix (status=0, mode=1).
gpsd: => client(4): GPSD,S=0,P=?,A=?


I'm wondering is there is continuing confusion over what the mode and
status levels mean, or this is normal. From my notes:

# mode gpsdrive1.06b(0=NG, 1=Good)    cvs1.09(1=NG, 2=2D fix 3=3D fix)
