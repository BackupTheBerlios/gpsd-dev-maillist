Rich Gibson <rich at testingrange.com>:
> I don't believe that we can assume that the link between the GPS and 
> the host is good enough to not have whole-packet dropouts.  RF is 
> intrinsically subject to interference.  Then think sensor arrays with 
> hundreds or more GPS units reporting their positions via god knows what 
> method.

Fine.  The way we solve this problem is to document the undisputed fact
that if the host-to-GPS link is lossy enough to lose entire sentences,
this flag is not reliable.  The relevant portion of gps.h now reads:

    int	online;			/* 1 if GPS is on line, 0 if not.
				 * Note: gpsd clears this flag when sentences
				 * fail to show up within the GPS's normal
				 * send-cycle time. If the host-to-GPS 
				 * link is lossy enough to drop entire
				 * sentences, this flag will be
				 * prone to false negatives.
				 */
