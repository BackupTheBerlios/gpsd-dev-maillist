James Cameron <quozl at us.netrek.org>:
> Whoops.  Good catch.  I haven't comprehended the ifdef's for PRWIZCH yet.

PRWIZCH is a proprietary. sentence sent by Zodiac chipsets.  It reports
signal quality per channel.  I have not been able to find any specification
of the units, except that the range is 0-7.  Even the Zodiac interface
manual is no more enlightening.

I've been meaning to write about this, to sort of apologize to Remco.

At an earlier stage in the proceeeding (like, about a week ago) I
stepped on Remco's design for a I command because I thought it violated the "no
junk, no confusion" rule.   I pointed out that PRWIZCH could change
our picture of signal-quality values without changing the constellation data,
so having these two pieces of data cohere in the API would be a bad thing.

But my current Y has signal intensities in it.  as I said it shouldn't do.
Sorry, Remco!  This is because, later, I looked at PRWIZCH and asked myself 
"why in the heck are we supporting this at all?"

We already get signal quality from GPGSV.  Zodiac chipsets send GPGSV.
PRWIZCH seems redundant.  At the moment, there is still functional 
code to handle PRWIZCH but it's conditioned out.

Should I remove it entirely?
