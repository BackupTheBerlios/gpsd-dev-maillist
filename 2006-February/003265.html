Indeed it makes no sense to replace 4 by ERRLEVEL_4 which is defined as 4.

This does not mean there is no possible improvement...  when debugging
certain parts of gpsd I sometimes wished I could switch on only a small
group of debugs without getting everything at that "level".
It could be an idea to assign each report a code (could be the individual
report or a small group of report calls that logically belong together). 
These could still be assigned a "level" for quick selection of lots of
reports, but for detailed work one could look at a single report.

Simple implementation: instead of a level 4, assign all those calls at
level 4 a code like 4001 4002 4003 etc.   Level 5 is codes 5001 5002 etc.
When -D 4 is passed, all debugs at 4000 and above are shown.   When -D
4002 is passed, only code 4002 is shown.

Furthermode, it would have been nice if there was a command or signal to
change the debuglevel, to look at things that have gone wrong after the
daemon has been running for a while.

Also, with my multiple-receiver setup it would be nice to be able to
select debug for one receiver and not for another.  But that is much
harder to implement.
