Hello,

when gpsd_report is called, errlovel is put as number now. Why no define is
used for that?

Examle:
gpsd_report(4, "PRN=%2d El=%3.2f Az=%3.2f ss=%3d stat=%04x %c\n",

change to

gpsd_report(ERRLEVEL_4, "PRN=%2d El=%3.2f Az=%3.2f ss=%3d stat=%04x %c\n",

ERRLEVEL_4 is not too creative, it is just example...

There are several definitions of gpsd_report function. Is that needed? Why 
not to use only one definition?

gpsd-2.31# grep gpsd_report *c | grep void
gpsd.c:void gpsd_report(int errlevel, const char *fmt, ... )
gpsflash.c:void gpsd_report(int errlevel, const char *fmt, ... )
packet_test.c:void gpsd_report(int errlevel, const char *fmt, ... )
report.c:void gpsd_report(int errlevel UNUSED, const char *fmt, ... )
rtcmdecode.c:void gpsd_report(int errlevel, const char *fmt, ... )
