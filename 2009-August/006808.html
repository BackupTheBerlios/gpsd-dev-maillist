Chris Kuethe wrote:
> does uclibc have some distinctive preprocessor defines? if it does, we
> could do like we do for strl* - use the libc interfaces directly where
> they exist, otherwise provide local copies or a wrapper.

Is there any reason why you don't want to use isfinite instead of finite?
finite() is obsolete.  C99 defines macros isfinite(), which replaces finite().
