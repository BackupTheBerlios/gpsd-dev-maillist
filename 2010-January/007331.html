(Resent due to odd bounce from Mailman.)

IPv6 is now fully supported in the base Linux code of GPSD,
both clients and servers. Thanks to Olivier Mehani for doing
most of the coding work.

Developers running on OS X and *BSD, please pull trunk and test.
Before shipping 2.91, we'd like to be certain that it at least
builds on these systems.

Test results would be nice, too.  If you can start gpsd and see
results from "cgps '[::1]'" -- that is, by pointing the client at the
default port on the IPv6 loopback address -- it's working.  (The daemon
now listens on both IPv4 and IPv6.)
