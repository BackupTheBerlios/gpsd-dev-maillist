The libgps_core.c rejection was easily resolved; it doubled the size of the
privdata buffer, which I had previously done separately as a response to
Berlios bug #17034.  I'm guessing the *.i diffs were due to a line-ending
problem; if they are actually needed, please advise.

This patch requires documentation in the INSTALL file.  Please write a note
explaining:

1. What the target environments are (Linux? Windows? Both) 
2. Any build prerequisites (C++ compiler?  Qt library? Other libraries?)
3. What the new files libQgpsmm_global.h and libQgpsmm.pro are for,
   and how the are used.

I also need to know how to regression-test this code under Linux so
we can verify that it's not broken as the client library evolves.
