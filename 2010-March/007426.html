I just performed a git-filter-branch operation on the gpsd repo to
rewrite remnant Subversion IDs into references that actually make
sense - usually by committer and date, sometimes with key text from
the referenced commit.

It's not impossible we might change VCSes again in the future.  When
you refer to a commit, something like "esr's commit of 2010-04-22
01:05:00" is good practice and won't go stale.

Please re-clone the repo with

git clone git://git.berlios.de/gpsd 

Ordinary git pull won't work as most of the commit IDs have been
rewritten. Commit IDs are actually a hash of the revision content and
the parent IDs; this enables the grapg to be validated, but means that
whenever you modify a non-tip commit all the IDs downsream of it
change.
