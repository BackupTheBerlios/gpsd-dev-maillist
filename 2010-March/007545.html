Eric Raymond <esr at thyrsus.com> writes:

> Second, we need to fix the regular build so that if Qt and qmake
> are present on a Linux machine, the libQgpsmm library and an
> associated regression test get built by qmake and the regression
> test becomes part of the regular "make check" sequence.  This is necessary,
> otherwise the Qt support might get silently broken as the code evolves.

And we need a --disable-qt so that one can avoid this even if present.
Packaging systems need repeatable builds even if extra components are
present.  (pkgsrc does this by hiding the headers, libs and pkgconfig
files, but a --disable switch is easy.)

> This probably means you need to 
>
> 1. Teach configure.ac to detect Qt/qmake
>
> 2. Set an automake conditional flag appropriately
>
> 3. Use that conditional to wrap a Makefile.am production that
>    runs qmake to make the libQgpsmm library.
>
> Once you get that done, I can handle setting up the regression test.

Now that we're using git (or maybe anyway - it's not like svn didn't
have branching support), we should take advantage of branches, and put
new features like this in a branch so they can be tested for lack of
regressions on more platforms before being put in master.  Problems are
likely easily fixable, but I think master should be a little more stable
than it is.
