The point of the patch is not Qt nor C++, it is to have a win32 (and the
other Qt-supported platforms like symbian, wince and whatever port might
exists) compatible client library.

The POSIX socket api is not available under mingw/windows and it is replaced
by the portable Qt api. All the socket code is in libgps.

The patch could also benefit the KDE Marble win32 port, for instance..
