Nirgal <contact_gpsd at nirgal.com>:
> I've been using gpsdecode for a while, and I have a local patch I would like to get rid of: I have replaced the coma separator by a '|' character. This is because the ships destinations - amongst other fields - sometimes contain a coma, that makes the stdout results almost impossible to analyze. '|' is a good choice for me because it's not part of the AIVDM 6 bits alphabet. A tabulation would do, too.
> I imagine this is not a feature many people want, but still, it might be usefull to others.
> Ideally, we would have to quote fields containing comas, but then you need to choose how to escape quotes: May be a '\' but then you have to escape backslash too. I'm a bit lazy I guess.

I've changed gpsedode to use '|' all the time. I could have
backslash-escaped embedded commas, but I think it's better to make old
scripts fail in a way that's likely to be noticed than perpetuate a
situation in which unescaped commas could cause output to be unpacked
wrongly. I changed the Python decoder as well.

> By the way, one should modify dpsdecode.xml and specify that -u option is implicit when you don't have -j.

It already specifies this.
