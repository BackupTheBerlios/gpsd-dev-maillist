On Saturday, May 29, 2010, Jon Schlueter wrote:
...
> an Idea that came to me as I was looking at how the FD got into
> packet_get so that the UDP socket FD could be guarded to read
> everything available...
>

I thought about it some more. Since "an fd is an fd is an fd" is not
true any more, more information about the nature of the fs is needed.

I don't know what the udp read is used for, but I feel a bit queasy about
using read to read udp packages. This package could have been sent by
anybody. How do we safeguard against some malicious guy sending a
valid nmea sentence with bad information? Should we not use recvfrom
and check the address?

Should we have this information in the lexer struct so that packet_get
can do the right thing?
