Yeah, that'd make a *big* difference. I took a first look at scons
around the same time you were using (maybe a few nmonths earlier) it
and rejected it as promising but immature.

We did the Wesnoth conversion about nine months ago, at which point
scons had improved really dramatically, to the point where it was a
clear win over autotools.

I've occasionally checked on scons development since and it appears
they haven't slowed down.  Notably, a module to support generating
autotools-like config files for backwards compatibility, experimental
when I used it for Wesnoth, now seems to be part of base scons.
 
> IMHO the build system also needs to support cross builds and builds in

I think it does, but you're quite right.  Moving to scons, if we do
