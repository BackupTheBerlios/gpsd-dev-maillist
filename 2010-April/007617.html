> That sounds great - you can put an 'ignored' designator in the mode line
> and have C-x v v error out with 'this file is ignored'.

Exactly. And a good idea about the modeline designator, too.  I'll add
that to my to-do list for the feature.

The reason they ripped out C-x q, by the way, is because it coincides with
another command for toggling the read-only state of a file.  These notions
coincided in a useful way under locking VCSes, but diverge confusingly
on modern systems with no locking and merge orientation.
