On Sat, 17 Apr 2010, Eric Raymond wrote:

> These patches look good, but I get an error when I try to apply the first.
> I am a relative git newbie; is there something I can tell git am to do about
> these end-of-line whitespace errors? 

I believe these errors are symptoms of a mailer mangled patch.  I've 
had trouble sending and receiving inline patches before, so prefer to 
either send them as attachments or put up a repository with the 
patches on a branch.  The procedure which should work (just checked) 
for attachments is: save the attachments as files 0001-<...>, 
0002-<...> ... into your gpsd directory, and then run "git-am 000*" 
there.
