> Not to dis your test framework, which looks useful, but
> 
> Do you know about Simon Josefsson's autobuild?  It's essentially a way
> to do daily builds via cron but reported to a central place.
> 
> http://autobuild.josefsson.org/
> http://autobuild.josefsson.org/guile/
> 
> It may well be more comfortable for people, security policy wise, to set
> up to pull and make check than to allow shell access to a group.  (I
> realize one can argue it's not so different.)

That's very clever -- a build log aggregator for groups that are willing
to run cron jobs for each other but not allow shell access.

It has the limitation, though, that you can only get the results of
regression tests at preprogrammed times rather than on demand.  This
will chafe a little during periods of really aggressive hacking.
