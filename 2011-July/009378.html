> Summary:
> Is there a way to interface with gpsfake without extraneous NMEA data, just
> latitude and longitude?

gpsfake expects to see raw sensor data in one of the supported packet formats.

With the repository version of gpsd, JSON objects are a supported
"raw" format (this is so we can slave remote gpsd instances). You
might be able to hack your data into that.
