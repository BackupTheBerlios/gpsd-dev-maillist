Hi, I built gpsd-2.96 today on Mac OS X 10.6.8, which went off without a hitch, until I tried to run gpsprof, at which point I got the message "ImportError: No module named gps".

A little bit of digging turned up that "sudo make install" was putting python libraries in /usr/local/lib/python2.6/site-packages/, which is not in PYTHONPATH.

However, running "sudo python setup.py install" installed the libraries into the correct location, /Library/Python/2.6/site-packages/, and everything ran perfectly after that.

I don't know why the Makefile isn't getting the correct installation paths, but I thought somebody who knows more about the build system might be able to figure it out.

Here's the output of "sudo make install" -- http://pastebin.com/g1GHGDWb
Let me know if you need any more information.
