It looks like bad things can happen when the daemon reads a GSV
sequence that is missing one or more sentences from the front.  The
counter for the number of satellite fields gets set wrong, leading
to a bad GPSD sentence being passed to the client library, which
then bombs out during a scanf(3) call.
