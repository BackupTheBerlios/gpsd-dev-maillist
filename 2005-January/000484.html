> Speed-tuning advice: strdup() uses malloc and is therefore slow.  If
> you're only going to use the duplicate locally, use alloca() to
> dynamically allocate on the call stack.  This is faster, and also
> means you don't actually need a free() call if you can wait until
> function exit for deallocation.

Sure.  But alloca() is not ANSI C, and not portable to all
architectures, so I prefer malloc-based functions.  strdup() can be
emulated using ANSI C on platforms where it is missing, while alloca()
need compiler assistance. :)
