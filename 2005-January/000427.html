Um, thats because I don't have one, seeing that the debian package gets
installed by the standard package management tools. What I meant was
that I ship a stripped-down version of INSTALL as TROUBLESHOOTING.

I guess the short version of an INSTALL would be:

1. Run './configure --help' to see what options you have.
2. Run './configure' with any options you want to include (or without
any, in which case gpsd gets installed in /usr/local/bin, /usr/local/lib
etc. with the most common options enabled).
3. Run 'make'.
4. Run 'make install'. Make sure your user has the appropriate
permissions to do so.

The long version would be the one shipping with autoconf (attached, from
autoconf 2.59a).
