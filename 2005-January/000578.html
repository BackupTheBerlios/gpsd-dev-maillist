Only if you drove over to his house.  ;-)  I've fixed bugs in NMEA
parsers in the past that affected only Southern Hemisphere users, for
example.  Not a solution to the issue at hand; just quibbling over the
methodology.  

Running the original NMEA stream through the same version of the code
should be a good test; but the best test is to re-run the client
interaction as well, in case there's something the client does to change
the program state and thus the error cause.  

I agree that in the absense of reproducibility, a memory test might be a
good thing.  I use memtest.
