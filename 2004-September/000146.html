Hi,

I just tested the gpsd-1.96-1.i386.rpm from developer.berlios.de and ran into
some problems.  Firstly, the gpsd binary seem to be an autoconf/makefile shell
script placeholder!  eep!

also, since 1.95, the initscript has changed to check for the existence of
/dev/gps, but it uses test -f which looks for a regular file.  In my case I was
using a symlink to the correct device which it wouldn't recognise.  Perhaps use
[ -e /dev/gps ], or explicitly test for character devices or symlinks (if
you're paranoid).

1.95 works fine for me.

Hope this is useful,

John.
