Note, this is a request for comments on what happens *after 2.0*.

I have written a gpsd.py that does most of the low-level interface to
GPSes.  It's missing support for the non-NMEA types, notably Zodiac.
But it does a good enough job of NMEA parsing (thank you, Russ!) to
be the engine for gpsprobe.

I'm inviting comment on the idea that after 2.0 I should go the rest
of the way -- recode the daemon in Python and scrap the low-level
libgpsd(3) C code entirely.  If I did this, the only C code we would
distribute would be the high-level libgps(3) client library.

I estimate this would cut the distribution size in lines of code in
half.  It would also allow us to throw away all the configuration 
machinery.  Finally, it would render gpsd immune to overrun attacks.

The downside is that Python my be too big for some very small embedded
systems.

Discuss...
