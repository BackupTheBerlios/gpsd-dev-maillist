the project site here:

http://developer.berlios.de/projects/ttyproc/

Look for the latest code at around 5PM CST in the subversion tree.  I'm 
debating whether or not to make an official release package with reason 
below.

    From what I can tell it works fine, except for one really nasty bug.  If 
you use the path returned by ttyproc_getpath_node with a call to open(), the 
file descriptor returned is invalid and the system will crash hard when 
performing read/write access or termios calls on the fd.  However, 
ttyproc_getfd_node works fine.  It works when grabbing a file descriptor via 
open() from within the object library... no idea what is causing this one. 
I'm hoping someone with more programming experience can point out the root 
cause.
