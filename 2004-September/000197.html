Russell Nelson <nelson at crynwr.com>:
> It is.  I'd really like this sequence of events to happen:
> 
> 1) ship gpsd with every iPAQ running Linux.

I think we can get there.

> 2) autostart gpsd when a gps CF or SDIO card is inserted, or
>    when a Navman sleeve is inserted.

Autostart isn't what you want.  I've given this one a lot of thought, and
a conventional boot-time daemon that only opens the device when clients are
active is better.   Why?  Because it can always report the last fix, which 
can be useful even if it's old.

> 3) programmers presume that gpsd is present.

I think we can get there, too.

> 4) programmers include a place for GPS coordinates in all their data.
> 5) alarms can be spatial instead of merely temporal.
>    "Beep Beep Beep, if you leave now you can get to rowing in time."
