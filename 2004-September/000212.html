Bletch.  Well, then, let's try a different approach,  The part of that code
that really interests me is the device scanning.  Once I've got a udev, is
there any way to get a pathname or conventional read/write file descriptor 
from that?

I know that I can use termios(2) to set baud etc. on a USB device that I
have a file descriptor for, because I'm doing that now.  My goal is to
be able to write an initialization stage that does this:

1. If user specified a -p option, open that device

2. Otherwise, scan for a USB device matching any one of a list
   of specified vendor/product IDs.  If you find one, open that.

3. If no matching USB device is found, open /dev/gps.

4. Now set 4800/N/1 on the opened device using termios(2) and go.
