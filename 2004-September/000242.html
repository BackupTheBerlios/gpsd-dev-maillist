I got one more problem with /etc/init.d/gpsd initialisation script.
My GPS is operating at 9600 bauds instead of standard 4800, so the gpsd must 
be started with -s 9600 option. Some time ago I prepared similar init script 
which invoked gpsd in the following way:

gpsd $(cat /etc/gpsd.conf)

and /etc/gpsd.conf contained all needed options in single line,
something like (for gpsd 1.16):

-p /dev/ttyS0 -s 9600 -K

I am not a master of shell programing, so there may be better way of solving 
the problem of passing parameters to gpsd.
