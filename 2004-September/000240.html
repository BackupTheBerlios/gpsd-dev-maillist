Hamish, actually it's TTL level and polarity, just tying FV-18 to most
computers won't work, because the signal polarity isn't being inverted.
The MAX232 chip does the inversion in either direction.

"RS-232 (5V TTL)" is more ambiguous than I thought then.  Oops.

The RS-232 convention says that inputs should detect something like 0.6V
to -15V as the range for a binary one, and 2.4V to 15V as the range for
a binary zero.

The convention also says that outputs should provide -5V to -15V for a
binary one, and 5V to 15V for a binary zero.

The difference is there to allow for voltage drop and noise.

Note the detection range; this is why TTL level RS-232 that has been
pre-inverted does actually work, by direct connection, as with the
Garmin GPS 12XL.  I use the same technique with my temperature sensor
http://quozl.linux.org.au/ts/ which is powered from the serial port.
