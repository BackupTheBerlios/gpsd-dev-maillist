Quickly checking through the latest changes in gpsctl.c, I saw that the
control string escape code advances the buffer pointer too late. The
attached patch fixes this and adds the escape for backslash as well.

It's still possible to read behind the end of the command line argument,
though, if it ends with "\x" ;-)
