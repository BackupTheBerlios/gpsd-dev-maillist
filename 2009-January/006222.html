> >> At the moment I'm wondering how the detection of Garmin devices works on *BSD
> >> and other systems - do they have their own kernel module, too?
> 
> > The only support we have for GPS in the OS itself is a tty line discipline
> > that decodes NMEA data and extracts the time information (but not location
> > information; for that we have gpsd).
> 
> Wouldn't it make sense to move the tty stuff into userspace, using libusb? As

> kernel module breaks gpsbabel, which seems to talk directly to the device - I
> didn't look into the code yet, though (will have to do that anyway as I'll be
> the new gpsbabel maintainer on Debian...).

No, in the OpenBSD case this makes no sense, since we use the data from the
GPS in the kernel (we feed the time data to a timedelta sensor).

It's done with a line discipline, so a different process can still consume
the NMEA data, if needed.
