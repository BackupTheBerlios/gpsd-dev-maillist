On Jan 13, 2009, at 1:18 PM, Chris Kuethe wrote:

> Perhaps some environment variable? if you set "LIBTOOL='glibtool'"
> when running the autogen.sh, you get glibtool, otherwise you get
> libtool.

In the patch I gave I like that it works it out automatically. No need  
to figure out why something didn't work. We know the use case for the  
most part and cover it.

Another approach would have autogen.sh be interactive by default. I'm  
not sure if autogen.sh is always supposed to be non-interactive, but  
it could ask the user what to do if Darwin is encountered with  
glibtool available. Then allow for autogen.sh to still be run in non- 
interactive mode if the appropriate argument is given as well as  
arguments for what LIBTOOL should be and LIBTOOLIZE should be. If in  
non-interactive and no alternative arguments are given it could try to  
make the best decision available to it.

This all seems overly complex to me, I mean this is just  
autogen.sh... ;-)
