Sascha Silbe <sascha-ml-gis-gpsd at silbe.org> writes:

> On Tue, Jan 13, 2009 at 08:12:51AM -0500, Greg Troxel wrote:
>
>> if [ `uname` = 'Darwin' ] && [ `glibtool --version > /dev/null` -eq
>> 0 ]; then
> The quotes around Darwin are not necessary (it's a literal without any
> special character) but don't hurt either.
> The glibtool invocation, on the other hand, should be quoted, as it
> will return an empty string. But I'm surprised the check works at all,
> since "-eq" is for comparing integers.
>
> I'd suggest the following:
>
> if glibtool --version > /dev/null 2> /dev/null ; then
>   LIBTOOL=glibtool
> elif libtool --version > /dev/null 2> /dev/null ; then
>   LIBTOOL=libtool
> else
>   echo "libtool missing"
> fi
>
> If glibtool is available, it's surely the preferred tool on the
> current system, no need to check for Darwin. The check is fully
> POSIX-conform.

Good points - I agree.
