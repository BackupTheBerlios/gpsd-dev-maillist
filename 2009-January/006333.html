There is a new file, gpscap.ini, in the GPSD source tree.  It
is a capability database of the information about GPS vendors, 
engines and devices that was formerly expressed as the table at
<http://gpsd.berlios.de/hardware.html>.  That table is now
generated from the database.

Please examine the capability database.  It is in a readable format,
documented in a header comment.  If you can add to or correct any of
the information, please do so.  If you can add an entry for a GPS not
yet covered, please do that as well.

This database is intended for more than simply generating the 
hardware compatibility table. It already includes some information
not yet expressed in the web page, such as which Bluetooth-supporting
devices have the configuration glitch.

I hope to represent in it control-string capabilities associated with
different GPS engines.  The goal is to use this capability information to
drive gpsctl, our setting-tweaker utility for GPSes, in the same way
tset(1) is driven by a termcap or terminfo file.

I'm therefore taking nominations for which GPS capabilities can be
usefully added to the database.
