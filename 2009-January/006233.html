Found the following problem (quoted below) when attempting to build  
the rc1 just posted to the list.  And, just found the workaround as  
well.

On Mac OS X 10.5.6, "./configure" generates no errors, but "make"  
fails with the error below.

The solution seems to be to run configure with the command:

./configure --x-includes=/Developer/SDKs/MacOSX10.5.sdk/usr/X11R6/ 
include --x-libraries=/Developer/SDKs/MacOSX10.5.sdk/usr/X11R6/lib

... which I was able to find with some thrashing around, but it might  
be nice if a released version of configure would pick up the default  
location of the X11 libraries on OS X.

Just a thought.  It's now working here (at least to build), so will  
post any behavioural bugs I find.

Cheers,
Robin
