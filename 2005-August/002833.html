On 8/5/05, Rob Janssen <gpsd at xs4all.nl> wrote:
> I think it is good that gpsd saves the serial port status and restores it
> when exiting without finding anything, but when it has actually detected a
> receiver at a certain bitrate wouldn't it be better to leave the port at
> that setting?

So you want some sort of persistent storage of state?

I'm not sure that such a thing would work: my usb-serial dongle is
indistinguishable from my sirf3 gps: they're both the same chip, the
same revision, the same usb ids, etc. And if I were to hang a handheld
receiver off the dongle, it'd be tough to tell which was the gps and
which was the port...

Some days I plug in multiple usb serial things, and not in the same
order which would make it tough to tell which device to
(re)configure...

I can see the utility of this, but this is also anathema to the
philosopy that gpsd should be able to fully reconfigure itself at
runtime. Same as specifying the port settings with command line flags.
