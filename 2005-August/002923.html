"Eric S. Raymond" <esr at snark.thyrsus.com> writes:
> * The out-of-order GSV problem.  I think I believe Wolfgang's scenario
>   that this is a problem with data received while a PPS pulse is
>   holding CD high.  Question is, what we're going to do about it...

Later tests with a terminal emulator put some doubt into this
explanation.  Why would the terminal emulator show data while DCD is
off but gpsd doesn't.

Could the emulator be using the call-out device while gpsd is using
the call-in device?  (eg. /dev/cua0 vs. /dev/tty0 type issues?)

On some BSD's (freebsd, netbsd) there is a PPS interface to the UARTS.
You can basically register to receive timestamp information from the
kernel whenever the DCD wiggles.  If the problem shows up on a system
with this PPS capability it should be possible to add a debug printf
for "DCD went away" "DCD came back".  If those printf's bracket the
missing sentences we have the smoking gun.
