I can build, install, and use gpsd all without having python
installed. Python is not a hard gpsd dependency. Again, there are
examples of things to do with gpsd written in python, and some nice
bindings, but for the core purpose of making a gps available to
multiple applications via the net, there is no python or perl
dependency.

Anyway, this is why I stuck the script in contrib: it falls neatly
into the category of "it works for me, might be useful to some others,
though it may not be in preferred programing language XYZ".
