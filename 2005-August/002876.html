> How about adding a small tool to gpsd (or an option to gpsd itself) that
> puts a receiver back into its default mode?
> 
> Cheers
> Tobias
> 
> PS: I can post the required phrases for Nemerix receivers.

Good idea, but there's one wee tiny little problem: a receiver's
default mode is not exactly standard. I have (and seen) SiRF receivers
that default to NMEA at 4800, NMEA at 19200, SiRF at 38400 and SiRF at 57600
depending on their firmware. I'm sure that other receivers may have
the same issue.

I can see a tool that allows you to specify "NMEA" or "Binary" and a
speed, and reuses the probe logic from gpsd to set the protocol as you
request. Perhaps the tool could default to setting NMEA at 4800 because
certain clue-challenged software houses based in the pacific northwest
don't think a gps can run at any different speed[*], and because it
seems like most other NMEA things speak 4800 anyway.

[*]maybe this is an argument for not supplying such a tool. :)
