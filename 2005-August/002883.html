Yo Belgabor!

On Mon, 15 Aug 2005, Belgabor wrote:

> I'd propose the following:
> - Enhance 'n'. N=ALL (or something like it) switches to NMEA and turns
> everything on.

This would be very vendor specific.  Garmin/NMEA can do this without
by more work.  Not very usefull at 4800 bps.

Right now the Garmin/NMEA driver sets close to the minimum sentences to
get the required data for gpsd.  I suppose some listner may want some
other types of raw NMEA passed through, like VTG.

What are the reasons not to always force the GPS to it's fastest serial
rate?

> - Add a command line option to gpsd that connects to the gps (probably with
> some retrys), sends some specified commands (eg. B=4800,N=ALL) and exits.

Should this be in gpsd?  Maybe a standalone reset tool is best.  The main
reason to do this is to then use the gps with something other than gpsd.

> - As switching the baud rate requires user interaction, it may be better to
> leave it alone. If the user changed it, he should change it back =)

Needing every user to understand the nitty-gritty of his GPS is to
avoid the problem instead of fixing it.  Getting the Garmin/NMEA to
change baud rates is non-trivial.  As soon as I get it figured out
solidly I would prefer to code it in gpsd and then be able to forget
about it.
