> - Most NMEA sentences carry a timestamp, all common that carry a position
> (and as such are likely to be sent first) do. Therefore, if the first
> approach doesn't work, we could assume that whatever has the 'new'
> timestamp
> is the first sentence of a burst.

I suggested this approach too, but apparently some receivers send the
entire set of sentences with a timestamp referring to when the measurement
was taken, and other receivers send each sentence with a timestamp
indicating when the message was actually sent.
So the first class of receivers will send a series of messages with equal
timestamps that often end in .00 or .99, while the second class will send
all sorts of fractional second values for each different message.
This of course would be easily recognizable, but in that case it would be
more difficult to find first or last sentence.

When all timestamps are the same, the detection of "first message" is
simple.  It is the message that has a different timestamp from the one
before.
But what you actually want is a detection of "last message".  That is
harder, because you never know if another message with the same timestamp
will follow.
