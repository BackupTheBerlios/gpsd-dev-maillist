On 8/19/05, Rob Janssen <gpsd at xs4all.nl> wrote:
> > Belay that last... here's a better version of that patch, this time
> > explicitly testing for a return value of -1. From my read() manpage:
> >
> >      Error checks should explicitly test for -1.  Code such as
> >
> >              while ((nr = read(fd, buf, sizeof(buf))) > 0)
> >
> >      is not maximally portable, as some platforms allow for nbytes to
> > range
> >      between SSIZE_MAX and SIZE_MAX - 2, in which case the return value of
> > an
> >      error-free read() may appear as a negative number distinct from -1.
> 
> Of course this cannot happen when you read to a buffer less than 2GB in size.

Indeed. I thought the same thing, but I forgot to put that in my message.

Still, if we know that read returns -1 as a flag value to indicate
error, it makes sense to test for that particular flag value. If
that's not found, then we can start making decisions about the
reported datasize (0? +ve integer? less than requested read length?
etc...)
