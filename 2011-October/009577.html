Taking a step back, you first need to decide what you really want out of
one of these sites before you can properly plump for one.  There are a
bunch of reasonable options out there, although several support only one
type of VCS, so that can rule them out if you don't want to change
source code system (which I think is probably a given in this case!)

The way I see the various options is that

- you have a clear winner in the "I'm a programmer", "I use Git"
intersection and that's GitHub.  It's strength is it's a good solution
for coders because the front page is basically your code and you can get
stuck in rapidly from there.  It's programmer focused, eg you can
"subscribe" to the code, blog about the code, comment on the code, fork
the code privately and edit it, etc, etc

- On the flip side, github is a kind of "code blogging system" and
whilst that's great for programmers, its not necessarily the best choice
for all projects.  Github (and largely true of Gitorious) is only ok to
moderate at documentation (you get a wiki/website), it's (IMHO) weak at
issue tracking and release management and it has no mailing lists that I
know of.

- Sourceforge/Berlios/Savannah I personally loathe with a passion
because they are all (IMHO) clunky, ugly, slow as hell, extremely hard
to navigate and so technical/geeky looking as to make Github look
attractive...  At least several of them can be customised to show your
own web pages as the default project pages, which kind of gives you some
escape, but eventually you need to show the users the hub pages and you
are undone (I will concede that the current gpsd site hides the berlios
engine almost completely, it's a good example of escaping from the
underlying engine...!)

- Your other main option is a self hosted system, possibly using an
opensource management system, eg Redmine/Gitorious, etc.  This seems
unfashionable unless you happen to have the resources available and
prior experience, but the point is it's an option


- The final "option" is the easiest to overlook and that is that you
bond together your own structure from the individual best of breed
pieces... Eg right now you appear to mainly have a) a website (that
hides the underlying source code control, etc), b) Sourcecode control
(easy to outsource and only programmers care, so no problem to jump to
another site for code), c) mailing lists (again can be outsourced).  I
don't believe you currently use automated issue tracking, but again it
can be outsourced to a "best of breed" location if desired


My suggestion is that you have such straightforward requirements
(website, git repo, mailing list...) that you just pick the best
locations for each and use the website to pull the pieces together... 
Consider github for code.  Dunno about mailing lists. Github can also
host your website (it has a "whole site hosting" mode - see
http://pages.github.com/ ).  Other options of course also possible
