I'm soliciting advice.  Currently I see two possibilities:

1. Move to savannah.gnu.org. Use their bugtracker, their mailing list 
facilities, put the web pages there, etc.

Advantages:  Everything in one place, pretty nice web interface.

Disadvantages: Site run on a shoestring - could have failure modes
similar to Berlios.  And it's run by the FSF, which is prone to
political fits.

2. Move the repo and issue tracker to Google Code.  They don't allow
static web content, so that goes elsewhere (possibly under my hosting
space at ibiblio.org).  Google Groups sucks, so we host the mailing
list at librelist.

Advantages: Google isn't goinmg away any time soon.

Disadvantages: Using disconnected bits at several different places.
I don't think ibiblio is risky but librenet might be.

Opinions?  Alternatives?
