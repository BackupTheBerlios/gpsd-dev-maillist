However, I will leave a tiny hint in case you are inclined to read what
should have been a helpful message: Berlios and Savannah both *host*
mailman mailing lists, however, it's really unimportant that the mailing
lists are *hosted* by the same provider as your source code control.  By
splitting the hosting you can allocate each to the "best" provider for
each. 

Picking a random large project as an example: Ruby on rails uses Github
for issue tracking (moved from Lighthouse - wow!), github for code,
unknown hosting for website and blog, freenode for irc and google groups
for mail (yeah yuck, but I just subscribe as per mailman and it works
exactly the same as mailman...)

You could easily use SF for code hosting, Savannah for mailing lists and
wordpress for the website... Experiment a little!
