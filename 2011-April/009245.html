On 06/04/11 08:24, Bernd Zeimetz wrote:
> On 04/05/2011 11:08 PM, Diego Berge wrote:
>> -pyexec_PYTHON = gpscap.py
>> +python_PYTHON = gpscap.py
> 
> Unfortunately I fail to understand how this change should fix any
> packaging issues as it neither affects the rpm spec file nor is
> gpscap.py installed.

   It does get installed in openSUSE as part of the the gpsd-python
package. The openSUSE .spec is here:

> https://build.opensuse.org/package/view_file?file=gpsd.spec&package=gpsd&project=openSUSE%3AFactory

   The packaging issue being that the above .spec expects this to be in
%{python_sitelib} instead of %{python_sitearch}. Based on my very
limited understanding of where Python expects things to be, it looks
like this is the right choice.

   The Fedora packaging guidelines appear to back this up:
https://fedoraproject.org/wiki/Packaging/Python

   On the other hand, openSUSE's own packaging guidelines appear to
indicate that for a given package, either all should go in
%{python_sitelib}, or it goes in %{python_sitearch}.

   So, apart from any packaging issues, which one would be the correct
location for that gpscap.py script, as per Python usage and customs?

   And btw, does that do anything else apart from generating an HTML
table of installed drivers?
