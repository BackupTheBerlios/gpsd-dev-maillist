On Fri, Apr 8, 2011 at 16:41 PM, Eric Raymond <esr at snark.thyrsus.com> wrote:
> 5. (Tracker bug #17800) ntpshm more than four.  Gary Miller owns the ntpshm
> code. We should dispose of this feature request either with implementation
> or a reasoned decision not to do it.

I would prefer if gpsd would avoid growing the existing ntpshm support
beyond four units.  ntpd is expected to use the same 127.127.28.x
driver to read both the existing and proposed cleaned-up and
POSIX-shm-based layouts, and I have proposed using units 0-3 for the
existing ntpshm and the new interface for units 4 and higher.

Thanks for your consideration,
Dave Hart
