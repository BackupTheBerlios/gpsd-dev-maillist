Dan Kane <dan at southernkane.com>:
> One query - you say "Your local gpsd just passes those reports to you";
> I presume that you intend to drop all source-related data at that point
> (such that it is totally transparent to the receiving GPSD instance
> where the data came from). Would it not be useful to pass this source
> data on? For example, a client app that wished to map the Internet
> topology of a GPSD network.

Actually, it was my plan to retain the source data in the path attribute.  

Let's say your local gpsd knows about a remote server at gpsd://thyrsus.com/. 
When you issue a ?WATCH, your local gpsd should issue a 

?WATCH={"path-prefix":"gpsd://thyrsus.com/"}

to the remote system at thyrsus.com. What this command says is "I want
you to prepend http://thyrsus.com/ to the path attribute in every 
response you send me, because that's my name for you."

Think about the implications...
