GPSD Folks,
I've gotten most of the Perl lib written and have a couple of comments on the new protocol.  I went with the Net::GPSD3 name space.

First the documentation does not have any examples of the input protocol.

?WATCH={"enable":true,"raw":1};

this would have save me some time...

The SKY class returns a list of satellites these should be a class as well.  I recommend something like "class":"SATELLITE".  Any reason for not using mixed case?  "Satellite" is so much easier on the eyes and in Perl all upper case classes are considered "reserved".  Currently, all of the gpsd classes are directly mapped in to the perl namespace Net::GPSD3::Return::*.

"TVP" is a locally used term.  We use PVT.  Let's use a term for the class name like "FIX".  Technically, "TPV" is probably the most accurate since you get time first, then position, then velocity.
