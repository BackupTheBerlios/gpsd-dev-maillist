I agree with Dan that package management is very important and in non-funded open source project it falls upon the person who wants it the most.  In this case, I believe that needs to be gpsd.  

However, I hate to say that it appears that I am now the maintainer of the Fedora SPEC file.  And, I admit, I have no idea what I'm really doing when it comes to portable RPMs nor how to get the package on the Fedora distribution list.  The current (SVN 6257) spec file does compile with `rpmbuild -ta gpsd-2.40dev.tar.gz`  However, `make dist-rpm` is currently broken in SVN (due to distdir error).  However, I personally deploy my GPSD via /etc/init.d/gpsd where other maintainers appear to use USB hot plugging.  I have not figured out how to truly handle that difference in the RPMs.  I guess since I ONLY use the TCP interface I want the TCP listener up all the time.  I believe other users ONLY use the lib interface.  I can only assume that the lib interface handles intermittent connectivity for the application.
