On /dev/ttyS0 is a motorola oncore, and on /dev/ttyS3 a garmin lvc.  When 
gpsctl is run without arguments, it most often fails to identify the 
device when running vs gpsd, but succeeds when doing it itself, lowlevel. 
Is this the expected behaviour?

-- gpsd running on /dev/ttyS0

# ./gpsctl
gpsctl: /dev/ttyS0 identified as  at -1

-- gpsd stopped

# ./gpsctl /dev/ttyS0
gpsctl: no gpsd running or network error: can't connect to host/port pair.
gpsctl: /dev/ttyS0 identified as a oncore binary at 9600.

-- gpsd running on /dev/ttyS3

# ./gpsctl
gpsctl: /dev/ttyS3 identified as  at -1

-- gpsd stopped

# ./gpsctl /dev/ttyS3
gpsctl: no gpsd running or network error: can't connect to host/port pair.
gpsctl: /dev/ttyS3 identified as a Garmin USB binary at 9600.

The attachments are from './gpsctl -D 2' with gpsd running on /dev/ttyS3. 
With the oncore device, I did not manage to get a ID from gpsd, with the 
garmin in about 1 out of 10 attempts.  First it looked like it would only 
succeed when connecting as the first client after gpsd restart, but this 
is not the case.

Håkan
